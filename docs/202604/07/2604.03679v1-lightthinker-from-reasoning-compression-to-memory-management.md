## 摘要
大语言模型（LLMs）在复杂推理方面表现卓越，但其效率受到长思维轨迹带来的激增认知开销的限制。在本文中，我们提出了 LightThinker，这是一种使 LLMs 能够将中间思维动态压缩为紧凑语义表示的方法。然而，静态压缩在处理复杂推理时往往面临困难，因为中间细节的不可逆损失可能导致逻辑瓶颈。为解决这一问题，我们将该框架演进为 LightThinker++，引入了显式自适应内存管理。该范式通过整合显式内存原语转向行为级管理，并由专门的轨迹合成流水线支持，以训练有目的的内存调度。广泛的实验证明了该框架在三个维度上的通用性：(1) LightThinker 在精度损失极小的情况下，将峰值 Token 使用量减少了 70%，推理时间缩短了 26%；(2) 在标准推理中，LightThinker++ 在相同上下文预算下将峰值 Token 使用量削减了 69.9%，同时在追求最大性能时实现了 +2.42% 的准确率提升；(3) 最值得注意的是，在长程智能体任务中，它在超过 80 轮的运行中保持了稳定的资源占用（减少了 60%-70%），在不同复杂场景下实现了 14.8% 的平均性能提升。总之，我们的工作为以极低开销在长时程内维持深度 LLM 推理提供了一个可扩展的方向。

## Abstract
Large language models (LLMs) excel at complex reasoning, yet their efficiency is limited by the surging cognitive overhead of long thought traces. In this paper, we propose LightThinker, a method that enables LLMs to dynamically compress intermediate thoughts into compact semantic representations. However, static compression often struggles with complex reasoning where the irreversible loss of intermediate details can lead to logical bottlenecks. To address this, we evolve the framework into LightThinker++, introducing Explicit Adaptive Memory Management. This paradigm shifts to behavioral-level management by incorporating explicit memory primitives, supported by a specialized trajectory synthesis pipeline to train purposeful memory scheduling. Extensive experiments demonstrate the framework's versatility across three dimensions. (1) LightThinker reduces peak token usage by 70% and inference time by 26% with minimal accuracy loss. (2) In standard reasoning, LightThinker++ slashes peak token usage by 69.9% while yielding a +2.42% accuracy gain under the same context budget for maximum performance. (3) Most notably, in long-horizon agentic tasks, it maintains a stable footprint beyond 80 rounds (a 60%-70% reduction), achieving an average performance gain of 14.8% across different complex scenarios. Overall, our work provides a scalable direction for sustaining deep LLM reasoning over extended horizons with minimal overhead.

*   **研究背景**：随着大语言模型（LLM）推理模式从“快思考”演进为以 OpenAI o1 为代表的“慢思考”（通过思维链 CoT 进行反复迭代、纠错和推理），模型生成的 Token 数量呈爆炸式增长。

*   **研究动机**：借鉴人类的“认知经济性”，即在思考时不保留每一个字，而是存储关键结论（摘要），仅在遇到逻辑瓶颈时才回溯细节。

## 2. 方法论：核心思想与技术细节
论文提出了两个阶段的演进框架：

### LightThinker（隐式思维压缩）
*   **核心思想**：在表示层进行优化，将长思维段落压缩为少量的隐藏状态（Gist Tokens）。
*   **关键技术**：
    *   **思维级注意力掩码**：设计特殊的 Mask，使模型在压缩阶段只能看到当前思维块，而在后续生成阶段只能看到压缩后的“缓存 Token”，从而强制模型将关键信息提取到极少数 Token 中。
    *   **解耦设计**：引入专门的 `[c]`（压缩 Token）和 `[o]`（输出 Token），分离压缩与推理职责。

### LightThinker++（显式自适应内存管理）
*   **核心思想**：将压缩从“表示层”提升到“行为层”，赋予模型主动管理内存的能力。
*   **关键原语**：
    *   `commit`：将当前详细推理归档为语义摘要，释放原始 Token 占用的空间。
    *   `expand`：当模型发现摘要不足以支撑后续逻辑时，主动调回并展开历史详细推理。
    *   `fold`：在利用完细节后，重新将其折叠以保持上下文简洁。
*   **轨迹合成流水线**：利用强模型（如 DeepSeek-V3）在模拟的内存受限环境下合成带有内存管理动作的专家数据，通过 SFT 使模型学会何时压缩、何时回溯。

*   **极致压缩**：LightThinker 在 Qwen 模型上减少了 70% 的峰值 Token 使用和 26% 的推理时间，准确率损失仅约 1%。

*   **长程鲁棒性**：在智能体任务中，Vanilla 模型在 60 轮后上下文会膨胀至 100k，而 LightThinker++ 稳定在 30k-40k，平均性能提升 14.8%，在硬样本上性能翻倍。

## 7. 优点与亮点
*   **解耦推理深度与显存**：通过显式管理，使模型能够处理远超其物理上下文窗口限制的复杂任务。
*   **可解释性与可控性**：显式的 `expand/fold` 动作让模型的思考过程透明化，用户可以清晰看到模型何时在调取记忆。
*   **双向调节**：解决了以往压缩方法“一旦压缩、细节永失”的痛点，通过回溯机制保证了逻辑的严密性。

## 8. 不足与局限
*   **数值敏感度**：在 GSM8K 等对数字极度敏感的任务中，隐式压缩仍可能导致关键数值丢失（如 Case Study 所示）。
