## 摘要
检索增强生成（RAG）系统在当代大语言模型（LLM）中得到了广泛应用，因为它们能够在提高生成质量的同时减少所需的输入上下文长度。在这项工作中，我们专注于用于长文档问答的 RAG 系统。当前的方法过度依赖 LLM 生成的描述，导致资源消耗高、延迟大、层级间内容重复，并且由于缺乏或仅有有限的源文本锚定而产生幻觉。为了通过锚定提高效率和事实准确性，我们提出了 GroundedKG-RAG，这是一种知识图谱显式提取自源文档并锚定于其中的 RAG 系统。具体而言，我们将 GroundedKG 中的节点定义为实体和动作，将边定义为时间或语义关系，每个节点和边都锚定在原始句子中。我们利用语义角色标注（SRL）和抽象语义表示（AMR）解析构建 GroundedKG，然后对其进行嵌入以进行检索。在查询过程中，我们对查询应用相同的转换，并从锚定的源文本中检索最相关的句子进行问答。我们在 NarrativeQA 数据集的示例上评估了 GroundedKG-RAG，发现它在成本更低的情况下，性能与最先进的商业长上下文模型相当，并优于具有竞争力的基准模型。此外，我们的 GroundedKG 具有可解释性和人类可读性，便于结果审计和错误分析。

## Abstract
Retrieval-augmented generation (RAG) systems have been widely adopted in contemporary large language models (LLMs) due to their ability to improve generation quality while reducing the required input context length. In this work, we focus on RAG systems for long-document question answering. Current approaches suffer from a heavy reliance on LLM descriptions resulting in high resource consumption and latency, repetitive content across hierarchical levels, and hallucinations due to no or limited grounding in the source text. To improve both efficiency and factual accuracy through grounding, we propose GroundedKG-RAG, a RAG system in which the knowledge graph is explicitly extracted from and grounded in the source document. Specifically, we define nodes in GroundedKG as entities and actions, and edges as temporal or semantic relations, with each node and edge grounded in the original sentences. We construct GroundedKG from semantic role labeling (SRL) and abstract meaning representation (AMR) parses and then embed it for retrieval. During querying, we apply the same transformation to the query and retrieve the most relevant sentences from the grounded source text for question answering. We evaluate GroundedKG-RAG on examples from the NarrativeQA dataset and find that it performs on par with a state-of-the art proprietary long-context model at smaller cost and outperforms a competitive baseline. Additionally, our GroundedKG is interpretable and readable by humans, facilitating auditing of results and error analysis.

*   **核心问题**：针对长文档（如整本书籍）的问答任务，现有的检索增强生成（RAG）系统存在三大痛点：

### 2. 方法论：GroundedKG-RAG
核心思想是构建一个**完全锚定在原文句子上**的知识图谱（KG），并以此作为检索索引。
*   **图定义**：
    *   **节点（Nodes）**：代表实体（Entities）和动作（Actions）。
    *   **边（Edges）**：代表语义关系（如施事者、受事者）或动作之间的时间顺序关系。
    *   **锚定属性**：每个节点和边都包含 `grounded_texts` 属性，直接指向原始文档中的具体句子。
*   **构建流程**：
    1.  **预处理**：进行指代消解（Coreference Resolution），确保实体一致性。
    2.  **语义解析**：利用 **SRL（语义角色标注）** 或 **AMR（抽象语义表示）** 将句子转化为结构化表示。实验证明 AMR 在处理概念抽象和去重方面表现更好。
    3.  **图构建**：将解析结果转化为有向图，并引入时间边连接同一句子或相邻句子中的动作。
*   **检索与生成**：
    1.  **查询转换**：将用户查询也转化为查询图（Query Graph）。
    2.  **节点嵌入与匹配**：使用轻量级模型（如 `all-MiniLM-L6-v2`）对节点进行嵌入。探索了基础嵌入、平均邻居嵌入和注意力邻居嵌入三种方式。
    3.  **文本提取**：根据匹配到的 Top-K 节点，提取其锚定的原始句子作为上下文。
    4.  **LLM 生成**：将提取的句子输入 LLM（如 GPT-5 系列）生成最终答案。

### 6. 主要结论与发现
*   **性能优异**：GroundedKG-RAG 在所有指标上均优于 GraphRAG。在短篇文档中，其表现甚至超过了全文本输入（Full Context）。
*   **AMR 胜过 SRL**：AMR 解析能更好地识别语义概念并减少节点冗余，从而提高检索准确率。
*   **锚定的重要性**：直接检索原文句子比检索 LLM 生成的摘要更能减少幻觉，ROUGE-L 指标的显著提升证明了这一点。
*   **过滤策略的负面影响**：简单的向量相似度过滤或频次过滤反而可能剔除关键证据句，导致性能下降。

### 7. 优点
*   **高事实准确性**：通过显式锚定原文，确保答案的每一个环节都有据可查，极大降低了 RAG 的幻觉风险。
*   **成本效益高**：索引构建不依赖昂贵的 LLM 调用，适合大规模文档处理。
*   **可解释性强**：构建的图谱是人类可读的，开发者可以轻松审计检索路径并进行错误分析。
*   **查询自适应**：相比固定层级树结构，基于图的检索能更灵活地根据查询聚合信息。

### 8. 不足与局限
*   **长文档挑战**：在处理超长文档（如《鲁滨逊漂流记》）时，BERTScore 仍落后于全文本输入，说明大规模图检索的精度仍有提升空间。
*   **解析器依赖**：系统的性能高度依赖于 SRL/AMR 解析器的准确性，解析错误会直接传导至后续环节。
*   **节点匹配精度**：目前的节点匹配主要依赖简单的向量相似度，对于复杂逻辑关系的匹配（如“谁的父亲”）仍存在困难。
*   **上下文碎片化**：检索到的句子可能存在碎片化问题，导致 LLM 在生成时面临信息不连贯的挑战。
