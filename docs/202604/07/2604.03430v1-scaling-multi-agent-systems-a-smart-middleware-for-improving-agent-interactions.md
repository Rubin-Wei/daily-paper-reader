## 摘要
随着基于大语言模型（LLM）的多智能体系统（MAS）从实验性试点演变为复杂且持久的生态系统，智能体间直接通信的局限性变得愈发明显。当前的架构受困于上下文碎片化、随机幻觉、僵化的安全边界以及低效的拓扑管理。本文引入了认知织网节点（Cognitive Fabric Nodes, CFN），这是一种新型中间件层，旨在智能体之间构建一个无处不在的“认知织网”。与传统的消息队列或服务网格不同，CFN 并非单纯的透传机制，而是主动的智能中介。该架构的核心在于将记忆（Memory）从简单的存储提升为主动的功能基质，为其他四项关键能力提供支持：拓扑选择、语义对齐（Semantic Grounding）、安全策略执行和提示词转换。我们提出，这些功能应分别由利用强化学习（RL）和优化算法的学习模块进行管理，以动态提升系统性能。通过拦截、分析和重写智能体间的通信，认知织网在确保单个智能体保持轻量化的同时，使整个生态系统实现了连贯性、安全性和语义一致性。我们在多智能体环境下的 HotPotQA 和 MuSiQue 数据集上评估了 CFN 的有效性，结果表明，相比于智能体间的直接通信，CFN 在这两个数据集上的性能提升均超过了 10%。

## Abstract
As Large Language Model (LLM) based Multi-Agent Systems (MAS) evolve from experimental pilots to complex, persistent ecosystems, the limitations of direct agent-to-agent communication have become increasingly apparent. Current architectures suffer from fragmented context, stochastic hallucinations, rigid security boundaries, and inefficient topology management. This paper introduces Cognitive Fabric Nodes (CFN), a novel middleware layer that creates an omnipresent "Cognitive Fabric" between agents. Unlike traditional message queues or service meshes, CFNs are not merely pass-through mechanisms; they are active, intelligent intermediaries.   Central to this architecture is the elevation of Memory from simple storage to an active functional substrate that informs four other critical capabilities: Topology Selection, Semantic Grounding, Security Policy Enforcement, and Prompt Transformation. We propose that each of these functions be governed by learning modules utilizing Reinforcement Learning (RL) and optimization algorithms to improve system performance dynamically. By intercepting, analyzing, and rewriting inter-agent communication, the Cognitive Fabric ensures that individual agents remain lightweight while the ecosystem achieves coherence, safety, and semantic alignment.   We evaluate the effectiveness of the CFN on the HotPotQA and MuSiQue datasets in a multi-agent environment and demonstrate that the CFN improves performance by more than 10\% on both datasets over direct agent to agent communication.

论文提出了**认知织网节点（Cognitive Fabric Nodes, CFN）**，作为一种智能中间件层，将通信从简单的“消息传递”提升为“语义处理”，确保系统在扩展时保持连贯、安全和高效。

### 2. 方法论：核心思想与关键技术
CFN 的核心思想是将**记忆（Memory）**从被动存储提升为主动的“功能基质”，并以此驱动四大核心模块。

*   **核心架构：五大支柱**
    1.  **主动记忆（Active Memory）**：作为共享的“世界模型”，为所有操作提供上下文。
    2.  **拓扑选择（Topology Selection）**：基于意图（Intent-based）而非地址的路由。利用强化学习（RL）将任务映射到最合适的智能体。
    3.  **语义对齐（Semantic Grounding）**：验证消息是否符合共享记忆中的事实，防止“幽灵实体”传播，并进行本体转换（如将“客户ID”翻译为数据库能理解的“UUID”）。
    4.  **安全策略执行（Security Enforcement）**：采用“混合守护者”模式，结合确定性规则（如PII脱敏）和概率性认知防御（检测复杂的提示词注入）。
    5.  **转换与重写（Transformation & Re-writing）**：这是执行层。公式定义为 $y_j = \Phi(x_{i \to j}, M_{state}, G_{context}, S_{policy} | \theta)$，即根据记忆、语义和安全策略，将原始提示词重写为更易被接收者理解的高质量指令。

*   **算法流程**：
    *   CFN 拦截智能体 A 发出的原始消息。
    *   通过认知引擎（Cognitive Engine）进行上下文注入、安全过滤和语义对齐。
    *   利用强化学习（如 Contextual Bandit 框架）优化重写策略，目标是最小化接收者的“认知摩擦”并提高任务成功率。

### 6. 主要结论与发现
*   **性能显著提升**：在 HotPotQA 和 MuSiQue 上，CFN 架构比直接通信模式性能提升了 **10% 以上**。

### 7. 优点（亮点）
*   **架构创新**：借鉴了微服务中的“服务网格”概念，但将其升华为“语义网格”，填补了 MAS 基础设施的空白。
*   **主动记忆机制**：将记忆视为一种主动干预手段而非静态数据库，有效解决了多智能体系统中的本体漂移问题。
*   **解耦设计**：实现了意图与实现的解耦，使智能体能够保持轻量化，而将复杂的协调逻辑下沉到网络层。

### 8. 不足与局限
*   **认知税（Latency）**：引入中间件必然增加推理延迟。虽然论文提出了混合部署模式，但在实时性要求极高的场景下可能存在瓶颈。

*   **实验覆盖面**：安全模块和拓扑选择模块的有效性更多停留在理论描述和示例展示，缺乏像 QA 任务那样的大规模定量评估。
*   **依赖性**：系统性能高度依赖于认知引擎（重写模型）的质量，如果重写模型产生偏差，可能会误导整个智能体集群。
