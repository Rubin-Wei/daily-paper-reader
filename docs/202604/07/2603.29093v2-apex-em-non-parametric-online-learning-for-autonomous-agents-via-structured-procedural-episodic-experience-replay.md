## 摘要
基于大语言模型（LLM）的自主智能体缺乏持久的程序性记忆：即使之前已经解决过结构完全相同的任务，它们仍会从头开始重新推导解决方案。我们提出了 APEX-EM，这是一种非参数在线学习框架，能够在不修改模型权重的情况下积累、检索和重用结构化程序计划。APEX-EM 引入了：(1) 一种结构化经验表示，编码了每次执行的完整程序-情境轨迹——包括规划步骤、中间产物、带有错误分析的迭代历史以及质量评分；(2) 一种包含任务验证器（Task Verifiers）的“计划-检索-生成-迭代-摄取”（PRGII）工作流，提供多维奖励信号；(3) 一种具有双重结果的经验记忆（Experience Memory），采用结合了语义搜索、结构特征匹配和计划有向无环图（DAG）遍历的混合检索方式——实现了在词汇无重叠但操作结构相似的任务之间的跨域迁移。成功的经验作为正向上下文示例；失败的经验作为带有结构化错误标注的负面示例。我们使用 Claude Sonnet 4.5 和 Opus 4.5 在 BigCodeBench、KGQAGen-10k 和 Humanity's Last Exam (HLE) 上进行了评估。在 KGQAGen-10k 上，APEX-EM 达到了 89.6% 的准确率，而无记忆时仅为 41.3%（提升 48.3 个百分点），超过了真值检索（oracle-retrieval）的上限（84.9%）。在 BigCodeBench 上，它将成功率（SR）从 53.9% 的基准提升至 83.3%（提升 29.4 个百分点），在骨干模型冻结的同等条件下，超过了 MemRL 11.0 个百分点的增益（注：我们的分析中对骨干模型的差异进行了控制）。在 HLE 上，实体图检索率从 25.2% 提升至 48.0%（提升 22.8 个百分点）。消融实验表明，各组件的价值取决于具体任务：丰富的评判反馈对代码生成的影响微乎其微，但对结构化查询至关重要（提升 10.3 个百分点），而二值信号迭代可以部分补偿较弱的反馈。

## Abstract
LLM-based autonomous agents lack persistent procedural memory: they re-derive solutions from scratch even when structurally identical tasks have been solved before. We present APEX-EM, a non-parametric online learning framework that accumulates, retrieves, and reuses structured procedural plans without modifying model weights. APEX-EM introduces: (1) a structured experience representation encoding the full procedural-episodic trace of each execution -- planning steps, artifacts, iteration history with error analysis, and quality scores; (2) a Plan-Retrieve-Generate-Iterate-Ingest (PRGII) workflow with Task Verifiers providing multi-dimensional reward signals; and (3) a dual-outcome Experience Memory with hybrid retrieval combining semantic search, structural signature matching, and plan DAG traversal -- enabling cross-domain transfer between tasks sharing no lexical overlap but analogous operational structure. Successful experiences serve as positive in-context examples; failures as negative examples with structured error annotations.   We evaluate on BigCodeBench, KGQAGen-10k, and Humanity's Last Exam using Claude Sonnet 4.5 and Opus 4.5. On KGQAGen-10k, APEX-EM achieves 89.6% accuracy versus 41.3% without memory (+48.3pp), surpassing the oracle-retrieval upper bound (84.9%). On BigCodeBench, it reaches 83.3% SR from a 53.9% baseline (+29.4pp), exceeding MemRL's +11.0pp gain under comparable frozen-backbone conditions (noting backbone differences controlled for in our analysis). On HLE, entity graph retrieval reaches 48.0% from 25.2% (+22.8pp). Ablations show component value is task-dependent: rich judge feedback is negligible for code generation but critical for structured queries (+10.3pp), while binary-signal iteration partially compensates for weaker feedback.

### 1. 核心问题与整体含义
*   **研究动机：** 现有的 LLM 智能体在程序执行层面是“无状态”的。即使面对结构相似的任务，智能体也会从零开始推导方案，无法像人类或强化学习（RL）智能体那样利用过往的成功经验或规避失败教训。
*   **核心问题：** 如何在不修改模型权重（非参数化）的前提下，让智能体能够积累、检索并重用结构化的程序化计划，从而实现“在线学习”效果。
*   **整体含义：** 论文提出了 **APEX-EM** 框架，通过构建一个“程序化-情境式”经验回放缓冲区，将任务执行的完整轨迹（包括规划、错误分析、修复补丁）转化为可重用的知识，显著提升了智能体在复杂任务中的成功率和效率。

### 2. 方法论
*   **核心思想：** 将经验视为一种结构化的图数据（Procedural Knowledge Graph, PKG），结合语义相似度和“结构特征（Structural Signature）”进行检索，并利用正负样本共同指导生成。
*   **关键技术细节：**
    *   **程序化知识图谱 (PKG)：** 包含实体（Entity）、经验（Experience）、子任务、抽象操作（Operation）等节点。它不仅记录“是什么”，还记录“怎么做”。
    *   **五层经验表示：** 每一条经验包含：目标反射（Goal）、程序模板（Procedure）、证据记录（Evidence）、错误登记册（Error Registry）和修复补丁（Patch）。
    *   **PRGII 工作流：**
        1.  **Plan (规划)：** 分解任务，提取抽象操作序列（如：实体解析 -> 时间过滤 -> 聚合）。
        2.  **Retrieve (检索)：** 混合检索机制。除了传统的语义向量搜索，还引入了**结构特征匹配**（计算操作序列的最长公共子序列 LCS）和图遍历。
        3.  **Generate (生成)：** 将检索到的成功经验作为正向示例，失败经验（带错误标注）作为负向约束。
        4.  **Iterate (迭代)：** 运行验证器，根据反馈进行自我修正。
        5.  **Ingest (摄取)：** 经过质量门控（Quality Gate）评估后，将新经验存入 PKG。

### 3. 实验设计
*   **数据集 / 场景：**
    *   **BigCodeBench (BCB)：** 复杂的 Python 代码生成，涉及多库调用。
    *   **KGQAGen-10k：** 基于 Wikidata 的多步结构化查询（SPARQL）生成。
    *   **Humanity’s Last Exam (HLE)：** 专家级跨学科知识推理。
*   **对比方法 (Baselines)：**
    *   **无记忆基准：** 直接使用 LLM 生成。
    *   **主流记忆框架：** RAG, Self-RAG, Reflexion, Mem0, MemP。
    *   **最强竞争对手：** **MemRL**（目前最先进的基于 Q-value 排序的冻结权重记忆系统）。
*   **评估指标：** 最终成功率 (SR) 和累积成功率 (CSR)。

### 4. 资源与算力
*   **模型使用：** 采用了 Claude 4.5 Sonnet 和 Claude 4.5 Opus 作为智能体骨干（Backbone）和评判器（Judge）。

### 5. 实验数量与充分性
*   **实验规模：** 涵盖了三个不同领域的数千个任务（BCB 1140个，KGQA 2000个采样，HLE 2500个）。
*   **消融实验：** 设计了 A0-A5 等多组消融实验，分别验证了记忆组件、评判器质量、迭代机制、检索策略（纯语义 vs 纯图 vs 混合）的贡献。
*   **公平性：** 在与 MemRL 对比时，尽可能保持了相同的任务划分和评估环境。实验设计较为客观，通过跨任务迁移实验（Transfer Learning）验证了系统不是简单的“死记硬背”。

### 6. 主要结论与发现
*   **性能飞跃：** 在 KGQA 任务上，准确率从 41.3% 提升至 89.6%（+48.3pp），甚至超过了提供真值子图的 Oracle 检索上限。
*   **结构匹配的威力：** 结构特征匹配允许智能体在词汇完全不同的任务间迁移逻辑（例如：从“分析球员数据”迁移到“分析财务报表”）。
*   **反馈的差异化价值：** 发现代码生成任务对详细反馈不敏感（二值对错即可），但结构化查询任务极度依赖评判器提供的定性分析。
*   **效率提升：** 随着记忆的积累，智能体解决任务所需的平均迭代次数显著下降（KGQA 任务减少了约 40% 的推理调用）。

### 7. 优点
*   **双重结果学习：** 创新性地利用失败经验作为“负面约束”，通过错误登记册防止智能体重蹈覆辙。
*   **跨域迁移能力：** 摆脱了对表面语义相似性的依赖，通过抽象操作序列实现了深层逻辑的重用。
*   **高透明度与可维护性：** PKG 的图结构使得经验可审计、可手动修正，且支持知识的自动更新（如 API 变更时的实体替换）。

### 8. 不足与局限
*   **依赖强评判器：** 系统性能高度依赖于作为“老师”的高级模型（如 Claude Opus）的反馈质量。
*   **上下文压力：** 随着检索到的结构化经验增加，Prompt 的长度会迅速膨胀，可能触及 LLM 的上下文窗口限制或增加成本。
*   **人工分类法：** 目前的操作序列（Operation）分类法仍带有一定的人工设计痕迹，尚未完全实现自动化的特征发现。
*   **收敛性证明：** 作为一个非参数化系统，虽然实验表现优异，但缺乏像强化学习那样的严谨收敛性理论证明。
