## 摘要
住房选择是一个高风险且在很大程度上不可逆的决策问题。我们将住房咨询作为住房选择的决策支持界面进行研究。现有的住房平台和许多基于大语言模型（LLM）的助手通常将这一过程简化为排序或推荐，导致推理过程不透明、多约束处理脆弱，且对事实性的保证有限。我们提出了 HabitatAgent，这是首个用于端到端住房咨询的 LLM 驱动多智能体架构。HabitatAgent 包含四个专门的智能体角色：记忆（Memory）、检索（Retrieval）、生成（Generation）和验证（Validation）。记忆智能体通过约束提取、记忆融合和验证门控更新等内部阶段来维护多层用户记忆；检索智能体执行混合向量-图检索（GraphRAG）；生成智能体产生引用证据的推荐和解释；验证智能体应用多层验证和针对性修复。这些智能体共同为端到端住房咨询提供了一个可审计且可靠的工作流。我们在端到端正确性协议下，针对 100 个真实用户咨询场景（300 个多轮问答对）对 HabitatAgent 进行了评估。一个强大的单阶段基准模型（Dense+Rerank）达到了 75% 的准确率，而 HabitatAgent 达到了 95%。

## Abstract
Housing selection is a high-stakes and largely irreversible decision problem. We study housing consultation as a decision-support interface for housing selection. Existing housing platforms and many LLM-based assistants often reduce this process to ranking or recommendation, resulting in opaque reasoning, brittle multi-constraint handling, and limited guarantees on factuality.   We present HabitatAgent, the first LLM-powered multi-agent architecture for end-to-end housing consultation. HabitatAgent comprises four specialized agent roles: Memory, Retrieval, Generation, and Validation. The Memory Agent maintains multi-layer user memory through internal stages for constraint extraction, memory fusion, and verification-gated updates; the Retrieval Agent performs hybrid vector--graph retrieval (GraphRAG); the Generation Agent produces evidence-referenced recommendations and explanations; and the Validation Agent applies multi-tier verification and targeted remediation. Together, these agents provide an auditable and reliable workflow for end-to-end housing consultation.   We evaluate HabitatAgent on 100 real user consultation scenarios (300 multi-turn question--answer pairs) under an end-to-end correctness protocol. A strong single-stage baseline (Dense+Rerank) achieves 75% accuracy, while HabitatAgent reaches 95%.

住房选择是一项高风险、高成本且具有强路径依赖的决策。传统的住房平台通常将其简化为简单的“搜索与排序”，而现有的 LLM 助手在处理此类复杂咨询时面临四大挑战：
*   **用户偏好演变：** 用户需求往往模糊且在多轮对话中不断变化。
*   **异构证据与关系约束：** 需整合楼盘、交通、学区、政策等复杂关系，单纯的语义搜索难以满足硬性约束。

### 2. 核心方法论
HabitatAgent 采用了由四个专业化智能体构成的闭环工作流：
*   **记忆智能体 (Memory Agent)：** 维护四层记忆结构（对话、实体、偏好、检索历史）。核心技术是**验证门控更新机制**：只有通过验证的响应信息才会被更新到长期记忆中，防止错误信息污染用户画像。
*   **检索智能体 (Retrieval Agent)：** 采用**自适应混合向量-图检索 (GraphRAG)**。通过一个路由模型预测查询复杂度，简单查询走向量检索，复杂查询（涉及空间、关系约束）则调用包含 6,016 个节点和 45,000 条边的知识图谱进行 Cypher 查询。
*   **生成智能体 (Generation Agent)：** 任务感知型生成。预设 14 种任务模板（如学区咨询、投资分析等），并在生成内容中强制要求**证据引用标签 (evidence_ref)**，确保每个数字和结论都有据可查。
*   **验证智能体 (Validation Agent)：** 执行三层验证（事实一致性、实体准确性、合规性）。若验证失败，执行**故障类型感知修复**：根据错误类型（实体缺失、约束冲突、事实错误）采取针对性补救措施（如放宽阈值或局部修正），而非盲目重新生成。

### 6. 主要结论与发现
*   **性能飞跃：** HabitatAgent 的端到端准确率达到 **95%**，远高于强基准模型 Dense+Rerank 的 75%。
*   **复杂约束处理：** 在处理包含 3 个以上约束的复杂查询时，传统检索的约束满足率（CSR）仅为 8%，而 HabitatAgent 保持在 95%。
*   **自适应路由的价值：** 移除自适应路由会导致准确率下降 20%，证明了在房产领域结合图谱检索的必要性。
*   **可靠性：** 验证门控和修复机制有效防止了多轮对话中的“偏好漂移”和事实幻觉。

### 7. 优点与亮点
*   **闭环可靠性：** 引入了“验证-修复-门控更新”的闭环，解决了 LLM 在高风险决策中容易产生幻觉的痛点。
*   **GraphRAG 应用：** 将知识图谱与向量检索有机结合，并利用路由机制平衡了检索精度与系统延迟。
*   **落地性强：** 该系统已在“房东东 (Fangdongdong)”产品中实际应用，具有很高的工程参考价值。
*   **证据溯源：** 强制性的证据引用标签显著提升了推荐结果的可解释性和透明度。

### 8. 不足与局限
*   **地理局限性：** 实验仅基于北京的数据，不同城市的房产政策、学区逻辑可能存在差异，系统的泛化性有待验证。
*   **延迟成本：** 虽然比固定 GraphRAG 快，但 P95 延迟仍达到 720ms，显著高于简单 RAG 系统，在极高并发场景下可能面临压力。
*   **数据依赖：** 系统高度依赖高质量的房产知识图谱，而构建和维护这样一个动态更新的图谱成本较高。
*   **私有数据偏差：** 使用私有数据集意味着外部研究者难以完全复现其推荐质量的绝对值。
