## 摘要
个人人工智能系统越来越多地保留用户活动的长期记忆，包括文档、电子邮件、消息、会议和环境录音。可信硬件可以保护这些数据的隐私，但难以随着数据存储规模的增长而扩展。这迫使数据被转移到外部存储，从而暴露了检索访问模式，导致隐私信息泄露给应用提供商。不经意随机存取存储器（ORAM）是一种可以隐藏这些模式的密码学原语，但它需要固定的访问预算，这排除了智能体记忆系统为保证准确性所依赖的查询相关遍历。我们提出了 Opal，一个用于个人人工智能的隐私内存系统。我们的核心见解是将所有与数据相关的推理与海量个人数据解耦，并将其限制在可信飞地（enclave）内。如此一来，不可信磁盘只能看到固定的、不经意的内存访问。这一驻留在飞地内的组件使用轻量级知识图谱来捕捉仅靠语义搜索会遗漏的个人上下文，并通过在每次 ORAM 访问中搭载重新索引和容量管理来处理持续的数据摄取。在由随机通信模型驱动的综合合成个人数据流水线上进行的评估显示，Opal 的检索准确率比语义搜索提高了 13 个百分点，且与安全基准相比，吞吐量提高了 29 倍，基础设施成本降低了 15 倍。目前，一家主要的人工智能提供商正考虑将 Opal 部署给数百万用户。

## Abstract
Personal AI systems increasingly retain long-term memory of user activity, including documents, emails, messages, meetings, and ambient recordings. Trusted hardware can keep this data private, but struggles to scale with a growing datastore. This pushes the data to external storage, which exposes retrieval access patterns that leak private information to the application provider. Oblivious RAM (ORAM) is a cryptographic primitive that can hide these patterns, but it requires a fixed access budget, precluding the query-dependent traversals that agentic memory systems rely on for accuracy.   We present Opal, a private memory system for personal AI. Our key insight is to decouple all data-dependent reasoning from the bulk of personal data, confining it to the trusted enclave. Untrusted disk then sees only fixed, oblivious memory accesses. This enclave-resident component uses a lightweight knowledge graph to capture personal context that semantic search alone misses and handles continuous ingestion by piggybacking reindexing and capacity management on every ORAM access. Evaluated on a comprehensive synthetic personal-data pipeline driven by stochastic communication models, Opal improves retrieval accuracy by 13 percentage points over semantic search and achieves 29x higher throughput with 15x lower infrastructure cost than a secure baseline. Opal is under consideration for deployment to millions of users at a major AI provider.

### 1. 核心问题与整体含义（研究动机和背景）
随着个人 AI（如 Microsoft Recall, Google Gemini）的普及，系统需要保留用户的长期记忆（邮件、会议录音、文档等）。然而，这带来了严峻的隐私挑战：
*   **可信硬件（TEE）的局限性：** 飞地（Enclave）内存有限且昂贵，无法容纳海量的个人数据，必须将数据存放在不可信的外部云存储中。
*   **访问模式泄露：** 即使数据是加密的，观察用户检索哪些记录（访问模式）也能推断出查询内容和用户行为。
*   **ORAM 的权衡：** 传统的不经意随机存取存储器（ORAM）虽然能隐藏访问模式，但要求固定的访问预算，这与现代 AI 智能体依赖的“数据依赖型遍历”（如知识图谱多步查询）相冲突。

**Opal 的目标：** 设计一个既能提供强隐私保护（隐藏内容和访问模式），又能支持复杂查询逻辑，且具备高吞吐量和低成本的个人 AI 长期记忆系统。

### 2. 方法论：核心思想与关键技术
Opal 的核心见解是**将所有“数据依赖型推理”与“海量数据存储”解耦**。

*   **系统架构：** 系统由客户端、三个相互协作的 TEE 飞地（控制器、嵌入模型、大语言模型）以及不可信的磁盘存储组成。磁盘上运行两个基于 Ring ORAM 的数据库（分别存储向量索引和原始数据块）。
*   **关键技术一：KG-filtered Search（知识图谱过滤搜索）：**
    *   在 Enclave 内部维护一个**轻量级、仅包含元数据**的知识图谱（KG），记录人名、时间、来源等关系，不存储具体内容。
    *   查询时，LLM 先在 Enclave 内提取谓词并遍历 KG 缩小候选范围，生成“准入集”。
    *   随后在准入集内进行向量评分，最后仅发起一次固定预算的 ORAM 取回。这保证了外部观察者只能看到固定模式的访问。
*   **关键技术二：Oblivious Dreaming（不经意梦境）：**
    *   为了处理持续的数据摄取和索引维护（如删除过期数据、修复聚类漂移），Opal 提出了“梦境”机制。
    *   它利用每次正常查询/摄取时带入 Enclave 内部的 ORAM 块（包括随机块和无关块），在不增加额外磁盘访问的前提下，顺带完成后台维护任务。
*   **关键技术三：合成个人数据流水线：**
    *   利用 **Hawkes 过程（自激点过程）** 模拟真实的通信爆发模式（如会议引发邮件，邮件引发文档修改），生成具有时空关联性的高质量合成数据集。
