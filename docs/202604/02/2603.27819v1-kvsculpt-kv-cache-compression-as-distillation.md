## 摘要
KV 缓存压缩对于高效的长上下文大语言模型（LLM）推理至关重要。减少每个键值对占用空间的方法（如量化和低秩分解）与减少缓存序列长度的方法是正交的。在序列长度维度上，现有方法涵盖了从纯逐出（选择保留哪些 KV 对）到合并（将相似的 KV 对组合成更少的对）的范围。这两者都受限于原始缓存条目。我们提出了 KVSculpt，它走向了这一频谱的另一端：我们不再选择或组合原始对，而是在连续嵌入空间中优化一组更小的、不受约束的 KV 对，以保留每一层的注意力行为。键（Keys）通过 L-BFGS 进行优化，值（Values）则通过最小二乘法以闭式解求解，每隔几步交替进行。在此基础上，我们引入了自适应预算分配，利用低成本的试点压缩运行，根据每个组件的难度在各层和 KV 头之间重新分配压缩预算。在具有 2048 个 token 上下文的 Qwen2.5-1.5B-Instruct 上，在压缩率 r ∈ {0.3, 0.5, 0.7} 的情况下，与 Select+Fit（基于注意力分数的逐出结合最小二乘值拟合）相比，KVSculpt 将 KL 散度降低了 3.5-4.1 倍。自适应分配在不增加额外推理成本的情况下，使 KL 散度进一步降低了 1.3 倍。分析表明，压缩难度高度不均匀：各层之间的试点均方误差（MSE）差异可达 100 倍，而单层内的两个 KV 头之间的差异可达 467 倍，这证明了细粒度预算分配的必要性。

## Abstract
KV cache compression is critical for efficient long-context LLM inference. Approaches that reduce the per-pair footprint -- quantization and low-rank decomposition -- are orthogonal to those that reduce the sequence length of the cache. Along the sequence-length dimension, existing methods range from pure eviction -- selecting which KV pairs to keep -- to merging, which combines similar pairs into fewer ones. Both remain anchored to the original cache entries. We propose KVSculpt, which moves to the other end of this spectrum: instead of selecting or combining original pairs, we optimize a smaller set of unconstrained KV pairs in continuous embedding space to preserve each layer's attention behavior. Keys are optimized via L-BFGS and values are solved in closed form via least squares, alternating every few steps. On top of this, we introduce adaptive budget allocation, which uses a cheap pilot compression run to redistribute the compression budget across layers and KV heads based on per-component difficulty. On Qwen2.5-1.5B-Instruct with 2048-token contexts, KVSculpt reduces KL divergence by 3.5-4.1x compared to Select+Fit -- attention-score eviction with least-squares value fitting -- across compression ratios r in {0.3, 0.5, 0.7}. Adaptive allocation provides an additional 1.3x KL reduction at no extra inference cost. Analysis reveals that compression difficulty is highly non-uniform: per-layer pilot MSE varies by up to 100x across layers, and the two KV heads within a single layer can differ by up to 467x -- demonstrating that fine-grained budget allocation is essential.

### KVSculpt: KV Cache 压缩即蒸馏 —— 论文深度总结

#### 1. 核心问题与整体含义（研究动机和背景）
在长上下文大语言模型（LLM）推理中，KV Cache（键值缓存）随着序列长度增加而急剧膨胀，成为显存瓶颈。现有的减少序列长度的方法主要分为**逐出（Eviction）**（如 H2O，选择保留子集）和**合并（Merging）**（如 CaM，将相似项加权平均）。
*   **核心痛点**：现有方法都受限于“原始缓存条目”的离散选择或组合。作者认为，由于位置信息已通过 RoPE 编码进入嵌入空间，压缩后的 KV 对不一定非要是原始 token 的子集。
*   **研究目标**：打破离散约束，在连续向量空间中直接优化出一组“虚拟”KV 对，使其能最大程度还原原始的注意力行为。

#### 2. 方法论：核心思想与关键技术
KVSculpt 将 KV 压缩重新定义为一个**蒸馏问题**，将原始 N 个 KV 对蒸馏为 k 个不受约束的连续向量。
*   **核心思想**：在连续嵌入空间 $R^d$ 中寻找 $k$ 个 KV 对，使压缩后的注意力输出与原始输出之间的差异最小化。
*   **损失函数**：包含两部分：
    1.  **输出 MSE**：压缩前后注意力输出向量的均方误差。
    2.  **LSE 匹配**：匹配 Log-Sum-Exp 分数，确保上下文块在未来解码中分配到的注意力权重比例正确。
*   **交替优化算法**：
    1.  **Key 优化**：使用 **L-BFGS** 算法（拟牛顿法）更新 Key。实验发现 L-BFGS 处理非凸的 Softmax 景观比 Adam 有效得多。
    2.  **Value 求解**：在 Key 固定时，利用**岭回归（最小二乘法）**以闭式解形式直接计算最优的 Value。
*   **自适应预算分配**：引入“试点运行（Pilot Run）”机制。先进行短时间的统一压缩测试，根据各层和各头（Head）的 MSE 表现（代表压缩难度），动态分配 $k$ 的数量。难度大的层/头分配更多配额。

#### 3. 实验设计
*   **模型与数据**：使用 **Qwen2.5-1.5B-Instruct**。测试数据来自 **PG19** 测试集，上下文长度 $N=2048$，评估后续 128 个 token 的生成质量。
*   **压缩率**：测试了 $r \in \{0.1, 0.3, 0.5, 0.7\}$。
*   **对比基准（Baselines）**：
    1.  **Random**：随机保留。
    2.  **Attention Score**：基于累积注意力分数逐出（类似 H2O）。
    3.  **Select+Fit**：先按分数选择位置，再用最小二乘法拟合 Value（目前最强的逐出变体）。
    4.  **Joint Opt**：使用硬混凝土松弛（Hard Concrete）学习离散掩码，是 KVSculpt 的离散对应版本。

#### 4. 资源与算力
*   **硬件**：实验主要在 **NVIDIA A100 GPU** 上完成。
*   **耗时**：对于 2048 token 的上下文，在单张 A100 上完成 100 步 L-BFGS 优化大约需要 **170 秒**。
*   **适用场景**：由于压缩过程涉及优化，该方法适用于**离线/摊销场景**（如系统提示词、长期存储的文档），而非单次在线推理。

#### 5. 实验数量与充分性
*   **实验规模**：基准测试涵盖了 20 组序列，详细分析和消融实验基于 5 组代表性序列。
*   **充分性**：
    *   进行了优化器对比（L-BFGS vs Adam）、查询策略对比（合成未来 Query 的不同方式）、分配策略对比（静态 vs 动态）。
    *   **客观性**：通过 KL 散度这一客观指标衡量分布差异，并提供了 Oracle（多次随机初始化取最优）作为性能上限参考。
    *   **局限性**：实验主要集中在单一模型（Qwen2.5-1.5B）和固定长度（2048），对于更大规模模型或超长上下文（如 100k+）的泛化性有待验证。

#### 6. 主要结论与发现
*   **性能飞跃**：在 $r=0.3$ 时，KVSculpt 的 KL 散度比最强基准 Select+Fit 降低了 **4.1 倍**。
*   **连续空间的威力**：仅仅优化 Value（Joint Opt）提升有限，允许 Key 在连续空间自由移动是性能提升的关键。
*   **非均匀难度**：模型内部压缩难度极度不均。层间 MSE 差异可达 100 倍，同层内不同头之间差异甚至高达 467 倍。
*   **瓶颈所在**：单层优化已接近理论极限（达到 Oracle 的 92%-98%），剩余误差主要源于**跨层误差累积**。

#### 7. 优点与亮点
*   **范式创新**：将 KV 压缩从“选择题”变为“优化题”，彻底释放了压缩的自由度。
*   **优化策略高效**：巧妙结合了 L-BFGS 处理非凸 Key 空间和最小二乘法处理线性 Value 空间的优势。
*   **细粒度分配**：自适应预算分配机制在不增加推理成本的前提下，显著提升了资源利用率。

#### 8. 不足与局限
*   **计算开销**：压缩过程需要迭代优化，时间成本远高于简单的逐出算法，不适合实时性要求极高的在线场景。
*   **模型覆盖**：缺乏在 Llama、Mistral 等其他主流架构上的验证。
*   **误差传播**：目前的逐层优化忽略了层与层之间的级联效应，导致在某些“敏感”序列上误差放大严重。
*   **下游任务评估**：主要评估了 KL 散度，缺乏在 MMLU、摘要等具体下游任务上的准确率测试。
