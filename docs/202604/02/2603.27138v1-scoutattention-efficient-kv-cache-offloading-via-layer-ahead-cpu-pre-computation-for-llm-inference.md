## 摘要
大语言模型在长上下文推理过程中面临严重的 GPU 显存容量限制，其中 KV 缓存的内存消耗严重限制了解码批处理大小。虽然现有研究已经探索了将 KV 缓存卸载到 DRAM，但这些方法要么需要频繁的 GPU-CPU 数据传输，要么对 CPU 计算提出了极高要求，导致系统在等待 I/O 操作或 CPU 处理完成时 GPU 利用率低下。我们提出了 ScoutAttention，这是一种新型的 KV 缓存卸载框架，通过 GPU-CPU 协同注意力计算来加速大语言模型推理。为了防止 CPU 计算成为系统瓶颈，ScoutAttention 引入了 GPU-CPU 协同的分块稀疏注意力，显著降低了 CPU 负载。与传统的并行计算方法不同，我们的框架采用了一种新颖的跨层提前 CPU 预计算算法，使 CPU 能够提前一层启动注意力计算，并辅以异步周期性召回机制，以保持极低的 CPU 计算负载。实验结果表明，ScoutAttention 在保持准确率与基准线差距在 2.4% 以内的同时，与现有的卸载方法相比实现了 2.1 倍的加速。

## Abstract
Large language models encounter critical GPU memory capacity constraints during long-context inference, where KV cache memory consumption severely limits decode batch sizes. While existing research has explored offloading KV cache to DRAM, these approaches either demand frequent GPU-CPU data transfers or impose extensive CPU computation requirements, resulting in poor GPU utilization as the system waits for I/O operations or CPU processing to complete.   We propose ScoutAttention, a novel KV cache offloading framework that accelerates LLM inference through collaborative GPU-CPU attention computation. To prevent CPU computation from bottlenecking the system, ScoutAttention introduces GPU-CPU collaborative block-wise sparse attention that significantly reduces CPU load. Unlike conventional parallel computing approaches, our framework features a novel layer-ahead CPU pre-computation algorithm, enabling the CPU to initiate attention computation one layer in advance, complemented by asynchronous periodic recall mechanisms to maintain minimal CPU compute load. Experimental results demonstrate that ScoutAttention maintains accuracy within 2.4% of baseline while achieving 2.1x speedup compared to existing offloading methods.

*   **核心问题**：在大语言模型（LLM）的长文本推理中，**KV Cache（键值缓存）** 消耗了巨大的 GPU 显存，限制了批处理大小（Batch Size）和吞吐量。
*   **研究背景**：现有的 KV Cache 卸载（Offloading）到内存（DRAM）的技术存在两大瓶颈：
    *   **I/O 瓶颈**：如 InfiniGen 等方法在需要时从 CPU 召回数据，受限于 PCIe 带宽，导致 GPU 长期处于等待数据的空闲状态（实验显示空闲率达 61%）。

*   **整体含义**：ScoutAttention 旨在通过 **GPU-CPU 协作计算** 和 **跨层预计算策略**，消除卸载过程中的 I/O 和计算延迟，实现高效的长文本推理。

### 2. 方法论：核心思想与关键技术
ScoutAttention 提出了三个核心设计：
*   **GPU-CPU 协作块稀疏注意力**：
    *   利用注意力的稀疏性，将 KV Cache 分块并生成紧凑的“摘要（Digest）”。
    *   GPU 保留摘要和最关键的块，其余卸载至 CPU。计算时，GPU 识别 Top-K 重要块，若块在 GPU 则由 GPU 计算，若在 CPU 则由 CPU 就地计算，最后在 GPU 端合并结果。
*   **跨层提前 CPU 预计算（Layer-Ahead Pre-computation）**：
    *   **核心创新**：在 GPU 计算第 $i$ 层时，利用第 $i$ 层的输入和第 $i+1$ 层的权重提前预测第 $i+1$ 层的 Query。
    *   **流程**：基于预测的 Query 提前触发 CPU 开始计算第 $i+1$ 层中位于 DRAM 的 KV 块。这使得 CPU 的计算时间窗从单纯的注意力算子扩展到了整个 Transformer 层的执行时间（包括 FFN 等），有效掩盖了 CPU 的计算延迟。
*   **异步周期性 KV 缓存召回**：
    *   由于推理过程中重要块会发生偏移（Drift），系统会根据预设阈值（如 CPU 计算占比超过 12%）异步地将重要块从 CPU 重新加载到 GPU，确保 CPU 负载始终保持在低位。
