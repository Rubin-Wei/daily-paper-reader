## 摘要
测试时扩展（test-time scaling）的兴起显著提升了大语言模型（LLMs）的推理能力和智能体水平。然而，标准 Transformer 难以高效地扩展推理时计算，因为传统的循环策略面临高昂的计算开销，且 KV 缓存会随模型深度增加而膨胀。我们提出了 Universal YOCO (YOCO-U)，它将 YOCO 的“解码器-解码器”（decoder-decoder）架构与递归计算相结合，实现了优于单一方法的协同效应。基于 YOCO 框架，YOCO-U 实现了一个通用自解码器（Universal Self-Decoder），通过参数共享进行多次迭代，同时将迭代过程限制在浅层的高效注意力层中。这种结合产生了一种优越的能力-效率权衡，这是 YOCO 或递归计算独立无法实现的。YOCO 架构提供了恒定的全局 KV 缓存和线性预填充，而部分递归则以有限的开销增强了表示深度。综上所述，YOCO-U 在保持高效推理的同时，提高了 Token 效用和扩展表现。实验结果证实，YOCO-U 在通用和长文本基准测试中保持了极强的竞争力，表明高效注意力架构与递归计算的融合是可扩展 LLM 的一个极具前景的方向。

## Abstract
The rise of test-time scaling has remarkably boosted the reasoning and agentic proficiency of Large Language Models (LLMs). Yet, standard Transformers struggle to scale inference-time compute efficiently, as conventional looping strategies suffer from high computational overhead and a KV cache that inflates alongside model depth. We present Universal YOCO (YOCO-U), which combines the YOCO decoder-decoder architecture with recursive computation to achieve a synergistic effect greater than either alone. Built on the YOCO framework, YOCO-U implements a Universal Self-Decoder that performs multiple iterations via parameter sharing, while confining the iterative process to shallow, efficient-attention layers. This combination yields a favorable capability-efficiency tradeoff that neither YOCO nor recursion achieves independently. The YOCO architecture provides a constant global KV cache and linear pre-filling, while partial recursion enhances representational depth with limited overhead. Together, YOCO-U improves token utility and scaling behavior while maintaining efficient inference. Empirical results confirm that YOCO-U remains highly competitive in general and long-context benchmarks, demonstrating that the integration of efficient-attention architectures and recursive computation is a promising direction for scalable LLMs.

### 1. 核心问题与整体含义（研究动机和背景）
随着“测试时扩展”（Test-time Scaling）技术的兴起，通过增加推理时的计算量来提升大语言模型（LLM）的推理能力已成为趋势。然而，传统的 Transformer 架构在扩展计算深度时面临两大瓶颈：
*   **计算开销巨大**：简单的循环（Looping）策略会导致计算量随深度线性增加。
*   **KV 缓存膨胀**：在标准 Transformer 中，KV 缓存的大小随模型深度线性增长，导致长文本推理时的内存压力剧增。
**研究动机**：提出一种既能通过递归增加表征深度，又能保持恒定 KV 缓存开销的高效架构。

### 2. 论文提出的方法论：YOCO-U
论文提出了 **Universal YOCO (YOCO-U)**，其核心思想是将 YOCO 的“解码器-解码器”架构与递归计算（Recursive Computation）相结合：
*   **架构组成**：
    *   **Universal Self-Decoder（通用自解码器）**：位于模型浅层，采用高效注意力机制（如滑动窗口注意力 SWA）。它通过参数共享进行 $T$ 次递归迭代，以极低的额外开销增强特征表示。
    *   **Cross-Decoder（交叉解码器）**：位于模型深层，利用自解码器生成的**单份全局 KV 缓存**进行交叉注意力计算。
*   **关键技术细节**：
    *   **递归逻辑**：仅在自解码器部分进行循环，而交叉解码器保持静态。
    *   **KV 缓存优化**：全局 KV 缓存仅生成一次，不随递归次数 $T$ 增加。只有局部的窗口 KV 缓存会随迭代微增，但相对于序列长度 $N$ 而言，这部分开销可忽略不计。
    *   **线性预填充**：继承了 YOCO 的特性，预填充阶段具有 $O(N)$ 的时间复杂度。

### 7. 优点（亮点）
*   **协同效应**：成功将“参数共享递归”与“单次 KV 缓存架构”结合，解决了递归模型内存爆炸的顽疾。
*   **推理友好**：在提升模型能力的同时，几乎没有增加推理延迟和显存负担，非常适合长文本和实时推理场景。
*   **扩展性强**：实验证明该架构在不同参数规模（从 300M 到 10B）下均表现稳定。

### 8. 不足与局限
*   **训练复杂度**：虽然推理高效，但递归结构在训练时的反向传播可能比标准模型更复杂（尽管文中提到训练过程很稳定）。
*   **高效注意力限制**：自解码器依赖滑动窗口等高效注意力，这在某些极度依赖精细局部信息的任务中可能存在理论上的表达上限。
*   **任务覆盖**：虽然在数学和通用任务上表现优异，但在多模态或极大规模（如 100B+）模型上的表现尚待验证。
