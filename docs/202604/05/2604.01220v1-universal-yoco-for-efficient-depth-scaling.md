## 摘要
推理时扩展（test-time scaling）的兴起显著提升了大语言模型（LLMs）的推理能力和智能体能力。然而，标准 Transformer 难以高效地扩展推理时计算，因为传统的循环策略面临高昂的计算开销，且 KV 缓存会随模型深度增加而膨胀。我们提出了 Universal YOCO (YOCO-U)，它将 YOCO 的“解码器-解码器”架构与递归计算相结合，实现了优于单一方法的协同效应。基于 YOCO 框架，YOCO-U 实现了一个通用自解码器（Universal Self-Decoder），通过参数共享进行多次迭代，同时将迭代过程限制在浅层的高效注意力层中。这种结合产生了一种良好的能力-效率权衡，这是 YOCO 或递归计算独立无法实现的。YOCO 架构提供了恒定的全局 KV 缓存和线性预填充，而部分递归则以有限的开销增强了表示深度。总之，YOCO-U 在保持高效推理的同时，提高了 Token 效用和扩展行为。实验结果证实，YOCO-U 在通用和长文本基准测试中保持了极强的竞争力，表明高效注意力架构与递归计算的结合是可扩展 LLM 的一个极具前景的方向。

## Abstract
The rise of test-time scaling has remarkably boosted the reasoning and agentic proficiency of Large Language Models (LLMs). Yet, standard Transformers struggle to scale inference-time compute efficiently, as conventional looping strategies suffer from high computational overhead and a KV cache that inflates alongside model depth. We present Universal YOCO (YOCO-U), which combines the YOCO decoder-decoder architecture with recursive computation to achieve a synergistic effect greater than either alone. Built on the YOCO framework, YOCO-U implements a Universal Self-Decoder that performs multiple iterations via parameter sharing, while confining the iterative process to shallow, efficient-attention layers. This combination yields a favorable capability-efficiency tradeoff that neither YOCO nor recursion achieves independently. The YOCO architecture provides a constant global KV cache and linear pre-filling, while partial recursion enhances representational depth with limited overhead. Together, YOCO-U improves token utility and scaling behavior while maintaining efficient inference. Empirical results confirm that YOCO-U remains highly competitive in general and long-context benchmarks, demonstrating that the integration of efficient-attention architectures and recursive computation is a promising direction for scalable LLMs.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）进入“推理时扩展”（Test-time Scaling）时代，通过增加推理计算量来提升逻辑推理能力成为主流。然而，传统的 Transformer 架构在扩展计算深度时面临两大瓶颈：
*   **计算开销巨大**：简单的循环（Looping）策略会导致计算量随深度线性增加。
*   **KV 缓存膨胀**：标准 Transformer 的 KV 缓存随模型深度增加而线性增长，导致长文本推理时的内存压力剧增。
论文旨在提出一种既能通过递归增加计算深度，又能保持恒定 KV 缓存开销的高效架构。

#### 2. 方法论：核心思想与关键技术
**YOCO-U (Universal YOCO)** 结合了 YOCO 的“解码器-解码器”架构与递归计算（Recursive Computation）：
*   **架构分层**：模型分为 **Self-Decoder（自解码器）** 和 **Cross-Decoder（交叉解码器）**。
*   **通用自解码器 (Universal Self-Decoder)**：这是 YOCO-U 的核心创新。它将浅层的自解码器模块设计为可递归的，通过参数共享进行 $T$ 次迭代计算。
*   **高效注意力机制**：在自解码器中使用滑动窗口注意力（SWA）或其他线性注意力机制。这使得递归过程中的 KV 缓存开销极低且恒定。
*   **一次性全局缓存**：自解码器迭代完成后，仅生成一次全局 KV 缓存供深层的交叉解码器使用。交叉解码器不再进行递归，从而避免了全局注意力的高额开销。
*   **协同效应**：这种设计将递归限制在计算廉价的浅层，既增强了表示深度，又保留了 YOCO 线性预填充和节省内存的优势。

#### 6. 主要结论与发现
*   **计算效率极高**：在相同 FLOPs 预算下，YOCO-U 的 Loss 显著低于非递归模型；达到相同性能仅需减少约 62% 的训练 Token。

#### 7. 优点与亮点
*   **架构优雅**：巧妙地解决了“深度扩展”与“内存开销”之间的矛盾，实现了“You Only Cache Once”的极致优化。
*   **推理友好**：线性预填充复杂度和恒定 KV 缓存使其成为处理超长文本（如 256K+）的理想架构。
*   **稳定性强**：实验显示递归训练过程非常平滑，没有出现传统递归模型常见的训练不稳定问题。

#### 8. 不足与局限
*   **递归深度收益递减**：表示分析显示，随着循环次数增加，层间表示的角距离逐渐减小，意味着单纯增加循环次数存在边际效用递减。
*   **高效注意力限制**：虽然支持多种线性注意力，但模型性能仍部分受限于浅层高效注意力层的表达能力。
*   **训练成本**：虽然推理高效，但训练时由于需要多次前向传播（递归），其训练时间成本仍高于同参数量的非递归模型（尽管 Token 效率更高）。
