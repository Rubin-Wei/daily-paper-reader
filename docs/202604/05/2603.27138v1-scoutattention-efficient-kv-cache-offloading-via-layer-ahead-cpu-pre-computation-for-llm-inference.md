## 摘要
大语言模型在长文本推理过程中面临严重的 GPU 显存容量限制，其中 KV 缓存的内存消耗严重限制了解码批处理大小。虽然现有研究已经探索了将 KV 缓存卸载到 DRAM，但这些方法要么需要频繁的 GPU-CPU 数据传输，要么对 CPU 计算提出了极高要求，导致系统在等待 I/O 操作或 CPU 处理完成时 GPU 利用率低下。我们提出了 ScoutAttention，这是一种新型的 KV 缓存卸载框架，通过 GPU-CPU 协同注意力计算来加速大语言模型推理。为了防止 CPU 计算成为系统瓶颈，ScoutAttention 引入了 GPU-CPU 协同的分块稀疏注意力，显著降低了 CPU 负载。与传统的并行计算方法不同，我们的框架采用了一种新颖的跨层提前 CPU 预计算算法，使 CPU 能够提前一层启动注意力计算，并辅以异步周期性召回机制，以保持极低的 CPU 计算负载。实验结果表明，ScoutAttention 在保持准确率与基准线差距在 2.4% 以内的同时，与现有的卸载方法相比实现了 2.1 倍的加速。

## Abstract
Large language models encounter critical GPU memory capacity constraints during long-context inference, where KV cache memory consumption severely limits decode batch sizes. While existing research has explored offloading KV cache to DRAM, these approaches either demand frequent GPU-CPU data transfers or impose extensive CPU computation requirements, resulting in poor GPU utilization as the system waits for I/O operations or CPU processing to complete.   We propose ScoutAttention, a novel KV cache offloading framework that accelerates LLM inference through collaborative GPU-CPU attention computation. To prevent CPU computation from bottlenecking the system, ScoutAttention introduces GPU-CPU collaborative block-wise sparse attention that significantly reduces CPU load. Unlike conventional parallel computing approaches, our framework features a novel layer-ahead CPU pre-computation algorithm, enabling the CPU to initiate attention computation one layer in advance, complemented by asynchronous periodic recall mechanisms to maintain minimal CPU compute load. Experimental results demonstrate that ScoutAttention maintains accuracy within 2.4% of baseline while achieving 2.1x speedup compared to existing offloading methods.

在长上下文（Long-context）大语言模型推理中，**KV Cache（键值缓存）** 的激增成为了 GPU 显存容量的主要瓶颈，严重限制了推理时的批处理大小（Batch Size）和吞吐量。
*   **现有方案的缺陷**：
    *   **基于召回的方法（如 InfiniGen）**：将 KV Cache 卸载到 CPU 内存（DRAM），需要时再通过 PCIe 传回 GPU。受限于 PCIe 带宽（远低于显存带宽），导致 GPU 频繁处于 I/O 等待状态（空闲率约 61%）。

*   **研究目标**：设计一种既能消除 I/O 瓶颈，又能掩盖 CPU 计算延迟的高效 KV Cache 卸载框架。

### 2. 方法论：核心思想与关键技术
ScoutAttention 提出了 **GPU-CPU 协同的块稀疏注意力机制**，核心思想是通过“空间换时间”和“异步流水线”来优化推理过程。

*   **层提前 CPU 预计算（Layer-Ahead CPU Pre-computation）**：
    *   **核心逻辑**：利用 Transformer 架构中残差连接的特性，观察到相邻层的输入高度相似。
    *   **算法流程**：在 GPU 计算当前第 $i$ 层时，利用第 $i$ 层的输入 $X_i$ 和第 $i+1$ 层的查询投影矩阵 $W^Q_{i+1}$ 提前预测下一层的查询向量 $Q^{pred}_{i+1}$。
    *   **并行化**：根据预测的 $Q^{pred}_{i+1}$ 识别出不在 GPU 显存中的重要 KV 块，并立即触发 CPU 异步计算这些块的注意力得分。当 GPU 运行到第 $i+1$ 层时，CPU 的计算结果通常已就绪，直接进行结果合并（Merge），从而掩盖了 CPU 的计算耗时。
*   **GPU-CPU 协同块稀疏（Collaborative Block-wise Sparsity）**：
    *   系统仅在 GPU 中保留 KV 块的紧凑摘要（Digests）和极少数核心块。
    *   通过计算 Query 与 Digest 的点积筛选 Top-K 重要块。GPU 处理驻留在显存中的块，CPU 处理驻留在 DRAM 中的块。
*   **异步周期性 KV 缓存召回（Async Periodic Recall）**：
    *   **重要性漂移修正**：随着解码进行，重要的 KV 块会发生变化。若 CPU 负担过重（超过阈值 $\beta$），系统会触发异步召回，将 DRAM 中的重要块传回 GPU。
    *   **非关键路径传输**：召回操作在层计算完成后触发，利用整个 Token 解码的时间窗口进行传输，不阻塞 GPU 计算。
