## 摘要
由于自注意力机制的二次复杂度，长上下文视频理解与生成对基于 Transformer 的视频模型构成了显著的计算挑战。虽然现有的稀疏注意力方法采用粗粒度模式来提高效率，但它们通常会导致冗余计算和次优性能。为了解决这一问题，本文提出了 VecAttention，这是一种新型的向量级稀疏注意力框架，为视频模型实现了卓越的准确率-效率权衡。我们观察到视频注意力图表现出强烈的垂直向量稀疏模式，并进一步证明，与现有的粗粒度稀疏模式相比，这种垂直向量模式提供了始终更好的准确率-稀疏度权衡。基于这一观察，VecAttention 通过最小化内存访问开销的轻量级重要向量选择以及优化的向量稀疏注意力算子，动态地仅选择并处理包含信息的垂直向量。在视频理解（VideoMME、LongVideoBench 和 VCRBench）和生成（VBench）任务上的全面评估表明，VecAttention 相比全注意力实现了 2.65 倍的加速，相比最先进的稀疏注意力方法实现了 1.83 倍的加速，且准确率与全注意力相当。我们的代码可在 https://github.com/anminliu/VecAttention 获取。

## Abstract
Long-context video understanding and generation pose a significant computational challenge for Transformer-based video models due to the quadratic complexity of self-attention. While existing sparse attention methods employ coarse-grained patterns to improve efficiency, they typically incur redundant computation and suboptimal performance. To address this issue, in this paper, we propose \textbf{VecAttention}, a novel framework of vector-wise sparse attention that achieves superior accuracy-efficiency trade-offs for video models. We observe that video attention maps exhibit a strong vertical-vector sparse pattern, and further demonstrate that this vertical-vector pattern offers consistently better accuracy-sparsity trade-offs compared with existing coarse-grained sparse patterns. Based on this observation, VecAttention dynamically selects and processes only informative vertical vectors through a lightweight important-vector selection that minimizes memory access overhead and an optimized kernel of vector sparse attention. Comprehensive evaluations on video understanding (VideoMME, LongVideoBench, and VCRBench) and generation (VBench) tasks show that VecAttention delivers a 2.65$\times$ speedup over full attention and a 1.83$\times$ speedup over state-of-the-art sparse attention methods, with comparable accuracy to full attention. Our code is available at https://github.com/anminliu/VecAttention.

随着长上下文视频理解（VLM）和视频生成（DiT）模型的快速发展，Transformer 架构中自注意力机制的**二次复杂度**成为了推理性能的瓶颈，导致长视频处理的延迟达到分钟甚至小时级别。
*   **研究动机**：现有的稀疏注意力方法（如行模式、块模式）粒度较粗，无法精准捕捉视频注意力图中细碎且分散的信息分布，导致在选中的区域内仍存在大量无效计算。
*   **核心发现**：论文观察到视频注意力图呈现出强烈的**垂直向量（Vertical-Vector）**稀疏模式，这种细粒度的模式比传统的行或块模式更接近理想的稀疏状态（Oracle），能实现更好的精度与稀疏度权衡。

#### 2. 方法论：核心思想与关键技术
VecAttention 提出了一个向量级稀疏注意力框架，主要包含两个阶段：

*   **重要向量选择（Important-Vector Selection）**：
    *   **查询池化（Query Pooling）**：通过对相邻查询（Query）进行均值池化，生成压缩的池化查询 $Q_p$，用以高效估计注意力图的列重要性。
    *   **免排序 minS 过滤器**：放弃了昂贵的 Top-K 或 Top-P 排序操作，改用基于行最大值的阈值过滤（minS），显著降低了计算开销。
    *   **分块融合选择（TilingSelect）**：将向量选择逻辑直接融合进分块（Tiled）GEMM 算子中。在 GPU 片上内存（SRAM）计算时即完成过滤，避免将完整的中间注意力得分矩阵写回显存（HBM），大幅减少了内存带宽压力。
*   **向量稀疏计算（Vector-Sparse Computation）**：
    *   基于第一阶段选出的索引，动态加载离散的 Key 和 Value 向量。
    *   优化 GPU Kernel，采用类似 FlashAttention 的分块策略，但仅针对选定的垂直向量进行计算和累加。

#### 6. 主要结论与发现
*   **性能提升**：VecAttention 相比全注意力实现了 **2.65×** 的算子加速，相比最先进的稀疏方法（如 XAttention）实现了 **1.83×** 的加速。
*   **端到端增益**：在视频理解任务中，首字延迟（TTFT）实现了约 **1.17×** 的端到端加速。
*   **精度保持**：在高达 **93%** 的稀疏度下，VecAttention 仍能保持与全注意力几乎一致的准确率。
*   **通用性**：垂直向量模式不仅存在于视频模型，在纯文本大模型（Llama3）和多模态模型（Qwen2.5-Omni）中也广泛存在。

#### 7. 优点（亮点）
*   **细粒度创新**：识别并利用了比“块”更细的“向量”级稀疏性，填补了粗粒度稀疏与理想稀疏之间的空白。
*   **系统-算法协同**：通过 `TilingSelect` 解决了细粒度选择带来的 HBM 读写瓶颈，使理论上的稀疏增益能转化为实际的硬件加速。
*   **免排序设计**：`minS` 过滤器巧妙避开了 GPU 不擅长的排序操作，极大地压缩了选择阶段的开销。

#### 8. 不足与局限
*   **下游任务覆盖**：虽然在理解和生成上表现优异，但在更复杂的动态场景（如 Agent 推理、长文档 RAG）中的泛化能力仍需进一步验证。
*   **模式单一性**：目前主要聚焦于垂直向量模式，虽然证明了其主导地位，但未深入探讨水平向量或对角线模式在特定层中的潜在补充作用。
*   **硬件依赖**：优化的 Kernel 高度依赖 NVIDIA GPU 架构（如 A100），在其他硬件平台（如昇腾、AMD）上的移植性和表现尚未讨论。
