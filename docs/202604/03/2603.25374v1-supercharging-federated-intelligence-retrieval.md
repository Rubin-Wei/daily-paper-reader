## 摘要
检索增强生成（RAG）通常假设可以集中访问文档，但当知识分布在私有数据孤岛中时，这种假设便不再成立。我们提出了一种使用 Flower 构建的安全联邦 RAG 系统，该系统执行本地孤岛检索，而服务端聚合和文本生成则在经认证的机密计算环境中运行。即使在存在“诚实但好奇”或被攻破的服务器的情况下，该系统也能实现机密的远程大语言模型（LLM）推理。我们还提出了一种级联推理方法，在不削弱机密性的前提下，将非机密第三方模型（如 Amazon Nova）作为辅助上下文引入。

## Abstract
RAG typically assumes centralized access to documents, which breaks down when knowledge is distributed across private data silos. We propose a secure Federated RAG system built using Flower that performs local silo retrieval, while server-side aggregation and text generation run inside an attested, confidential compute environment, enabling confidential remote LLM inference even in the presence of honest-but-curious or compromised servers. We also propose a cascading inference approach that incorporates a non-confidential third-party model (e.g., Amazon Nova) as auxiliary context without weakening confidentiality.

### 1. 核心问题与研究背景
传统的检索增强生成（RAG）依赖于将所有文档集中存储在中心化服务器上。然而，在医疗、金融等敏感领域，数据通常分布在不同的私有孤岛中，受限于法规和隐私保护，无法进行物理汇总。
**联邦 RAG (FedRAG)** 虽能解决数据本地化问题，但现有方案存在安全隐患：检索到的私密文档片段在服务器端以明文形式聚合，容易被“诚实但好奇”的服务器运营商或被攻破的基础设施窃取。本研究旨在构建一个端到端安全的联邦 RAG 系统，确保检索、聚合和生成全过程的机密性。

### 2. 方法论与核心技术
该系统基于 **Flower** 联邦学习框架构建，核心思想是将敏感操作移入**机密计算（Confidential Computing）**环境。
*   **本地检索**：每个数据孤岛（Client）维护本地的 FAISS 向量索引，根据查询检索 Top-k 个相关文档。
*   **机密聚合与生成**：服务器运行在受信任执行环境（TEE）中。它接收各孤岛返回的加密文档和评分，在 TEE 内部进行解密、重排序（使用 **Reciprocal Rank Fusion, RRF** 算法）并构建上下文。
*   **三种推理模式**：
    1.  **独立推理 (Standalone)**：在服务器 TEE 内的 CPU 上运行轻量级 LLM（如 SmolLM）。
    2.  **级联推理 (Cascading)**：在 TEE 内结合本地 LLM 和非机密第三方模型（如 AWS Nova）。第三方模型仅作为“概率性知识源”提供辅助背景，不接触原始私密文档，从而在不泄密的前提下提升性能。
    3.  **机密远程推理 (Confidential Inference)**：利用 **Flower CRC**，将加密的上下文发送至远程受保护的 GPU TEE（如 H100）运行超大规模模型（如 Qwen3）。

*   **数据集**：使用了医疗领域的 **MIRAGE 基准测试**，包含三个子集：
    *   **PubMedQA**（501个是/否/可能问题）
    *   **BioASQ**（619个是/否问题）
    *   **MedQA**（1272个多选题）
*   **数据分布**：模拟了四个联邦客户端，分别持有 PubMed (23.9M)、StatPearls (301k)、Textbooks (126k) 和 Wikipedia (29.9M) 的文档语料。
*   **对比方法**：对比了上述三种推理模式（Standalone vs. Cascaded vs. Confidential）。

*   **服务器端（TEE）**：采用 **CPU 绑定** 环境部署 SmolLM 1.7B，未在本地使用 GPU，强调了在受限算力的 TEE 中运行的可能性。

### 5. 实验数量与充分性
*   **实验规模**：在三个标准的医疗 QA 数据集上进行了端到端的性能和延迟测试。

### 6. 主要结论与发现
*   **性能提升**：级联推理模式在 PubMedQA 和 MedQA 上分别比独立推理提升了约 **40% 和 46%**，证明了引入外部非机密模型作为辅助的有效性。
*   **最优方案**：基于 Flower CRC 的机密推理（Qwen3 235B）获得了**最高的准确率和最低的延迟**，这得益于高性能 GPU 和大规模参数量。
*   **安全性验证**：证明了可以在不暴露原始文档给服务器或第三方供应商的情况下，实现跨孤岛的复杂知识检索与问答。

### 7. 优点与亮点
*   **端到端隐私**：不仅保护了存储，还通过 TEE 保护了聚合逻辑和生成过程，解决了 FedRAG 的“最后一公里”隐私问题。
*   **混合信任架构**：创新的“级联推理”允许在保护隐私的同时利用强大的公有云模型（如 AWS Nova），平衡了安全性与智能水平。
*   **工程落地性**：基于成熟的 Flower 框架，展示了在 CPU 绑定的 TEE 环境下部署 RAG 的可行性。

### 8. 不足与局限
*   **硬件依赖**：系统高度依赖支持 TEE（如 Intel SGX/TDX 或 NVIDIA H100 TEE）的硬件，这可能限制其在通用廉价服务器上的部署。
*   **延迟挑战**：在 CPU 绑定的 TEE 中运行 LLM 生成（Standalone 模式）延迟较高（约 43-48秒/查询），对于实时性要求极高的场景可能存在瓶颈。
*   **领域局限**：实验仅在医疗领域进行，对于其他领域（如法律、代码）的复杂检索效果尚待验证。
