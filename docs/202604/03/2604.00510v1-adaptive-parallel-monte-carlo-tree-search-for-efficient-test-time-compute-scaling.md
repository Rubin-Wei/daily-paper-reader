## 摘要
蒙特卡洛树搜索 (MCTS) 是一种有效的测试时计算缩放 (TTCS) 方法，用于提升大语言模型的推理性能，但在实践中，其高度多变的执行时间会导致严重的尾部延迟。现有的优化手段（如正向提前退出）在理想情况下可以降低延迟，但在搜索持续进行却无实质性进展时效果较差。我们引入了“负向提前退出”（negative early exit），用于剪枝无成效的 MCTS 轨迹，以及一种“自适应增强机制”（adaptive boosting mechanism），通过重新分配回收的计算资源来减少并发搜索之间的资源竞争。这些技术集成到 vLLM 中后，在保持推理准确性的同时，显著降低了 p99 端到端延迟并提高了吞吐量。

## Abstract
Monte Carlo Tree Search (MCTS) is an effective test-time compute scaling (TTCS) method for improving the reasoning performance of large language models, but its highly variable execution time leads to severe long-tail latency in practice. Existing optimizations such as positive early exit, reduce latency in favorable cases but are less effective when search continues without meaningful progress. We introduce {\it negative early exit}, which prunes unproductive MCTS trajectories, and an {\it adaptive boosting mechanism} that reallocates reclaimed computation to reduce resource contention among concurrent searches. Integrated into vLLM, these techniques substantially reduce p99 end-to-end latency while improving throughput and maintaining reasoning accuracy.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：虽然 MCTS 在测试时计算缩放（TTCS）中能显著提升 LLM 的推理准确率，但其**高度可变的执行时间**导致了严重的**长尾延迟（Tail Latency）**。
*   **研究背景**：
    *   MCTS 具有天然的串行依赖性，搜索深度增加会导致执行时间剧增。
    *   现有的“正向早停（Positive Early Exit）”仅在找到高质量答案时停止，但对于那些无法找到答案的“难题”，系统会耗尽所有搜索预算，造成资源浪费和排队延迟。
    *   在实际生产环境中，这种不确定的延迟使得 MCTS 难以大规模部署。

### 2. 论文提出的方法论
论文提出了一套自适应并行 MCTS 框架，核心包含以下三个关键技术：
*   **负向早停机制（Negative Early Exit, NE）**：
    *   **核心思想**：识别并提前终止那些不太可能产生正确答案的搜索轨迹。
    *   **技术细节**：利用过程奖励模型（PRM）的特性，即轨迹分数受限于叶子节点分数。如果当前树中所有叶子节点的分数都低于预设阈值（被定义为“徒劳”节点），则判定该请求无法通过后续搜索获益，立即终止。
    *   **优化**：引入“选择性徒劳检查”，利用第一步奖励与最终分数的相关性，排除低潜力分支。
*   **并行 MCTS 策略（WU-PUCT）**：
    *   基于 WU-UCT 算法，引入“未观测计数（Unobserved Counts）”来处理异步 Rollout。在选择节点时，将正在进行但尚未完成的搜索任务计入统计，防止多个并行 worker 涌向同一路径（避免探索崩溃）。
*   **自适应增强与调度（Adaptive Boosting & Scheduling）**：
    *   **动态并行度控制**：根据任务的“资历”（等待时间）和“紧迫性”（当前最高分接近阈值的程度）计算得分。
    *   **资源重分配**：将负向早停回收的计算资源（GPU 算力槽位）动态分配给那些最有希望快速完成的任务，通过提高其并行度（Boosting）来加速清理运行队列。
