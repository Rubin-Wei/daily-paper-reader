## 摘要
基于大语言模型（LLM）的购物智能体日益依赖长购买历史和多轮交互来实现个性化，然而，由于噪声、长度和相关性不匹配等问题，直接将原始历史记录附加到提示词中往往效果不佳。我们提出了 MemRerank，这是一个偏好记忆框架，它将用户购买历史提炼为简洁且与查询无关的信号，用于个性化商品重排序。为了研究这一问题，我们构建了一个端到端的基准测试和评估框架，其核心是一个基于 LLM 的“五选一”（1-in-5）选择任务，该任务同时衡量记忆质量和下游重排序效用。我们进一步利用强化学习（RL）训练记忆提取器，并使用下游重排序性能作为监督信号。针对两种基于 LLM 的重排序器的实验表明，MemRerank 始终优于无记忆、原始历史和现成的记忆基准，在“五选一”准确率上最高提升了 10.61 个百分点。这些结果表明，显式偏好记忆是智能体电商系统中实现个性化的一个实用且有效的构建模块。

## Abstract
LLM-based shopping agents increasingly rely on long purchase histories and multi-turn interactions for personalization, yet naively appending raw history to prompts is often ineffective due to noise, length, and relevance mismatch. We propose MemRerank, a preference memory framework that distills user purchase history into concise, query-independent signals for personalized product reranking. To study this problem, we build an end-to-end benchmark and evaluation framework centered on an LLM-based \textbf{1-in-5} selection task, which measures both memory quality and downstream reranking utility. We further train the memory extractor with reinforcement learning (RL), using downstream reranking performance as supervision. Experiments with two LLM-based rerankers show that MemRerank consistently outperforms no-memory, raw-history, and off-the-shelf memory baselines, yielding up to \textbf{+10.61} absolute points in 1-in-5 accuracy. These results suggest that explicit preference memory is a practical and effective building block for personalization in agentic e-commerce systems.

随着大语言模型（LLM）在电商智能体中的应用，如何利用长期的用户购买历史实现个性化推荐成为关键。传统方法直接将原始历史记录（Raw History）拼接在 Prompt 中，但这面临三大挑战：**噪声多**（历史中包含无关购买）、**长度长**（超出上下文窗口）以及**相关性不匹配**（历史行为与当前查询不直接相关）。论文旨在提出一种高效的“偏好记忆”机制，将冗长的历史提炼为简洁、结构化且与查询无关的信号，以提升个性化重排序的准确性。

#### 2. 方法论：核心思想与关键技术
MemRerank 框架包含两个主要阶段：**偏好记忆提取**和**下游重排序**。
*   **核心思想**：不直接喂入原始数据，而是先由一个 LLM 提取器将历史转化为“类内偏好”（Within-category）和“跨类偏好”（Cross-category）的结构化文本。
*   **关键技术细节**：
    *   **结构化提取**：提取器根据购买元数据（标题、描述）生成包含“维度（Aspect）”、“偏好（Preference）”和“证据（Evidence）”的记忆块。
    *   **强化学习训练（GRPO）**：使用组相对策略优化（GRPO）算法对提取器进行后训练。奖励函数（Reward Function）由两部分组成：**格式奖励**（确保输出符合特定的 XML 标签格式）和**重排序奖励**（如果下游重排序器根据该记忆选对了正确商品，则给予正向反馈）。
    *   **1-in-5 任务**：将重排序简化为从 5 个候选商品（1 个正样本，4 个同类别硬负样本）中选出最相关的一个，作为训练和评估的核心原子操作。

*   **Benchmark**：采用 1-in-5 选择任务的准确率（Accuracy）作为衡量标准。

#### 6. 主要结论与发现
*   **记忆优于原始数据**：显式提取的偏好记忆显著优于直接拼接原始历史，最高提升了 **10.61%** 的准确率。
*   **RL 训练至关重要**：经过 RL 训练的提取器生成的记忆比强力模型（如 GPT-5.2）生成的通用记忆更具“任务导向性”，对重排序更有利。
*   **Prompt 设计的影响**：半结构化且带有证据回溯（Evidence Grounding）的 Prompt 效果最好，因为它在灵活性和约束性之间取得了平衡。

#### 7. 优点
*   **任务对齐**：通过 RL 将记忆提取与下游任务性能直接挂钩，解决了“提取了但没用”的问题。
*   **高效性**：将长历史压缩为短文本，节省了推理时的 Token 开销和计算成本。
*   **可解释性**：生成的记忆包含证据片段，且配合 Think Tag 可以让用户或开发者理解系统为何做出特定推荐。

#### 8. 不足与局限
*   **类别覆盖有限**：主要实验集中在电子产品类别，不同类别（如服装 vs. 图书）的偏好特征差异巨大，需更多验证。
*   **规模限制**：目前仅验证了 1-in-5 的重排序环节，尚未在更大规模（如从 100 个候选者中排序）的端到端检索系统中测试。
*   **冷启动问题**：对于历史记录极少的用户，该框架的增益可能受限，文中未对此类边缘情况做深入探讨。
