## 摘要
基于大语言模型（LLM）的购物智能体日益依赖长期的购买历史和多轮交互来实现个性化，但由于噪声、长度和相关性不匹配，直接将原始历史记录附加到提示词中往往效果不佳。我们提出了 MemRerank，这是一个偏好记忆框架，它将用户的购买历史提炼为简洁且独立于查询的信号，用于个性化商品重排序。为了研究这一问题，我们构建了一个端到端的基准测试和评估框架，其核心是一个基于 LLM 的“五选一”（1-in-5）选择任务，该任务同时衡量记忆质量和下游重排序效用。我们进一步利用强化学习（RL）训练记忆提取器，并以下游重排序性能作为监督信号。对两个基于 LLM 的重排序器进行的实验表明，MemRerank 始终优于无记忆、原始历史记录和现成的记忆基准模型，在“五选一”准确率上最高提升了 10.61 个绝对百分点。这些结果表明，显式偏好记忆是代理式电子商务系统中实现个性化的实用且有效的构建模块。

## Abstract
LLM-based shopping agents increasingly rely on long purchase histories and multi-turn interactions for personalization, yet naively appending raw history to prompts is often ineffective due to noise, length, and relevance mismatch. We propose MemRerank, a preference memory framework that distills user purchase history into concise, query-independent signals for personalized product reranking. To study this problem, we build an end-to-end benchmark and evaluation framework centered on an LLM-based \textbf{1-in-5} selection task, which measures both memory quality and downstream reranking utility. We further train the memory extractor with reinforcement learning (RL), using downstream reranking performance as supervision. Experiments with two LLM-based rerankers show that MemRerank consistently outperforms no-memory, raw-history, and off-the-shelf memory baselines, yielding up to \textbf{+10.61} absolute points in 1-in-5 accuracy. These results suggest that explicit preference memory is a practical and effective building block for personalization in agentic e-commerce systems.

随着大语言模型（LLM）在电商领域作为“购物助手”的应用日益广泛，如何利用用户长期的购买历史实现**个性化推荐**成为关键。然而，直接将冗长的原始购买历史（Raw History）塞进 LLM 的 Prompt 中存在三大痛点：
*   **噪声大**：历史记录中包含大量与当前搜索无关的信息。

*   **相关性失配**：原始数据缺乏对用户底层偏好的提炼。
本文旨在通过构建一个**显式的偏好记忆（Preference Memory）模块**，将海量历史蒸馏为结构化的偏好信号，以提升商品重排序（Reranking）的准确性。

### 2. 方法论：MemRerank 框架
MemRerank 的核心思想是将个性化过程分为“记忆提取”和“下游重排序”两个阶段：
*   **偏好记忆提取**：使用一个 LLM（如 Qwen2.5-7B）作为提取器，将用户的购买历史转化为结构化的文本。记忆分为两部分：
    *   **类目内记忆（Within-category）**：用户在当前目标类目下的特定偏好（如：买电子产品时关注降噪、品牌）。
    *   **跨类目记忆（Cross-category）**：用户跨品类的通用消费习惯（如：追求性价比、极简风格）。
*   **强化学习训练（GRPO）**：这是本文的关键技术。提取器并非直接使用预训练模型，而是通过**强化学习（Group Relative Policy Optimization, GRPO）**进行后训练。
    *   **奖励函数（Reward）**：由“格式奖励”（确保输出符合结构化标签）和“重排序奖励”（如果提取的记忆能让下游重排序器选对正确商品，则给予正向反馈）组成。
*   **下游重排序**：将提取的记忆与当前查询（Query）和候选商品集一起输入重排序器（如 GPT-4o-mini），执行“5选1”任务。

### 6. 主要结论与发现
*   **显著提升**：MemRerank 在 o4-mini 上比无记忆基准提升了 **10.61%** 的准确率。
*   **RL 的必要性**：经过 RL 训练的提取器显著优于未经训练的基座模型，甚至优于更强大的闭源模型直接提取的效果。

### 7. 优点与亮点
*   **端到端优化**：通过 RL 将记忆提取与下游任务性能直接挂钩，解决了“提取的记忆看起来好但没用”的问题。
*   **结构化设计**：区分“类目内”和“跨类目”记忆，符合人类购物逻辑（既有特定需求，也有通用审美/预算习惯）。
*   **证据溯源**：在提取记忆时要求附带原始文本片段，增强了记忆的可解释性和可靠性。

### 8. 不足与局限
*   **类目覆盖有限**：实验主要集中在“电子产品”类目，虽然该类目数据丰富，但其结论在服装、生鲜等偏好更主观或多变的品类中是否同样有效尚待验证。

*   **实时性挑战**：虽然记忆是独立于查询提取的，但在实际生产环境中，如何高效地更新和检索这些文本记忆仍存在工程挑战。
