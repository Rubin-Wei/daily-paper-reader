## 摘要
长上下文生成日益受到解码阶段键值（KV）缓存传输流量的限制，特别是当 KV 缓存被卸载到 GPU 显存之外时。查询感知检索（如 Top-K 选择）通过仅加载 KV 对的子集来减少此类流量，但当注意力权重分散在未检索到的标记上时，对该子集进行 softmax 重新归一化会引入偏差。我们提出了一种检索-补全注意力模块，该模块保持骨干网络权重和 KV 缓存格式不变。对于每个查询，我们计算针对 sink/tail 锚点和查询相关的 Top-K 检索标记的精确注意力，并利用预填充阶段计算的固定大小特征图摘要来估计剩余中间区域的分子和分母。我们在未归一化域中将精确贡献和估计贡献相加，并执行统一归一化，从而在不增加额外 KV 读取的情况下恢复缺失的 softmax 权重。在长上下文基准测试中，在相同的等效标记读取预算下，所提方法优于仅采用选择的 Top-K 方法，且在高熵注意力头中提升最为显著。

## Abstract
Long-context generation is increasingly limited by decode-time key-value (KV) cache traffic, particularly when KV is offloaded beyond GPU memory. Query-aware retrieval (e.g., Top-K selection) reduces this traffic by loading only a subset of KV pairs, but renormalizing the softmax over the subset introduces bias when attention mass is spread over unretrieved tokens. We propose a retrieval-completion attention module that keeps backbone weights and the KV-cache format unchanged. For each query, we compute exact attention over sink/tail anchors and the query-dependent retrieved Top-K tokens, and estimate the remaining mid-region numerator and denominator using a fixed-size feature-map summary computed at prefill time. We add the exact and estimated contributions in the unnormalized domain and apply a single normalization, recovering the missing softmax mass without additional attention-side KV reads. Across long-context benchmarks, the proposed method improves over selection-only Top-K at matched token-equivalent read budgets, with the largest gains in high-entropy heads.

### 2. 方法论：检索-补全注意力机制（Retrieval-Completion Attention）
*   **核心思想**：将注意力计算分为“精确计算”和“线性估计”两部分。在非归一化域（Unnormalized Domain）将两者合并，最后进行统一归一化。
*   **关键技术细节**：
    *   **区域划分**：将预填充（Prefill）序列分为三部分：Sink（起始锚点）、Tail（末尾窗口）和 Mid（中间区域）。Sink 和 Tail 始终精确读取。
    *   **Top-K 检索**：在 Mid 区域根据查询（Query）检索最重要的 $K$ 个 Token 进行精确计算。
    *   **固定大小补全缓存（Completion Cache）**：在预填充阶段，利用线性注意力（Linear Attention）的思想，将 Mid 区域的所有 KV 对压缩成一个固定大小的特征图摘要（Feature-map Summary）。
    *   **单向减法（Subtractive Completion）**：在解码时，从补全缓存中减去已被 Top-K 检索到的 Token 的特征贡献，确保补全项只针对“未被检索”的残差部分。
    *   **统一归一化**：将精确部分的分子/分母与估计部分的分子/分母相加，执行一次统一的归一化操作，从而恢复缺失的 Softmax 质量。
*   **特征图训练**：通过蒸馏学习（Distillation）训练头相关的正向特征图（$\phi_q, \phi_k$），使其在低维度下尽可能模拟原生的 Softmax 核函数。

*   **效率平衡**：补全缓存是固定大小的，其读取开销随生成长度增加而被摊薄。

*   **计算开销**：虽然减少了 I/O，但增加了特征图计算和减法操作的计算量。在计算受限（Compute-bound）而非 I/O 受限的场景下，优势不明显。
