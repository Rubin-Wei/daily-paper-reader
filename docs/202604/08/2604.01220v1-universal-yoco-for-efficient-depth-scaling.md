## 摘要
测试时扩展（test-time scaling）的兴起显著提升了大语言模型（LLMs）的推理能力和智能体能力。然而，标准 Transformer 架构难以高效地扩展推理时计算，因为传统的循环策略面临高昂的计算开销，且 KV 缓存会随模型深度增加而膨胀。我们提出了 Universal YOCO (YOCO-U)，它将 YOCO 的“解码器-解码器”（decoder-decoder）架构与递归计算相结合，实现了优于单一方法的协同效应。基于 YOCO 框架，YOCO-U 实现了一个通用自解码器（Universal Self-Decoder），通过参数共享进行多次迭代，同时将迭代过程限制在浅层的高效注意力层中。这种结合产生了一种优异的能力-效率权衡，这是 YOCO 或递归计算独立无法实现的。YOCO 架构提供了恒定的全局 KV 缓存和线性预填充，而部分递归则以有限的开销增强了表示深度。综上所述，YOCO-U 在保持高效推理的同时，提高了 Token 效用和扩展表现。实验结果证实，YOCO-U 在通用和长文本基准测试中保持了极强的竞争力，表明高效注意力架构与递归计算的结合是可扩展大语言模型的一个极具前景的方向。

## Abstract
The rise of test-time scaling has remarkably boosted the reasoning and agentic proficiency of Large Language Models (LLMs). Yet, standard Transformers struggle to scale inference-time compute efficiently, as conventional looping strategies suffer from high computational overhead and a KV cache that inflates alongside model depth. We present Universal YOCO (YOCO-U), which combines the YOCO decoder-decoder architecture with recursive computation to achieve a synergistic effect greater than either alone. Built on the YOCO framework, YOCO-U implements a Universal Self-Decoder that performs multiple iterations via parameter sharing, while confining the iterative process to shallow, efficient-attention layers. This combination yields a favorable capability-efficiency tradeoff that neither YOCO nor recursion achieves independently. The YOCO architecture provides a constant global KV cache and linear pre-filling, while partial recursion enhances representational depth with limited overhead. Together, YOCO-U improves token utility and scaling behavior while maintaining efficient inference. Empirical results confirm that YOCO-U remains highly competitive in general and long-context benchmarks, demonstrating that the integration of efficient-attention architectures and recursive computation is a promising direction for scalable LLMs.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLMs）进入“测试时扩展”（Test-time Scaling）时代，通过增加推理侧计算量来提升模型逻辑推理能力成为主流。然而，标准 Transformer 架构在扩展计算深度时面临两大瓶颈：
*   **计算开销巨大**：简单的循环（Looping）策略会导致计算量随深度线性增加。
*   **KV 缓存膨胀**：标准架构的 KV 缓存随模型深度增加而增长，导致长文本推理时的内存压力剧增。
本文旨在提出一种既能通过增加计算深度提升性能，又能保持极低内存开销和高效推理速度的新型架构。

#### 2. 论文提出的方法论
论文提出了 **Universal YOCO (YOCO-U)**，其核心思想是将 **YOCO（You Only Cache Once）** 架构与**递归计算（Recursive Computation）** 有机结合：
*   **架构组成**：继承 YOCO 的“解码器-解码器”结构，分为 **Self-Decoder**（自解码器）和 **Cross-Decoder**（交叉解码器）。
*   **通用自解码器（Universal Self-Decoder）**：这是 YOCO-U 的核心创新。将原本静态的 Self-Decoder 替换为可递归的模块，通过参数共享进行 $T$ 次迭代计算。
*   **高效注意力机制**：在递归的 Self-Decoder 中使用高效注意力（如滑动窗口注意力 SWA），确保递归过程中的 KV 缓存开销极小。
*   **全局 KV 缓存共享**：无论 Self-Decoder 迭代多少次，它最终只为 Cross-Decoder 生成一份全局 KV 缓存。Cross-Decoder 在后续所有层中复用这一份缓存，从而实现了“深度扩展”与“内存占用”的解耦。

#### 6. 主要结论与发现
*   **高效的 Token 效用**：在相同 Loss 下，YOCO-U 相比 YOCO 可节省约 **62%** 的训练数据。

#### 7. 优点
*   **协同效应**：成功解决了递归计算在标准 Transformer 中导致的 KV 缓存爆炸问题。
*   **极高的推理效率**：在增加计算深度的同时，几乎不增加内存负担，且保持了线性预填充的优势。
*   **良好的扩展性**：实验证明该架构符合 Scaling Law，且在 MoE 架构下表现稳定。
*   **解耦设计**：将深度扩展与内存开销解耦，为未来超长文本推理提供了可行路径。

#### 8. 不足与局限
*   **递归延迟**：虽然比全注意力递归快，但多次迭代 Self-Decoder 仍会带来一定的序列计算延迟（虽然论文显示仅损失 5% 的解码吞吐量）。

*   **超大规模验证**：虽然测试到了 10B 规模，但在千亿级（100B+）参数模型上的表现和训练稳定性仍需进一步验证。
