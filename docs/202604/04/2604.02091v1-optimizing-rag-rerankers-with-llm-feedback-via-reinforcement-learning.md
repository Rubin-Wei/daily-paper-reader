## 摘要
重排序器在优化检索增强生成（RAG）的检索结果中起着至关重要的作用。然而，目前的重排序模型通常是在静态的人工标注相关性标签上进行独立优化的，与下游的生成过程相脱节。这种脱节导致了根本性的不一致：被信息检索指标认定为主题相关的文档，往往无法提供大语言模型（LLM）进行精确回答生成所需的实际效用。为了弥补这一差距，我们提出了重排序偏好优化（RRPO），这是一种通过强化学习框架，直接将重排序与 LLM 的生成质量相对齐的方法。通过将重排序建模为一个序列决策过程，RRPO 利用 LLM 反馈来优化上下文效用，从而消除了对昂贵人工标注的需求。为了确保训练稳定性，我们进一步引入了一种参考锚定的确定性基准。在知识密集型基准测试上的广泛实验表明，RRPO 显著优于强基准模型，包括强大的列表式重排序器 RankZephyr。进一步的分析强调了我们框架的多功能性：它可以无缝泛化到不同的阅读器（如 GPT-4o），与 Query2Doc 等查询扩展模块正交集成，并且在面对带有噪声的监督者训练时依然保持鲁棒性。

## Abstract
Rerankers play a pivotal role in refining retrieval results for Retrieval-Augmented Generation. However, current reranking models are typically optimized on static human annotated relevance labels in isolation, decoupled from the downstream generation process. This isolation leads to a fundamental misalignment: documents identified as topically relevant by information retrieval metrics often fail to provide the actual utility required by the LLM for precise answer generation. To bridge this gap, we introduce ReRanking Preference Optimization (RRPO), a reinforcement learning framework that directly aligns reranking with the LLM's generation quality. By formulating reranking as a sequential decision-making process, RRPO optimizes for context utility using LLM feedback, thereby eliminating the need for expensive human annotations. To ensure training stability, we further introduce a reference-anchored deterministic baseline. Extensive experiments on knowledge-intensive benchmarks demonstrate that RRPO significantly outperforms strong baselines, including the powerful list-wise reranker RankZephyr. Further analysis highlights the versatility of our framework: it generalizes seamlessly to diverse readers (e.g., GPT-4o), integrates orthogonally with query expansion modules like Query2Doc, and remains robust even when trained with noisy supervisors.

#### 1. 核心问题与研究动机
在检索增强生成（RAG）系统中，重排序器（Reranker）负责从初步检索的文档中筛选出最相关的部分。然而，传统重排序器通常基于静态的人工标注（如 NDCG 指标）进行独立训练，与下游的生成任务脱节。这导致了一个核心矛盾：**主题相关（Topic Relevance）的文档并不等同于对 LLM 生成准确答案有用的文档（LLM Usefulness）**。例如，一个详尽的历史词条可能主题相关，但一个简短的时间轴条目对 LLM 回答特定日期问题更有用。论文旨在通过强化学习直接对齐重排序结果与下游 LLM 的生成质量。

#### 2. 方法论：重排序偏好优化 (RRPO)
*   **核心思想**：将重排序建模为一个**有限步长的马尔可夫决策过程 (MDP)**。重排序器作为智能体（Agent），每一步从候选池中选择一个文档，直到选满 $k$ 个。
*   **关键技术细节**：
    *   **状态与动作**：状态是尚未被选中的候选文档集；动作是选择其中一个文档。
    *   **奖励机制 (Reward)**：将选定的文档序列输入给**冻结的下游 LLM 阅读器**，根据生成答案与标准答案的匹配度（如 EM、F1、Hit 等指标的加权和）计算奖励。
    *   **优化算法**：采用改进的 PPO（近端策略优化）算法。为了解决 LLM 奖励信号稀疏且方差大的问题，引入了**参考锚定确定性基准 (Reference-anchored Deterministic Baseline)**。
    *   **基准计算**：不使用传统的 Critic 网络，而是利用一个固定的参考模型（如原始重排序器）进行贪婪搜索生成的奖励值作为基准（Baseline），从而稳定训练。
    *   **训练约束**：结合了 PPO-clip 机制和 KL 散度惩罚，防止模型更新幅度过大导致性能崩溃。

*   **性能提升**：RRPO 显著提升了所有基础重排序器的表现，在多个指标上超越了参数量更大的 RankZephyr。
*   **强泛化性**：在一个 LLM 阅读器（如 Qwen2.5-7B）上训练的重排序器，可以直接提升 GPT-4o、Claude-3.5 等闭源模型的 RAG 效果。
*   **正交性**：RRPO 与查询扩展技术（如 Query2Doc）可以叠加使用，分别从提高召回率和提高精确度两个维度优化系统。

#### 7. 优点与亮点
*   **端到端对齐**：解决了检索与生成之间的“目标失配”问题，真正以“对生成有用”为优化目标。
*   **无需人工标注**：利用 LLM 自反馈实现自我进化，极大地降低了数据获取成本。
*   **训练稳定性**：创新的“参考锚定基准”避开了难以收敛的 Critic 网络训练，使强化学习在重排序任务中变得可行。
*   **即插即用**：训练后的重排序器具有极强的通用性，不绑定特定的阅读器或提示词模板。

#### 8. 不足与局限
*   **召回率依赖**：作为重排序器，其性能上限受限于初始检索器（如 BM25）的召回能力。如果前 $N$ 个文档中根本没有答案，RRPO 也无法挽救。
*   **计算开销**：尽管有加速策略，但训练过程中仍需频繁调用 LLM 生成答案以获取奖励，相比传统的监督学习，计算成本依然较高。
*   **任务局限**：目前主要在知识密集型问答（QA）任务上验证，对于长文本摘要或创意写作等其他 RAG 场景的适用性尚待进一步探索。
