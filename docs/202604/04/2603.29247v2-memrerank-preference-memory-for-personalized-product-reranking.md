## 摘要
基于大语言模型（LLM）的购物智能体日益依赖长购买历史和多轮交互来实现个性化，然而，由于噪声、长度以及相关性不匹配等问题，直接将原始历史记录附加到提示词（prompts）中往往效果不佳。我们提出了 MemRerank，这是一个偏好记忆框架，它将用户购买历史提炼为简洁且独立于查询的信号，用于个性化商品重排序。为了研究这一问题，我们构建了一个端到端的基准测试和评估框架，其核心是一个基于 LLM 的“五选一”选择任务，该任务同时衡量记忆质量和下游重排序效用。我们进一步利用强化学习（RL）训练记忆提取器，并使用下游重排序性能作为监督信号。针对两种基于 LLM 的重排序器的实验表明，MemRerank 始终优于无记忆、原始历史记录以及现成的记忆基准模型，在“五选一”准确率上最高提升了 10.61 个绝对百分点。这些结果表明，显式偏好记忆是智能体电商系统中实现个性化的一个实用且有效的构建模块。

## Abstract
LLM-based shopping agents increasingly rely on long purchase histories and multi-turn interactions for personalization, yet naively appending raw history to prompts is often ineffective due to noise, length, and relevance mismatch. We propose MemRerank, a preference memory framework that distills user purchase history into concise, query-independent signals for personalized product reranking. To study this problem, we build an end-to-end benchmark and evaluation framework centered on an LLM-based \textbf{1-in-5} selection task, which measures both memory quality and downstream reranking utility. We further train the memory extractor with reinforcement learning (RL), using downstream reranking performance as supervision. Experiments with two LLM-based rerankers show that MemRerank consistently outperforms no-memory, raw-history, and off-the-shelf memory baselines, yielding up to \textbf{+10.61} absolute points in 1-in-5 accuracy. These results suggest that explicit preference memory is a practical and effective building block for personalization in agentic e-commerce systems.

随着大语言模型（LLM）在电商导购智能体中的应用，如何利用用户长期的购买历史实现**个性化推荐**成为关键。传统做法是直接将原始历史记录（Raw History）拼接在 Prompt 中，但这面临三大挑战：
*   **噪声干扰**：历史记录中包含大量与当前查询无关的信息。
*   **上下文限制**：长历史容易撑爆 LLM 的上下文窗口，导致处理效率低下。
*   **相关性失配**：原始数据缺乏结构化提炼，LLM 难以直接提取出稳定的用户偏好。
论文提出了 **MemRerank** 框架，旨在将冗长的历史记录提炼为简洁、结构化的“偏好记忆”，以提升商品重排序（Reranking）的准确性。

### 2. 方法论：核心思想与关键技术
MemRerank 的核心是将个性化信号从“原始数据”转变为“提炼后的记忆”，其流程包含两个阶段：
*   **偏好记忆提取（Preference Memory Extraction）**：
    *   使用一个 LLM（如 Qwen2.5-7B）作为提取器，将用户的购买历史（包括类目、标题、描述等元数据）转化为结构化的文本。
    *   区分**类目内（Within-category）**和**跨类目（Cross-category）**偏好。类目内记忆关注特定商品的细节偏好，跨类目记忆捕捉通用的购物风格（如预算敏感度、品牌倾向）。
*   **强化学习训练（RL Training with GRPO）**：
    *   **核心创新**：不依赖人工标注的记忆模板，而是使用**下游重排序的准确率**作为奖励信号（Reward）。
    *   **奖励函数**：由“格式奖励”（确保输出符合特定的 XML 标签结构）和“重排序奖励”（如果下游重排序器选对了目标商品则给予正向反馈）组成。
    *   通过 GRPO 算法优化提取器，使其生成的记忆对重排序任务最有帮助。

*   **数据集与基准（Benchmark）**：

    *   主要在 **Electronics（电子产品）** 类目上进行深入实验。
*   **对比方法**：
    *   **无记忆（No memory）**：仅凭查询和候选集排序。

### 6. 主要结论与发现
*   **性能显著提升**：MemRerank 在 o4-mini 上的准确率比无记忆基准提升了 **10.61%**。
*   **训练优于原生**：经过 RL 训练的 7B 提取器生成的记忆，效果优于未训练的 GPT-5.2 提取的记忆。
*   **Prompt 设计至关重要**：半结构化且带有证据溯源（Evidence Grounding）的 Prompt（v3）效果最好，因为它在灵活性和约束性之间取得了平衡。
*   **原始历史的局限**：简单增加原始历史长度并不总是有效，甚至可能引入噪声导致性能下降，而提炼后的记忆则更稳健。

### 7. 优点与亮点
*   **闭环优化**：通过 RL 将记忆提取与下游任务性能直接挂钩，解决了“什么样的记忆才是好记忆”的定义难题。
*   **结构化设计**：区分内/外类目偏好，符合人类购物逻辑，增强了模型的可解释性。
*   **实用性强**：生成的记忆是查询无关的，可以一次提取、多次复用，显著降低了在线推理的 Token 消耗。

### 8. 不足与局限
*   **类目覆盖有限**：主要实验集中在电子产品类目，不同类目（如服装 vs 食品）的偏好特征差异巨大，泛化性有待验证。
*   **规模限制**：目前仅关注“五选一”的重排序阶段，尚未在更大规模（如从 100 个候选物中排序）的端到端检索场景中测试。
*   **数据依赖**：依赖于高质量的元数据和评论，如果用户历史数据稀疏，记忆提取的效果可能会大幅打折。
