## 摘要
大语言模型（LLM）在实际服务中表现出强大的性能，但在跨用户和会话存在重复或近乎重复查询的工作负载下，其推理成本高昂。在这项工作中，我们提出了 MemBoost，这是一种记忆增强的 LLM 服务框架，它使轻量级模型能够重用先前生成的答案并检索相关的支持信息以实现低成本推理，同时选择性地将困难或不确定的查询升级给更强大的模型。与主要针对单次响应提供依据的标准检索增强生成（RAG）不同，MemBoost 专为交互式场景设计，支持答案重用、记忆持续增长和成本感知路由。在模拟工作负载下针对多个模型的实验表明，MemBoost 显著减少了昂贵的大模型调用次数和整体推理成本，同时保持了与强模型基准相当的高质量回答。

## Abstract
Large Language Models (LLMs) deliver strong performance but incur high inference cost in real-world services, especially under workloads with repeated or near-duplicate queries across users and sessions. In this work, we propose MemBoost, a memory-boosted LLM serving framework that enables a lightweight model to reuse previously generated answers and retrieve relevant supporting information for cheap inference, while selectively escalating difficult or uncertain queries to a stronger model. Unlike standard retrieval-augmented generation, which primarily grounds a single response, MemBoost is designed for interactive settings by supporting answer reuse, continual memory growth, and cost-aware routing. Experiments across multiple models under simulated workloads show that MemBoost substantially reduces expensive large-model invocations and overall inference cost, while maintaining high answer quality comparable to the strong model baseline.

### 1. 核心问题与研究动机
*   **核心问题**：在实际的 LLM 服务中，推理成本极高，且存在大量跨用户、跨会话的**重复或语义近似查询**。现有的技术（如 RAG、语义缓存、模型路由）往往孤立存在，缺乏一个统一的框架来同时实现答案复用、动态记忆增长和成本感知的模型升级。
*   **研究背景**：部署前沿大模型需要昂贵的 GPU 资源。虽然 RAG 可以降低知识存储压力，但其主要用于知识接地（Grounding），而非减少冗余计算。本文旨在通过一个“检索-决策-升级”的闭环系统，在保证回答质量的前提下，最大限度降低昂贵大模型的调用频率。

### 2. 方法论（MemBoost 框架）
MemBoost 由三个核心组件构成，形成一个动态循环系统：
*   **关联记忆引擎 (AME)**：
    *   存储历史生成的“问题-答案”对及元数据。
    *   使用向量嵌入（Embedding）和 FAISS 索引进行快速语义检索。
    *   支持**写回机制**：当系统调用强模型生成新答案后，可将其存入 AME 供未来复用。
*   **大语言模型 Oracle (强模型)**：
    *   作为系统的“兜底”方案，仅在记忆库无法提供可靠答案时调用。它具备高能力但高成本。
*   **元控制器 (MC)**：
    *   由一个轻量级 LLM 担任，负责调度。
    *   **流程**：接收查询 $x_t$ → 从 AME 检索候选答案 $M_t$ → 决策：是直接利用检索内容合成答案，还是升级（Escalate）给 Oracle。
    *   **成本感知路由**：MC 评估检索内容的可靠性，若确定性高则低成本输出；若不确定则调用 Oracle，并决定是否将 Oracle 的高质量输出写回 AME。

*   **数据集与场景**：使用 **MMLU-Pro** 评估集（特别是 Business 类别）。为了模拟真实世界的重复查询，研究者采用 **Zipf 分布**（参数 $\alpha \in \{0.8, 1.1, 1.4\}$）对问题进行采样，生成包含 5000 个请求的查询流。
*   **Benchmark（基准）**：

    *   **小模型基准**：独立运行 Qwen3.5-2B、Ministral-3B、Qwen3-4B。
*   **对比方法**：将上述小模型作为 MemBoost 的元控制器（MC），观察其在集成 AME 和 Oracle 后的表现。

*   **实验规模**：进行了 5000 步的连续流式推理模拟。
*   **维度覆盖**：
    *   测试了 3 种不同的重复率（Zipf $\alpha$ 值），覆盖了从低重复到高重复的场景。

*   **充分性评价**：实验设计较为客观，通过对比准确率、内存使用率（It）和响应延迟三个维度，全面衡量了成本与质量的权衡。但实验主要集中在选择题形式的 MMLU-Pro 上，任务类型略显单一。

### 6. 主要结论与发现
*   **性能持平甚至超越**：MemBoost 在所有设置下均显著优于小模型基准，且准确率与 14B 的 Oracle 持平。在某些情况下（如使用 Qwen3.5-2B 作为 MC），其准确率甚至超过了 Oracle，这得益于对历史正确答案的稳定复用。
*   **显著降本增效**：随着记忆库的持续填充，Oracle 的调用率显著下降，系统延迟随之降低。在重复率较高的负载下（Zipf 1.4），内存使用率（即无需调用 Oracle 的比例）可接近 90%。
*   **动态适应性**：工作负载越倾斜（重复率越高），MemBoost 的优势越明显。

### 7. 优点与亮点
*   **闭环学习能力**：不同于静态缓存，MemBoost 具有“写回”机制，使系统在运行过程中变得越来越“聪明”且廉价。
*   **灵活的模块化设计**：MC、AME 和 Oracle 均可独立更换，方便适配不同的业务需求和预算。
*   **实用性强**：直接针对工业界 LLM 服务中“重复查询多、推理贵”的痛点，具有很高的落地价值。

### 8. 不足与局限
*   **任务多样性不足**：实验仅限于 MMLU-Pro 的选择题，未在长文本生成、代码编写或开放式对话等更复杂的任务上验证。
*   **语义误判风险**：在处理“语义相似但意图不同”的查询时，AME 检索可能存在“假阳性”（False Hit），导致 MC 给出错误答案，文中对这一鲁棒性问题的深入探讨较少。
*   **冷启动问题**：在系统运行初期，记忆库为空，此时系统成本与直接调用 Oracle 无异，且增加了 MC 的额外开销。
