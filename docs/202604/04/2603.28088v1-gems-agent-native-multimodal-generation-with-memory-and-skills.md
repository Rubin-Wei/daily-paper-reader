## 摘要
近期多模态生成模型在通用生成任务上取得了显著进展，但在处理复杂指令和特定下游任务时仍面临挑战。受 Claude Code 等先进智能体框架成功的启发，我们提出了 GEMS（具备记忆与技能的智能体原生多模态生成），这是一个旨在突破基础模型在通用及下游任务中固有局限性的框架。GEMS 构建于三个核心组件之上：智能体循环（Agent Loop）引入了一个结构化的多智能体框架，通过闭环优化迭代提升生成质量；智能体记忆（Agent Memory）提供了一种持久的轨迹级记忆，分层存储事实状态和压缩后的经验总结，在减少冗余的同时实现了对优化过程的全局视野；智能体技能（Agent Skill）提供了一套可扩展的领域特定专业知识库，支持按需加载，使系统能够有效处理多样化的下游应用。在五项主流任务和四项下游任务中，通过在多个生成后端上的评估，GEMS 一致地实现了显著的性能提升。最值得注意的是，它使轻量级 6B 模型 Z-Image-Turbo 在 GenEval2 上超越了当前最先进的 Nano Banana 2，证明了智能体治理在扩展模型能力、突破其原始极限方面的有效性。

## Abstract
Recent multimodal generation models have achieved remarkable progress on general-purpose generation tasks, yet continue to struggle with complex instructions and specialized downstream tasks. Inspired by the success of advanced agent frameworks such as Claude Code, we propose \textbf{GEMS} (Agent-Native Multimodal \textbf{GE}neration with \textbf{M}emory and \textbf{S}kills), a framework that pushes beyond the inherent limitations of foundational models on both general and downstream tasks. GEMS is built upon three core components. Agent Loop introduces a structured multi-agent framework that iteratively improves generation quality through closed-loop optimization. Agent Memory provides a persistent, trajectory-level memory that hierarchically stores both factual states and compressed experiential summaries, enabling a global view of the optimization process while reducing redundancy. Agent Skill offers an extensible collection of domain-specific expertise with on-demand loading, allowing the system to effectively handle diverse downstream applications. Across five mainstream tasks and four downstream tasks, evaluated on multiple generative backends, GEMS consistently achieves significant performance gains. Most notably, it enables the lightweight 6B model Z-Image-Turbo to surpass the state-of-the-art Nano Banana 2 on GenEval2, demonstrating the effectiveness of agent harness in extending model capabilities beyond their original limits.

尽管当前的多模态生成模型（如 T2I 模型）在通用任务上表现出色，但在面对**复杂的多面指令**和**专业化的下游任务**（如精确文字渲染、复杂空间关系、特定艺术风格）时，仍存在“长尾效应”挑战。现有的推理侧优化方法（如简单的提示词重写或基础的迭代循环）往往面临引导信息不足、历史上下文冗余或领域知识难以灵活集成等局限。

本论文提出了 **GEMS** 框架，旨在从“智能体原生（Agent-Native）”的角度重新设计生成流程，通过引入结构化的循环优化、持久化的分层记忆和可扩展的技能库，突破基础生成模型的能力边界。

#### 2. 论文提出的方法论
GEMS 框架由三个核心支柱组成：

*   **Agent Loop（智能体循环）**：构建了一个多智能体协作闭环。
    *   **Planner & Decomposer**：规划器负责识别任务领域并调用技能；分解器将原始指令拆解为多个可验证的原子视觉准则（二进制判断题）。
    *   **Generator & Verifier**：生成器（模型无关）产生图像；验证器（基于 MLLM）根据原子准则对图像进行逐项检查。
    *   **Refiner**：若未通过验证，细化器结合当前状态、反馈和历史记忆，进化出更高质量的提示词进入下一轮迭代。
*   **Agent Memory（智能体记忆）**：采用**分层压缩（Hierarchical Compression）**策略。
    *   **事实工件（Factual Artifacts）**：如提示词、图像和反馈，以原始形式存档。
    *   **经验总结（Experiential Summaries）**：将冗长的思维链（CoT）过程通过压缩器蒸馏为高层次的策略经验，减少 Token 冗余并提供全局优化视角。
*   **Agent Skill（智能体技能）**：建立了一个可扩展的领域专家库。
    *   采用**按需加载（On-demand Loading）**机制：平时仅加载轻量级的技能清单（Manifest），只有当 Planner 触发特定技能时，才加载详细的领域指令（Instructions），从而降低推理负担并提高系统扩展性。

#### 7. 优点（亮点）
*   **智能体原生架构**：不同于简单的提示词工程，GEMS 实现了真正的闭环反馈和轨迹管理。
*   **高效的记忆管理**：通过“事实+蒸馏经验”的分层设计，平衡了上下文丰富度与计算效率。
*   **模块化与可扩展性**：Agent Skill 允许用户通过简单的 Markdown 文件添加新技能，无需修改核心逻辑，极具实用价值。
*   **推理效率优化**：引入早期停止机制，在保证质量的前提下降低了多轮迭代的平均成本。

#### 8. 不足与局限
*   **推理延迟**：尽管有优化，但迭代生成本质上比单次生成慢，不适用于对实时性要求极高的场景。

*   **任务范围限制**：目前主要集中在图像生成，尚未扩展到视频生成或复杂的图像编辑任务。
*   **预定义流的刚性**：目前的协作逻辑仍基于预定义的工作流，智能体自主管理工具和记忆的灵活性仍有提升空间。
