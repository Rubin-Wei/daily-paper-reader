## 摘要
现有的语言智能体记忆系统主要解决记忆管理问题：即如何在上下文预算内检索和分页更多信息。我们则探讨了一个互补的问题——记忆效用：哪些经验值得保留，以及这些经验应如何改变智能体的行为。我们提出了 Atlas，这是一个记忆内核，它能将积累的任务经验编译进智能体的指令结构中，且无需微调、检索增强生成（RAG）或人工干预。在这里，记忆是蒸馏而非存储；交付是指令重写而非上下文注入。从智能体的失败和成功案例中提取的事实经过三步晋升门控验证，并通过使用学到的子要点重写智能体的系统提示词来交付。在 CUAD 合同分析任务中，进化后的提示词将 GPT-4o 的词元级 F1 提升了 8.7 个百分点，精确率提升了 12.5 个百分点。在 HotpotQA 多跳问答中，联合 F1 提升了 3.16 个百分点。消融实验分离出了该机制的核心特性——训练信号约束：进化后的提示词精确地学习所教内容，不多也不少。将同样的进化提示词（由 GPT-4o 的错误编译而成，未经修改）应用于 Claude Sonnet 4.5，其联合 F1 提升了 2.31 个百分点，且增益集中在 Claude 较强基准线留有最大提升空间的领域——这证实了编译后的知识是任务导向的，而非模型导向的。

## Abstract
Existing memory systems for language agents address memory management: how to retrieve and page more information within a context budget. We address a complementary problem -- memory utility: what experience is worth keeping, and how it should change agent behavior. We present Atlas, a memory kernel that compiles accumulated task experience into an agent's instruction structure -- without fine-tuning, RAG, or human intervention. Memory is distillation, not storage; delivery is instruction rewriting, not context injection. Facts extracted from agent failures and successes are verified through a three-step promotion gate and delivered by rewriting the agent's system prompt with learned sub-bullets. On CUAD contract analysis, the evolved prompt improves GPT-4o token-level F1 by $+8.7$pp and precision by $+12.5$pp. On HotpotQA multi-hop QA, joint F1 improves $+3.16$pp. An ablation isolates the mechanism's defining property -- the training signal constraint: the evolved prompt learns exactly what it is taught, and nothing more. Applied to Claude Sonnet~4.5 using the same evolved prompt -- compiled from GPT-4o errors, unchanged -- joint F1 improves $+2.31$pp, with gains concentrating where Claude's stronger baseline leaves the most room -- confirming that the compiled knowledge is task-shaped, not model-shaped.

### 2. 方法论：核心思想与关键技术
*   **核心思想**：**指令重写（Instruction Rewriting）优于上下文注入**。通过将验证后的事实转化为系统提示词中的子要点（Sub-bullets），在不增加推理成本的前提下提升性能。
*   **四层蒸馏模型**：
    1.  **Fresh/Task 层**：捕获运行中的原始信号。
    2.  **Contextual 层**：存储单次任务的完整情节。
    3.  **Historical 层**：存储经过验证的、具有高置信度的持久事实。
*   **三步验证门控（Promotion Gate）**：
    1.  **去重**：通过向量嵌入拒绝相似度 > 0.92 的候选事实。
    2.  **LLM 验证**：批量评估事实的有效性，分为接受、拒绝、合并或需人工复核。
    3.  **接地（Grounding）**：通过词法启发式检查（至少 2 个关键词重合）防止 LLM 幻觉。
*   **置信度算法**：采用公式计算事实的置信度 $c$，通过多次“佐证（Corroboration）”提升分值，若出现“矛盾（Contradiction）”则大幅降分并标记复核。
*   **读写路径**：
    *   **写路径**：分析错误产生“边界规则”（如：在哪里停止提取），分析成功产生“保护事实”（防止规则被过度弱化）。
    *   **读路径**：筛选 $c \ge 0.85$ 的事实，通过 LLM 进化提示词，并注入“锚点指令”以平衡精确率和召回率。

### 6. 主要结论与发现
*   **性能提升**：在 CUAD 上 F1 提升 8.7pp，精确率大幅提升 12.5pp；在 HotpotQA 上联合 F1 提升 3.16pp。

*   **任务导向而非模型导向**：由 GPT-4o 错误生成的指令对 Claude 同样有效，且增益集中在 Claude 自身的薄弱环节。这证明了编译后的知识反映的是任务本身的逻辑结构。

### 7. 优点
*   **高效性**：无需微调，不增加推理延迟，不占用额外的上下文窗口。
*   **鲁棒性**：通过三步验证门控和置信度机制，有效过滤了噪声和幻觉。
*   **通用性**：证明了经验可以在不同架构的 LLM 之间迁移。
*   **逻辑清晰**：将记忆视为“指令编译”而非“数据存储”的视角非常新颖且具有实践意义。

### 8. 不足与局限
*   **负样本处理**：在 CUAD 实验中仅使用了正样本，可能无法全面反映模型在处理无关信息时的误报率。
*   **提示词长度**：进化后的提示词长度约为原提示词的 1.5 倍（从 ~960 增加到 ~1,570 tokens），虽然不影响推理逻辑，但会略微增加首字延迟（TTFT）。

*   **验证机制**：目前的接地（Grounding）检查基于简单的关键词重合，而非深层的语义蕴含分析。
