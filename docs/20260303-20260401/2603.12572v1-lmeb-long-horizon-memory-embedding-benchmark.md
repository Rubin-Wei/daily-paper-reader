## 摘要
记忆嵌入对于 OpenClaw 等记忆增强系统至关重要，但在目前的文本嵌入基准测试中，对其评估的研究尚不充分。现有的基准测试往往局限于传统的段落检索，未能评估模型处理涉及碎片化、上下文相关且时间跨度较大的长程记忆检索任务的能力。为了解决这一问题，我们推出了长程记忆嵌入基准（LMEB），这是一个评估嵌入模型处理复杂长程记忆检索任务能力的综合框架。LMEB 涵盖了 22 个数据集和 193 个零样本检索任务，涉及四种记忆类型：情景记忆、对话记忆、语义记忆和程序记忆，并包含 AI 生成和人工标注的数据。这些记忆类型在抽象程度和时间依赖性方面各不相同，捕捉了记忆检索的不同维度，反映了现实世界中的多样化挑战。我们评估了 15 个广泛使用的嵌入模型，其参数量从数亿到百亿不等。结果表明：（1）LMEB 提供了合理的难度水平；（2）参数量更大的模型并不总是表现更好；（3）LMEB 与 MTEB 表现出正交性。这说明该领域尚未收敛出一个能在所有记忆检索任务中均表现优异的通用模型，且在传统段落检索中的性能表现可能无法推广到长程记忆检索。总之，通过提供一个标准化且可复现的评估框架，LMEB 填补了记忆嵌入评估领域的关键空白，推动了文本嵌入在处理长期、上下文相关的记忆检索方面的进一步发展。LMEB 已在 https://github.com/KaLM-Embedding/LMEB 开源。

## Abstract
Memory embeddings are crucial for memory-augmented systems, such as OpenClaw, but their evaluation is underexplored in current text embedding benchmarks, which narrowly focus on traditional passage retrieval and fail to assess models' ability to handle long-horizon memory retrieval tasks involving fragmented, context-dependent, and temporally distant information. To address this, we introduce the Long-horizon Memory Embedding Benchmark (LMEB), a comprehensive framework that evaluates embedding models' capabilities in handling complex, long-horizon memory retrieval tasks. LMEB spans 22 datasets and 193 zero-shot retrieval tasks across 4 memory types: episodic, dialogue, semantic, and procedural, with both AI-generated and human-annotated data. These memory types differ in terms of level of abstraction and temporal dependency, capturing distinct aspects of memory retrieval that reflect the diverse challenges of the real world. We evaluate 15 widely used embedding models, ranging from hundreds of millions to ten billion parameters. The results reveal that (1) LMEB provides a reasonable level of difficulty; (2) Larger models do not always perform better; (3) LMEB and MTEB exhibit orthogonality. This suggests that the field has yet to converge on a universal model capable of excelling across all memory retrieval tasks, and that performance in traditional passage retrieval may not generalize to long-horizon memory retrieval. In summary, by providing a standardized and reproducible evaluation framework, LMEB fills a crucial gap in memory embedding evaluation, driving further advancements in text embedding for handling long-term, context-dependent memory retrieval. LMEB is available at https://github.com/KaLM-Embedding/LMEB.

#### 1. 核心问题与整体含义（研究动机和背景）
随着智能体（Agent）和记忆增强系统（如 Open-Claw）的发展，**记忆嵌入（Memory Embeddings）**成为存储、检索和推理海量信息的基石。然而，现有的文本嵌入基准（如 MTEB、BEIR）主要关注传统的段落检索，即从组织良好的语料库中寻找匹配信息。这忽略了现实场景中**长程记忆检索（Long-horizon Memory Retrieval）**的复杂性，这类任务通常涉及碎片化、高度依赖上下文、且在时间上跨度极大的信息。论文旨在填补这一评估空白，提出一个能够衡量模型在复杂记忆任务中表现的统一框架。

#### 2. 核心方法论
论文提出了 **LMEB (Long-horizon Memory Embedding Benchmark)**，其核心思想是将记忆检索划分为四个维度，并建立标准化的评估协议：
*   **记忆分类学（Taxonomy）**：基于“抽象程度”和“时间依赖性”两个维度，将记忆分为四类：
    *   **情景记忆（Episodic）**：低抽象、高时间依赖（如特定事件及其顺序）。
    *   **对话记忆（Dialogue）**：高抽象、高时间依赖（如跨多轮对话的用户偏好）。
    *   **语义记忆（Semantic）**：低抽象、低时间依赖（如稳定的通用知识）。
    *   **程序记忆（Procedural）**：高抽象、低时间依赖（如技能、动作序列）。
*   **标准化协议**：采用类似 IR（信息检索）的格式（Queries, Corpus, Qrels），并引入可选的 `candidates` 文件来模拟现实中的“局部记忆池”限制（如仅在特定对话历史中检索）。
*   **指令增强**：评估模型在有/无任务指令（Instructions）情况下的表现，以衡量模型对自然语言任务描述的遵循能力。

*   **实验规模**：在 22 个数据集、193 个任务上对 15 个模型进行了双重设置（有指令 vs 无指令）的全面测试，实验数据量巨大且覆盖面广。

*   **难度适中**：顶级模型在 NDCG@10 上达到约 61.41 分，说明 LMEB 既有挑战性又非不可逾越。

#### 7. 优点（亮点）
*   **填补空白**：首次系统性地针对“长程记忆”这一 Agent 核心能力构建嵌入基准。
*   **分类科学**：借鉴认知心理学对记忆的分类，使评估具有理论支撑。
*   **易扩展性**：提供了开源工具包，支持通过简单的配置文件集成新模型和新数据集。
*   **现实模拟**：通过 `candidates` 机制模拟了受限内存空间的检索，比全语料检索更符合实际应用。

#### 8. 不足与局限
*   **语言局限**：目前 LMEB 主要集中在**英语**数据集，缺乏对多语言记忆检索的深入探讨。
*   **动态性缺失**：虽然涉及长程记忆，但实验仍基于静态数据集的检索，未能完全模拟记忆在交互过程中的**动态更新与遗忘**机制。
*   **长文本限制**：部分模型受限于 512 或 1024 的上下文窗口，可能无法完全发挥处理极长记忆序列的潜力。
