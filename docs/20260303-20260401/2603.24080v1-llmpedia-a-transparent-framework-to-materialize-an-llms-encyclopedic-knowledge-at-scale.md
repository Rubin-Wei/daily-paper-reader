## 摘要
MMLU 等基准测试表明，旗舰级语言模型在事实性方面已接近饱和，得分超过 90%。我们证明这一现状并不完整。LLMpedia 完全基于参数化记忆生成百科全书文章，在不使用检索的情况下，跨三个模型系列生成了约 100 万篇文章。对于 gpt-5-mini，在维基百科覆盖的主题上，可验证的真实率仅为 74.7%——比基于基准测试的结果低 15 个百分点以上，这与固定问题评估中的可用性偏差一致。在维基百科之外，仅能通过精选网络证据验证的前沿主题，其真实率进一步下降至 63.2%。维基百科仅覆盖了 61% 的已出现主题，且三个模型系列在主题选择上的重合度仅为 7.3%。在受先前 Grokipedia 分析启发的 capture-trap 基准测试中，LLMpedia 在与维基百科文本相似度仅为约一半的情况下，实现了显著更高的事实性。与 Grokipedia 不同，LLMpedia 的每个提示词、中间产物和评估结论均已公开发布，使其成为首个完全开放的参数化百科全书，架起了事实性评估与知识具象化之间的桥梁。所有数据、代码和可浏览界面均可在 https://llmpedia.net 获取。

## Abstract
Benchmarks such as MMLU suggest flagship language models approach factuality saturation, with scores above 90\%. We show this picture is incomplete. \emph{LLMpedia} generates encyclopedic articles entirely from parametric memory, producing ${\sim}$1M articles across three model families without retrieval. For gpt-5-mini, the verifiable true rate on Wikipedia-covered subjects is only 74.7\% -- more than 15 percentage points below the benchmark-based picture, consistent with the availability bias of fixed-question evaluation. Beyond Wikipedia, frontier subjects verifiable only through curated web evidence fall further to 63.2\% true rate. Wikipedia covers just 61\% of surfaced subjects, and three model families overlap by only 7.3\% in subject choice. In a capture-trap benchmark inspired by prior analysis of Grokipedia, LLMpedia achieves substantially higher factuality at roughly half the textual similarity to Wikipedia. Unlike Grokipedia, every prompt, artifact, and evaluation verdict is publicly released, making LLMpedia the first fully open parametric encyclopedia -- bridging factuality evaluation and knowledge materialization. All data, code, and a browsable interface are at https://llmpedia.net.

### 2. 方法论：核心思想与关键技术
LLMpedia 采用纯参数化生成（不依赖推理时的检索），其核心流程如下：
*   **递归具象化（Recursive Elicitation）**：从种子实体（如 Vannevar Bush）开始，利用广度优先搜索（BFS）不断扩展。
*   **动态大纲生成**：为每个实体生成定制化的 6-7 个章节大纲，而非使用固定模板，以提高生成质量。
*   **三阶段实体清洗流水线（Sanitization Pipeline）**：
    1.  **规范化（Canonical Normalization）**：处理 Unicode、大小写和标点，消除简单重复。
    2.  **基于 LLM 的百科过滤**：识别并剔除普通名词（如“工程师”）或循环链接，仅保留具有百科价值的命名实体。
    3.  **基于嵌入的去重（Embedding-based Deduplication）**：利用向量相似度（阈值 0.90）和 LLM 仲裁，解决语义重复（如“Germany”与“Deutschland”）。
*   **自接地（Self-Grounding）**：可选步骤，生成事实清单并标注置信度，以锚定后续文章内容。
*   **编辑人格注入（Persona Injection）**：在系统提示词中注入科学中立、左翼或保守派人格，研究意识形态对知识选择和框架的影响。

### 3. 实验设计
*   **数据集与场景**：

    *   **Capture-trap 基准**：选取维基百科编辑量最大的 1000 个条目，对比 LLMpedia 与 xAI 的 Grokipedia。
*   **对比模型**：GPT-5-mini、Llama-3.3-70B-Instruct、DeepSeek-V3。
*   **验证基准（Benchmark）**：
    *   **Tier 1 (Wikipedia)**：通过 MediaWiki API 验证。

### 4. 资源与算力
*   **算力设备**：
    *   **Llama-3.3-70B** 与 **DeepSeek-V3**：部署在机构内部的 **4 × A100 GPU** 集群上。
    *   **GPT-5-mini**：通过 OpenAI Batch API 运行。
*   **成本与时长**：项目总 API 开销约为 **3,000 美元**（含生成、消融实验及 GPT-4.1-nano 作为评估法官的费用）。具体训练/生成时长未详细列出，但提到使用了异步批处理模式。

### 5. 实验数量与充分性
*   **实验规模**：生成了约 100 万篇文章，其中 GPT-5-mini 贡献了约 74 万篇。
*   **消融实验**：进行了 **2×2×2 因子消融实验**，涵盖自接地（开/关）、提示策略（基准/校准）、推理预算（极小/低），共 8 组配置，每组评估 500 篇文章。
*   **充分性与客观性**：实验设计考虑了 BFS 深度对事实性的影响，并引入了跨模型的实体重合度分析。通过双层验证体系（维基+Web）确保了对非维基知识的客观评价。

### 6. 主要结论与发现
*   **事实性缺口**：GPT-5-mini 在维基百科覆盖主题上的真实率仅为 **74.7%**，比 MMLU 等基准测试显示的 90%+ 低了 15 个百分点以上。
*   **知识独特性**：三个模型系列在主题选择上的重合度极低（仅 **7.3%**），说明模型参数化记忆具有高度的模型特异性。
*   **前沿知识挑战**：维基百科仅覆盖了模型生成主题的 **61%**；在维基百科之外的主题中，真实率进一步下降至 **63.2%**。
*   **Grokipedia 的“捕获陷阱”**：实验证明 Grokipedia 与维基百科文本相似度极高（疑似检索驱动），但事实性显著低于 LLMpedia。LLMpedia 在相似度仅为前者一半的情况下，事实性更高。

### 7. 优点
*   **高度透明**：不同于闭源的 Grokipedia，LLMpedia 开源了所有提示词、中间产物、评估代码和数据。
*   **创新的验证体系**：建立了一套严格的 Tier 2 网络证据验证流程，能够处理 LLM 生成的“维基百科之外”的知识。
*   **规模化能力**：展示了如何通过自动化的实体清洗流水线，在不引入检索的情况下构建百万量级的知识库。

### 8. 不足与局限
*   **单次采样偏差**：所有文章均基于 Temperature=0 的单次生成，未能完全捕捉模型潜在的知识分布（即“模型知道但没写出来”的部分）。
*   **评估法官依赖**：依赖单一模型（GPT-4.1-nano）进行事实提取和裁决，可能存在模型偏见。
*   **Web 证据覆盖有限**：约 24.3% 的前沿主题因无法找到高质量网络证据而被排除在 Tier 2 统计之外，可能导致结果偏向于文档丰富的条目。
*   **维度局限**：主要关注命题事实性，未深入评估文章的连贯性、中立性、重要性缺失（Omission）等编辑质量指标。
