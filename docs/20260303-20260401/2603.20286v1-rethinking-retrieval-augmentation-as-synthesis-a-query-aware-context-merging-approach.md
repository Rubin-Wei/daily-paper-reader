## 摘要
检索增强生成（RAG）通过动态整合外部信息，使大语言模型（LLM）能够扩展其现有知识。然而，实际部署受到 LLM 有限上下文窗口的根本限制，迫使在信息充分性与 Token 消耗之间进行权衡。标准流程通过“检索后选择”策略来解决这一问题，通常仅保留基于相关性的前 k 个数据块。尽管如此，这种方法并非最优：它固有的缺陷在于会截断位于相关性分布长尾中的关键桥接证据，同时在语义冗余的高排名数据块上浪费 Token 预算。在本文中，我们将检索增强重新构想为一个旨在最大化信息密度的动态优化问题。我们提出了 MergeRAG，这是一个将范式从静态过滤转变为查询感知合成的新颖框架。MergeRAG 采用评分代理通过双路径机制重构检索到的上下文：1）对称合并（Symmetric Merging），通过整合弱信号来恢复丢失的桥接证据；2）非对称合并（Asymmetric Merging），利用熵引导锚定在不牺牲语义完整性的情况下消除冗余。我们进一步引入了一种层级并行合并策略，在最大化计算并行性的同时减轻信息损失。在标准基准测试上的广泛实验表明，MergeRAG 显著优于最先进的 RAG 基准模型，在 F1 分数和精确匹配（EM）上分别实现了高达 13.7 点和 11.5 点的提升。

## Abstract
Retrieval-Augmented Generation (RAG) enables Large Language Models (LLMs) to extend their existing knowledge by dynamically incorporating external information. However, practical deployment is fundamentally constrained by the LLM's finite context window, forcing a trade-off between information sufficiency and token consumption. Standard pipelines address this via a retrieve-then-select strategy, typically retaining only the top-k chunks based on relevance. Nevertheless, this approach is suboptimal: it inherently truncates critical bridging evidence located in the long tail of the relevance distribution, while simultaneously wasting the token budget on semantically redundant high-ranking chunks.   In this paper, we rethink retrieval-augmentation as a dynamic optimization problem aimed at maximizing information density. We propose MergeRAG, a novel framework that shifts the paradigm from static filtering to query-aware synthesis. MergeRAG employs a scoring agent to restructure retrieved contexts through a dual-pathway mechanism: 1) Symmetric Merging, which consolidates weak signals to recover lost bridging evidence; 2) Asymmetric Merging, which utilizes entropy-guided anchoring to eliminate redundancy without sacrificing semantic integrity. We further introduce a Hierarchical Parallel Merging strategy that mitigates information loss while maximizing computational parallelism. Extensive experiments on standard benchmarks demonstrate that MergeRAG significantly outperforms state-of-the-art RAG baselines, achieving up to 13.7 points improvement in F1 score and 11.5 points in Exact Match (EM), respectively.

#### 1. 核心问题与研究动机
传统的检索增强生成（RAG）通常采用“检索后选择（Retrieve-then-Select）”的范式，即根据相关性得分保留前 $k$ 个数据块（Chunks）。这种做法存在两个核心缺陷：
*   **信息丢失（长尾效应）**：复杂的查询往往需要多个碎片化证据的支撑，而关键的“桥接证据”可能隐藏在相关性得分较低的排名靠后的数据块中，被硬性截断。
*   **语义冗余**：排名靠前的数据块往往包含大量重复信息，白白浪费了 LLM 有限的上下文窗口（Token 预算），挤占了其他独特细节的空间。
*   **静态局限**：现有的预处理方法（如静态聚类或摘要）是查询无关的，无法灵活适应动态的用户意图。

#### 2. 方法论与核心技术
论文将 RAG 上下文构建重新定义为一个**率失真优化问题（Rate-Distortion Optimization）**，旨在最大化查询相关信息量，同时最小化 Token 长度。核心框架 **MergeRAG** 包含以下关键技术：

*   **对称合并（Symmetric Merging）**：
    *   **目标**：恢复弱信号。
    *   **流程**：迭代地选择相关性得分最低的两个数据块，利用 LLM 作为融合算子，根据查询意图将它们合成一个连贯且精简的新块，直到达到 Token 预算。
*   **非对称合并（Asymmetric Merging）**：
    *   **目标**：消除冗余。
    *   **流程**：基于信息论，利用条件负对数似然（NLL）作为经验估计器，为低分数据块寻找语义重合度最高的“锚点块（Anchor Chunk）”。LLM 将低分块中的增量信息注入锚点块，剔除重复内容。
*   **层级并行合并（Hierarchical Parallel Merging）**：
    *   **优化**：为了解决串行合并带来的高延迟和递归生成导致的语义漂移（幻觉累积），提出树状结构的并行合并策略。将合并深度从线性降低到对数级别（$\log N$），显著提升推理速度并保持语义忠实度。

#### 6. 主要结论与发现
*   **性能飞跃**：MergeRAG 在多跳推理任务上表现尤为突出，MuSiQue 数据集的 F1 分数比最强基准提升了 12.1 点。
*   **高信息密度**：在极端的 Token 限制下（如 $k=1$），MergeRAG 的表现远超所有基准，证明其能将多源证据有效压缩至极小窗口。

*   **策略差异**：非对称合并更擅长处理冗余严重的多跳任务，而对称合并在需要高召回率的长尾场景（如 QASPER）中更具优势。

#### 7. 优点与亮点
*   **范式创新**：从简单的“过滤”转向“动态合成”，更符合复杂问答对信息整合的需求。
*   **理论支撑**：引入信息瓶颈（IB）原则和条件熵概念，为上下文合并提供了严谨的数学解释。
*   **工程实用性**：层级并行设计解决了 LLM 级联处理的延迟痛点，使其具备实际部署的潜力。

#### 8. 不足与局限
*   **计算开销**：虽然有并行优化，但相比于直接 Top-k 过滤，MergeRAG 仍需额外的 LLM 融合推理步骤，增加了首字延迟（TTFT）。
*   **NLL 计算依赖**：非对称合并依赖于计算 NLL，这要求推理框架能够高效暴露模型内部的概率输出，增加了系统实现的复杂性。
*   **幻觉风险**：尽管层级策略有所缓解，但多次 LLM 重新生成（Regeneration）理论上仍存在丢失原始文本精确事实或引入幻觉的风险。
