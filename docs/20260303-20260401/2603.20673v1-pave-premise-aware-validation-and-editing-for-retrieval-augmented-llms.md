## 摘要
检索增强型语言模型虽然能够检索到相关证据，但在明确检查检索到的上下文是否支持结论之前，往往仍会直接给出答案。我们提出了 PAVE（基于前提的答案验证与编辑），这是一种用于证据驱动型问答的推理阶段验证层。PAVE 将检索到的上下文分解为以问题为条件的原子事实，起草答案，评分该草案受提取前提支持的程度，并在最终确定前修正支持度较低的输出。由此产生的追踪过程使得答案的确定在显式前提、支持评分和修正决策层面具有可审计性。在固定检索器和骨干模型的受控消融实验中，PAVE 在两种证据驱动型问答设置下均优于更简单的检索后基准模型，在基于片段落地的基准测试中，最大增益达到了 32.7 个准确度百分点。我们认为这些发现是概念验证性的证据，表明显式前提提取加上支持门控修正可以增强检索增强型大语言模型系统中的证据一致性。

## Abstract
Retrieval-augmented language models can retrieve relevant evidence yet still commit to answers before explicitly checking whether the retrieved context supports the conclusion. We present PAVE (Premise-Grounded Answer Validation and Editing), an inference-time validation layer for evidence-grounded question answering. PAVE decomposes retrieved context into question-conditioned atomic facts, drafts an answer, scores how well that draft is supported by the extracted premises, and revises low-support outputs before finalization. The resulting trace makes answer commitment auditable at the level of explicit premises, support scores, and revision decisions. In controlled ablations with a fixed retriever and backbone, PAVE outperforms simpler post-retrieval baselines in two evidence-grounded QA settings, with the largest gain reaching 32.7 accuracy points on a span-grounded benchmark. We view these findings as proof-of-concept evidence that explicit premise extraction plus support-gated revision can strengthen evidence-grounded consistency in retrieval-augmented LLM systems.

### 1. 核心问题与研究动机
*   **核心问题**：现有的检索增强生成（RAG）系统存在“过早承诺”的问题。即模型在获取检索内容后，往往直接生成最终答案，而没有显式地校验检索到的证据是否真的支持该结论。
*   **研究背景**：在医疗、法律等高风险领域，即使答案听起来很通顺，如果它与检索到的前提相矛盾或忽略了证据中的例外情况，也会造成严重后果。现有的 RAG 流程缺乏透明度，用户难以判断错误是源于检索失败、推理错误还是对证据的误读。

### 2. 方法论：PAVE 核心思想与流程
PAVE（Premise-Grounded Answer Validation and Editing）在检索与最终生成之间插入了一个显式的验证层，其核心流程如下：
1.  **原子分解（Atomic Decomposition）**：将检索到的原始长文本分解为一系列与问题相关的“原子事实”（Atomic Facts）。这些事实是简短、自包含的陈述，旨在保留关键条件和例外情况。
2.  **草稿生成**：基于这些原子事实，模型生成初步答案（Draft Answer）和简短的推理过程（Rationale）。
3.  **支持度评分（Support Scoring）**：使用专门的提示词评估原子事实对草稿答案的支持程度，输出一个标量评分 $s \in [0, 1]$。
4.  **阈值化修正（Thresholded Revision）**：设置修正阈值 $\tau$。如果 $s \ge \tau$，直接输出草稿；如果 $s < \tau$，则触发修正步骤，要求模型根据证据重新编辑答案。
5.  **审计追踪（Audit Trace）**：系统最终返回一个包含原子前提、评分和修正决策的完整轨迹，增强了结果的可解释性。

### 3. 实验设计
*   **数据集**：
    *   **PubMedQA**：生物医学问答，要求回答“yes/no/maybe”，重点考察对不确定性和混合证据的处理。
    *   **SQuAD**：事实型问答，要求从上下文中提取精确片段，重点考察证据的对齐精度。
*   **基准测试（Baselines）**：
    *   **Baseline RAG**：直接从检索上下文生成答案，无验证层。
    *   **RAG + Importance Weighting**：对原子事实进行重要性加权的变体。
    *   **RAG + Support Scoring**：有评分机制但直接针对原始长文本，无原子分解。
*   **实验设置**：固定检索器和 LLM 骨干网络（GPT-4o mini），以隔离验证层带来的性能增益。

### 4. 资源与算力

### 5. 实验数量与充分性
*   **实验规模**：在 PubMedQA 上测试了 1,000 个样本，在 SQuAD 上分别测试了 1,000 个（消融）和 5,000 个（规模验证）样本，并对 100 个样本进行了深入的配对诊断分析。
*   **充分性评价**：实验设计较为客观，通过固定变量（检索器和模型）来专门验证 PAVE 组件的有效性。消融实验清晰地展示了“原子分解”与“支持度评分”结合的必要性。不过，实验目前主要集中在短答案任务上，对于长文本生成的适用性尚待验证。

### 6. 主要结论与发现
*   **性能显著提升**：PAVE 在所有任务中表现最优。在 SQuAD 上比基准 RAG 提升了 **32.7 个百分点**，在 PubMedQA 上提升了 2.1 个百分点。
*   **分解的重要性**：单纯增加评分机制（不进行原子分解）提升有限。实验证明，当评分器面对精简的原子前提而非冗长的原始段落时，其判断准确度大幅提高。
*   **答案质量优化**：配对分析显示，PAVE 能显著缩短答案长度（从 71 个 token 降至 2 个），使其更符合金标准答案的简洁性，减少了无关信息的干扰。

### 7. 优点与亮点
*   **可审计性**：PAVE 提供的“审计追踪”让开发者能看清模型为何修改答案，定位是检索证据不足还是推理逻辑出错。
*   **模块化与即插即用**：作为推理层，它不需要重新训练模型，可以轻松集成到现有的 RAG 管道中。
*   **鲁棒性**：通过显式提取前提，模型能更好地捕捉到隐藏在长文中的“例外条件”，减少幻觉。

### 8. 不足与局限
*   **推理开销**：相比单次生成的 RAG，PAVE 需要 3-4 次 LLM 调用，显著增加了延迟和 API 成本。
*   **评估维度单一**：目前主要通过下游任务的准确率间接评估“原子分解”的质量，缺乏对分解本身完整性和忠实度的直接量化评估。
*   **模型依赖**：实验仅基于 GPT-4o mini，尚未验证在更弱（如 7B 参数量）或更强模型上的泛化表现。
*   **证据局限性**：支持度评分仅反映答案与“检索证据”的一致性，如果检索到的证据本身是错误或过时的，PAVE 无法纠正事实性错误。
