## 摘要
检索增强生成 (RAG) 使大语言模型能够利用外部知识，但外包 RAG 服务会引发数据所有者和用户的隐私担忧。隐私保护 RAG 系统通过执行安全 Top-$k$ 检索来解决这些问题，这通常涉及通过安全排序来识别相关文档。然而，现有系统在支持任意 $k$ 值方面面临挑战，原因包括无法更改 $k$、出现新的安全问题或在大 $k$ 值下效率下降。这是一个重大限制，因为现代长上下文模型通常在检索集较大时获得更高的准确率。我们提出了 $p^2$RAG，一种支持任意 Top-$k$ 检索的隐私保护 RAG 服务。与现有系统不同，$p^2$RAG 避免了对候选文档进行排序。相反，它使用一种交互式二分法来确定 Top-$k$ 文档集。在安全性方面，$p^2$RAG 在两个半诚实且互不串通的服务器上使用秘密共享，以保护数据所有者的数据库和用户的提示词。它通过实施限制和验证来防御恶意用户，并严格限制数据库的信息泄露。实验表明，在 $k = 16$ 至 $1024$ 的情况下，$p^2$RAG 的速度比最先进的 PRAG 快 3 至 300 倍。

## Abstract
Retrieval-Augmented Generation (RAG) enables large language models to use external knowledge, but outsourcing the RAG service raises privacy concerns for both data owners and users. Privacy-preserving RAG systems address these concerns by performing secure top-$k$ retrieval, which typically is secure sorting to identify relevant documents. However, existing systems face challenges supporting arbitrary $k$ due to their inability to change $k$, new security issues, or efficiency degradation with large $k$. This is a significant limitation because modern long-context models generally achieve higher accuracy with larger retrieval sets. We propose $p^2$RAG, a privacy-preserving RAG service that supports arbitrary top-$k$ retrieval. Unlike existing systems, $p^2$RAG avoids sorting candidate documents. Instead, it uses an interactive bisection method to determine the set of top-$k$ documents. For security, $p^2$RAG uses secret sharing on two semi-honest non-colluding servers to protect the data owner's database and the user's prompt. It enforces restrictions and verification to defend against malicious users and tightly bound the information leakage of the database. The experiments show that $p^2$RAG is 3--300$\times$ faster than the state-of-the-art PRAG for $k = 16$--$1024$.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）的发展，检索增强生成（RAG）成为解决模型幻觉和知识滞后的关键技术。然而，将 RAG 服务外包给第三方平台会引发严重的隐私风险：**数据所有者**担心其专有数据库泄露，**用户**担心其查询（Prompt）被监控。
现有的隐私保护 RAG 系统通常将问题转化为“安全 Top-$k$ 检索”，依赖于复杂的**安全排序（Secure Sorting）**算法。这些方法存在三大痛点：
*   **灵活性差**：难以支持用户动态调整 $k$ 值。
*   **效率瓶颈**：随着 $k$ 值增大（现代长文本模型通常需要更大的 $k$ 以获得更高准确率），排序开销呈指数级或大幅增长。
*   **安全隐患**：部分方案在处理大规模检索时会引入额外的泄露风险。

#### 2. 论文提出的方法论
$p^2$RAG 的核心思想是**摒弃“排序”逻辑，改用“交互式二分查找”来确定距离阈值**。

*   **核心架构**：采用两服务器模型（Two-Server Model），假设两台服务器半诚实（Honest-but-curious）且互不串通。
*   **关键技术细节**：
    *   **秘密共享（Secret Sharing）**：使用 Shamir 秘密共享将数据库和用户 Prompt 拆分为分片，存储在两台服务器上，确保单台服务器无法获取任何原始信息。
    *   **距离计算**：利用 **Beaver Triples** 优化点积运算，在加密状态下计算 Prompt 与文档向量的余弦相似度。
    *   **交互式二分法（Interactive Bisection）**：用户与服务器通过多次迭代，不断缩小相似度阈值范围。服务器利用**分布式比较函数（DCF）**统计处于当前阈值内的文档数量并返回给用户，用户根据数量调整下一步的阈值。
    *   **恶意用户防御**：为了防止恶意用户通过无限次查询探测数据库，系统严格限制了二分查找的迭代次数（$O(\log N)$）和最终返回的文档数量上限。
    *   **文本检索**：确定索引后，通过单服务器私有信息检索（PIR）技术获取原文，确保服务器不知道用户具体拿走了哪些文档。
