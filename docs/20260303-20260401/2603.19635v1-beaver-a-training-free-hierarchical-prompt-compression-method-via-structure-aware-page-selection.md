## 摘要
大语言模型（LLM）上下文窗口的指数级扩张解锁了长文档理解能力，但也带来了推理延迟和信息利用方面的严重瓶颈。现有的压缩方法通常面临高昂的训练成本，或因激进的 Token 剪枝而导致语义碎片化。在本文中，我们提出了 BEAVER，这是一个全新的无需训练的框架，它将压缩方式从线性的 Token 移除转变为结构感知的分层选择。BEAVER 通过双路径池化将变长上下文映射为稠密的页面级张量，从而最大化硬件并行性，并通过结合语义和词法双分支选择及句子平滑的混合规划器来保持语篇完整性。在四个长上下文基准测试上的广泛评估表明，BEAVER 达到了与 LongLLMLingua 等最先进（SOTA）方法相当的性能。值得注意的是，在 RULER 基准测试中，BEAVER 在基准模型性能下降的多针检索任务中保持了高保真度。在效率方面，BEAVER 在 128k 上下文中将延迟降低了 26.4 倍，为高吞吐量应用提供了一个可扩展的解决方案。我们的代码可在 https://cslikai.cn/BEAVER/ 获取。

## Abstract
The exponential expansion of context windows in LLMs has unlocked capabilities for long-document understanding but introduced severe bottlenecks in inference latency and information utilization. Existing compression methods often suffer from high training costs or semantic fragmentation due to aggressive token pruning. In this paper, we propose BEAVER, a novel training-free framework that shifts compression from linear token removal to structure-aware hierarchical selection. BEAVER maximizes hardware parallelism by mapping variable-length contexts into dense page-level tensors via dual-path pooling, and preserves discourse integrity through a hybrid planner combining semantic and lexical dual-branch selection with sentence smoothing. Extensive evaluations on four long-context benchmarks demonstrate that BEAVER achieves comparable performance to state-of-the-art (SOTA) methods like LongLLMLingua. Notably, on the RULER benchmark, BEAVER maintains high fidelity in multi-needle retrieval where baselines deteriorate. Regarding efficiency, BEAVER reduces latency by 26.4x on 128k contexts, offering a scalable solution for high-throughput applications. Our code is available at https://cslikai.cn/BEAVER/.

### 论文总结：BEAVER —— 基于结构感知页面选择的免训练分层提示词压缩方法

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）上下文窗口扩展至百万级，长文档理解能力显著提升，但也带来了两大瓶颈：
*   **计算墙（Computation Wall）**：自注意力机制的 $O(L^2)$ 复杂度导致推理延迟剧增，尤其是首字延迟（TTFT）。
*   **信息利用率下降**：模型常出现“迷失中段”（Lost in the Middle）现象，无法有效利用长文本中的关键信息。
现有的提示词压缩方法存在局限：**监督学习类**方法训练成本高且跨模型泛化差；**无监督 Token 剪枝类**方法则会破坏文本的语义和语法连贯性，导致上下文碎片化。BEAVER 旨在通过一种**免训练、硬件友好且结构感知**的分层压缩方案来解决这些问题。

#### 2. 方法论：核心思想与技术细节
BEAVER 将压缩范式从“线性 Token 剪枝”转变为“结构化页面选择”，主要包含三个组件：
*   **Segmenter（分段器）**：利用自然分隔符（如换行符）将长文本切分为逻辑段落，并采用贪婪分页策略将其映射为固定大小的二维页面张量（Page Tensor）。这种设计便于 GPU 进行并行张量运算。
*   **PageEncoder（页面编码器）**：
    *   **双路径池化**：结合“加权平均池化”（捕获全局语义）和“最大池化”（捕获局部显著特征，如稀疏关键词）。
    *   **上下文 ITF 权重**：引入逆词频（In-Context ITF）机制，自动降低高频但无意义 Token 的权重，抑制语义坍缩。
*   **QueryPlanner（查询规划器）**：
    *   **混合匹配**：结合语义分支（加权余弦相似度）和词法分支（Token 精确重叠评分）。
    *   **结构化先验**：引入**Anchor（锚点页，保留开头元数据）**、**Flow（流动页，保留查询前的滑动窗口）**和**Flash（闪速页，根据评分选择的关键证据）**。
*   **Sentence Smoothing（句子平滑）**：将选中的页面映射回 Token 跨度时，向外扩展至最近的句子边界，确保语法完整性。

#### 3. 实验设计
*   **数据集/场景**：
    *   **LongBench & ZeroSCROLLS**：评估通用的长文本理解、摘要和多文档问答能力。
    *   **RULER**：高难度合成基准，测试多针检索（Multi-needle Retrieval）和变量跟踪。
    *   **L-Eval**：测试在法律、金融、小说等特定领域的域外泛化能力。
*   **对比方法**：
    *   无监督统计类：Selective-Context, LongLLMLingua。
    *   监督/专用学习类：LLMLingua, LLMLingua-2, SBERT, OpenAI Embeddings。
*   **后端模型**：主要使用 `gpt-3.5-turbo-instruct`，并在 Qwen3 系列（0.6B-32B）上验证了扩展性。

#### 4. 资源与算力
*   **硬件**：所有延迟和吞吐量指标均在 **NVIDIA A100 (80GB) GPU** 上测量。
*   **训练时长**：由于 BEAVER 是**免训练（Training-free）**框架，因此**无训练算力消耗**，表现出极高的部署经济性。

#### 5. 实验数量与充分性
*   **实验规模**：在 4 个主流基准测试、数十个子任务上进行了评估，涵盖了从 16k 到 128k 的不同上下文长度。
*   **消融实验**：非常充分。作者针对页面大小（M）、池化方式、ITF 评分、语义/词法分支权重、结构化先验策略以及句子平滑机制分别进行了消融分析。
*   **客观性**：通过跨模型规模（0.6B 到 32B）的鲁棒性测试，证明了该方法不依赖于特定模型的参数量，实验设计较为客观公平。

#### 6. 主要结论与发现
*   **性能卓越**：在 LongBench 和 L-Eval 上达到或超过了 SOTA（如 LLMLingua-2）。在 RULER 多针检索任务中，BEAVER 几乎翻倍领先于基准方法（83.7 vs 47.9）。
*   **极高效率**：在 128k 上下文下，BEAVER 比 LongLLMLingua 快 **26.4 倍**，且延迟随长度增长的曲线非常平缓。
*   **小模型友好**：在 0.6B 模型上，BEAVER 能保持 98% 的原始性能，而其他压缩方法在小模型上往往失效。
*   **结构重要性**：实验证明，保留 Anchor 和 Flow 页面对于维持模型的指令遵循能力和局部逻辑至关重要。

#### 7. 优点
*   **免训练（Plug-and-Play）**：无需更新参数，可直接集成到任何 LLM 推理流水线中。
*   **硬件并行优化**：通过 2D 页面张量化处理，规避了 Token 级处理的串行瓶颈。
*   **语义保真度高**：句子平滑和结构先验有效解决了传统压缩导致的文本“支离破碎”问题。
*   **鲁棒性强**：对罕见实体（如代码标识符、特定数值）的保留能力远强于基于概率的剪枝方法。

#### 8. 不足与局限
*   **粒度限制**：页面级选择虽然保护了结构，但相比 Token 级剪枝，在选中的页面内部可能仍存在少量冗余。
*   **多步推理挑战**：在证据与查询之间缺乏直接语义/词法重叠的深层多步推理场景中，检索机制可能面临挑战。
*   **超参数依赖**：作为免训练方法，其性能依赖于预设的超参数（如页面大小、融合权重），在极端不同的领域可能需要手动微调。
