## 摘要
生产环境中的 AI 智能体经常接收到具有高度重复性的用户特定查询，其中高达 47% 的查询与之前的交互在语义上相似，但每个查询通常都以相同的计算成本进行处理。我们认为，可以通过对话记忆利用这种冗余，将重复从成本负担转化为效率优势。我们提出了一种记忆增强的推理框架，其中轻量级的 8B 参数模型利用检索到的对话上下文，通过低成本推理路径回答所有查询。在没有任何额外训练或标注数据的情况下，该方法实现了 30.5% 的 F1 分数，恢复了全上下文 235B 模型 69% 的性能，同时将有效成本降低了 96%。值得注意的是，没有记忆的 235B 模型（13.7% F1）甚至不如独立的 8B 模型（15.4% F1），这表明对于用户特定的查询，获取相关知识的重要性超过了模型规模。我们进一步分析了路由和置信度的作用。在实际的置信度阈值下，仅靠路由就已经将 96% 的查询导向小模型，但由于高置信度幻觉，其准确率较低（13.0% F1）。记忆并没有实质性地改变路由决策；相反，它通过将响应锚定在检索到的用户特定信息中来提高正确性。随着对话记忆随时间积累，对重复出现话题的覆盖范围不断扩大，从而进一步缩小了性能差距。我们在 152 个 LoCoMo 问题（Qwen3-8B/235B）和 500 个 LongMemEval 问题上进行了评估。引入混合检索（BM25 + 余弦相似度）使性能额外提升了 +7.7 F1，证明了检索质量直接增强了端到端系统的性能。总之，我们的结果强调，在持久化 AI 智能体中，记忆而非模型规模是准确性和效率的主要驱动因素。

## Abstract
Production AI agents frequently receive user-specific queries that are highly repetitive, with up to 47\% being semantically similar to prior interactions, yet each query is typically processed with the same computational cost. We argue that this redundancy can be exploited through conversational memory, transforming repetition from a cost burden into an efficiency advantage. We propose a memory-augmented inference framework in which a lightweight 8B-parameter model leverages retrieved conversational context to answer all queries via a low-cost inference path. Without any additional training or labeled data, this approach achieves 30.5\% F1, recovering 69\% of the performance of a full-context 235B model while reducing effective cost by 96\%. Notably, a 235B model without memory (13.7\% F1) underperforms even the standalone 8B model (15.4\% F1), indicating that for user-specific queries, access to relevant knowledge outweighs model scale. We further analyze the role of routing and confidence. At practical confidence thresholds, routing alone already directs 96\% of queries to the small model, but yields poor accuracy (13.0\% F1) due to confident hallucinations. Memory does not substantially alter routing decisions; instead, it improves correctness by grounding responses in retrieved user-specific information. As conversational memory accumulates over time, coverage of recurring topics increases, further narrowing the performance gap. We evaluate on 152 LoCoMo questions (Qwen3-8B/235B) and 500 LongMemEval questions. Incorporating hybrid retrieval (BM25 + cosine similarity) improves performance by an additional +7.7 F1, demonstrating that retrieval quality directly enhances end-to-end system performance. Overall, our results highlight that memory, rather than model size, is the primary driver of accuracy and efficiency in persistent AI agents.

### 1. 研究动机与核心问题
*   **核心问题**：在生产环境中，AI 智能体（如个人助理、客服）面临大量**重复性或语义相似**的用户查询（占比高达 47%）。目前的处理方式通常是对每个查询都投入相同的计算成本，且小模型因缺乏长期记忆无法回答特定用户事实，而大模型成本极高。
*   **研究动机**：探索如何利用“对话记忆”将重复性查询从成本负担转化为效率优势。论文旨在证明：对于个性化任务，**获取相关知识（记忆）比增加模型规模（参数量）更重要**。

### 2. 方法论：记忆增强路由框架
该框架包含三个核心组件，形成一个“检索-注入-探测-路由”的闭环：
*   **跨模型记忆注入（Cross-Model Memory Injection）**：
    *   将所有交互（无论是来自 8B 还是 235B 模型）以“问答对”形式存入向量数据库（Milvus）。
    *   新查询进入时，检索 top-k 相关记忆并作为系统上下文注入。这种方式实现了“大模型产生的知识可供后续小模型复用”的摊销效应。
*   **基于置信度的路由（Confidence-Based Routing）**：
    *   **探测（Probe）**：先让 8B 小模型处理带记忆的提示词，并输出 token 级别的对数概率（log-probabilities）。
    *   **评分公式**：计算平均对数概率 $\bar{\ell}$，并归一化为置信度分数 $c = (\bar{\ell} - \ell_{min}) / |\ell_{min}|$（其中 $\ell_{min}$ 为底数，常设为 -3）。
    *   **决策**：若 $c \geq \tau$（阈值），接受小模型结果；否则升级（Escalate）至 235B 大模型。
*   **混合检索（Hybrid Retrieval）**：
    *   结合 **BM25（关键词匹配）** 和 **余弦相似度（语义向量）**。BM25 负责捕捉特定实体（如人名、日期），向量检索负责捕捉语义意图。

### 3. 实验设计
*   **数据集**：
    *   **LoCoMo**：长对话记忆基准，包含 152 个关于特定对话事实的问题（单步、多步、开放、时间推理）。
    *   **LongMemEval**：包含 500 个问题，每个问题背后有约 48 个会话（约 12 万 token）的“大海捞针”式背景。
*   **对比方法（2x2 因子实验）**：
    *   **无记忆 vs 有记忆**。
    *   **无路由（固定模型） vs 有路由（8B 探测 + 235B 升级）**。
    *   **基准线**：冷启动 8B、冷启动 235B、全上下文 235B（性能上限）、以及 Mem0 等已发表的 SOTA 方法。

### 4. 资源与算力
*   **硬件**：使用单台 **AMD Instinct MI300X GPU** (192 GB HBM3)。
*   **模型**：同时运行 Qwen3-VL-8B-Instruct 和 Qwen3-VL-235B-A22B-Instruct-AWQ（量化版）。

### 5. 实验数量与充分性
*   **实验规模**：
    *   在 LoCoMo 上进行了 6 种主要条件的对比。
    *   在 LongMemEval 上进行了 500 个问题的检索消融实验。
    *   进行了针对不同问题类型（单步 vs 多步 vs 时间）的细分分析。
*   **充分性评价**：实验设计较为充分且客观。通过 2x2 因子分析清晰地隔离了“路由”和“记忆”各自的作用，揭示了两者之间的非直观交互。使用了标准 F1 和有效成本（EffCost）指标，对比具有说服力。

### 6. 主要结论与发现
*   **知识胜过规模**：带记忆的 8B 模型（30.5% F1）性能远超**不带记忆的 235B 模型（13.7% F1）**。
*   **路由的“陷阱”**：在没有记忆的情况下，小模型往往会以高置信度产生幻觉（Confidently Wrong）。路由机制本身无法识别这种错误，只有注入记忆后，高置信度才等同于正确性。
*   **成本效益**：该策略在保持 100% 查询留在小模型路径上的同时，恢复了大型模型 69% 的性能，**有效推理成本降低了 96%**。
*   **混合检索的必要性**：在处理知识更新和特定实体查询时，BM25 的加入使 F1 提升了 7.7 个百分点。

### 7. 优点与亮点
*   **协同效应发现**：首次指出记忆不是路由的“优化项”，而是路由在个性化场景下的“正确性前提”。
*   **零成本部署**：无需标注数据或训练分类器，仅利用模型自身的 logprobs 信号，极易在现有推理框架（如 vLLM）中实现。
*   **跨模型知识摊销**：提出了一种优雅的机制，让昂贵的大模型在第一次遇到新知识时“付费”，后续重复查询由廉价小模型通过记忆“免费”复用。

### 8. 不足与局限
*   **时间推理薄弱**：实验显示记忆注入反而降低了时间推理问题的表现（-3.8 F1），说明简单的文本记忆无法替代结构化的时间逻辑处理。
*   **冷启动轨迹缺失**：实验主要评估了记忆已满的“稳态”，未详细展示记忆从零积累到产生协同效应的具体速度。
*   **置信度校准风险**：完全依赖 logprobs 可能受模型过度自信（Overconfidence）的影响，在不同模型族群间的泛化性有待验证。
*   **检索瓶颈**：性能高度依赖于 top-k 检索的质量，对于需要跨越大量记忆片段进行综合推理的任务，表现仍有局限。
