## 摘要
尽管视觉-语言-动作（VLA）模型作为通用机器人控制器具有广阔前景，但由于缺乏长期记忆、因果故障归因和动态干预能力，它们在分布外（OOD）任务中应对感知噪声和环境变化的鲁棒性仍然受到根本限制。为了解决这一问题，我们提出了 SOMA，这是一个策略编排与记忆增强系统，它在无需参数微调的情况下，升级冻结的 VLA 策略以实现鲁棒的上下文自适应。具体而言，SOMA 通过一个在线流水线运行，该流水线包括对比式双记忆检索增强生成（RAG）、归因驱动的大语言模型（LLM）编排器以及可扩展的模型上下文协议（MCP）干预，同时一个离线记忆整合模块不断将执行轨迹提炼为可靠的先验知识。在 LIBERO-PRO 和我们提出的 LIBERO-SOMA 基准测试上，针对三种骨干模型（pi0、pi0.5 和 SmolVLA）的实验评估表明，SOMA 实现了 56.6% 的平均绝对成功率提升。这包括在长程任务链中 89.1% 的显著绝对提升。项目页面和源代码可在以下网址获取：https://github.com/LZY-1021/SOMA。

## Abstract
Despite the promise of Vision-Language-Action (VLA) models as generalist robotic controllers, their robustness against perceptual noise and environmental variations in out-of-distribution (OOD) tasks remains fundamentally limited by the absence of long-term memory, causal failure attribution, and dynamic intervention capability. To address this, we propose SOMA, a Strategic Orchestration and Memory-Augmented System that upgrades frozen VLA policies for robust in-context adaptation without parameter fine-tuning. Specifically, SOMA operates through an online pipeline of contrastive Dual-Memory Retrieval-Augmented Generation (RAG), an Attribution-Driven Large-Language-Model (LLM) Orchestrator, and extensible Model Context Protocol (MCP) interventions, while an offline Memory Consolidation module continuously distills the execution traces into reliable priors. Experimental evaluations across three backbone models (pi0, pi0.5, and SmolVLA) on LIBERO-PRO and our proposed LIBERO-SOMA benchmarks demonstrate that SOMA achieves an average absolute success rate gain of 56.6%. This includes a significant absolute improvement of 89.1% in long-horizon task chaining. Project page and source code are available at: https://github.com/LZY-1021/SOMA.

#### 1. 核心问题与整体含义（研究动机和背景）
视觉-语言-动作（VLA）模型虽在机器人控制领域展现出潜力，但在面对**分布外（OOD）**任务时表现脆弱。核心痛点在于：
*   **缺乏长期记忆**：模型通常是无状态的，无法从过去的失败或成功中学习。
*   **缺乏因果归因**：无法诊断失败是由感知噪声（如光照、遮挡）还是规划错误引起的。
*   **缺乏动态干预**：难以在执行过程中主动修正感知输入或调整任务上下文。
现有的微调方法成本高且难以覆盖所有噪声场景，而传统的 RAG 方法往往存在“成功偏差”，忽略了失败案例的诊断价值。

#### 2. 方法论：核心思想与关键技术
SOMA 提出了一种**无需微调参数**、通过**上下文自适应（In-Context Adaptation）**升级冻结 VLA 模型的框架。其核心由在线执行和离线整合两个流水线组成：

*   **双存储检索增强生成（Dual-Memory RAG）**：
    *   构建一个包含成功（$M_{succ}$）和失败（$M_{fail}$）案例的存储库。
    *   利用 CLIP 编码器根据当前任务的视觉和指令特征，从库中检索出最相关的正向引导和负向警告。
*   **归因驱动的 LLM 编排器（LLM Orchestrator）**：
    *   使用大模型（如 Qwen3-VL-32B）作为大脑，对比当前状态与检索到的存储案例，诊断失败原因。
    *   根据诊断结果，从工具库中选择并参数化一系列干预策略。
*   **MCP 干预工具链（Model Context Protocol）**：
    *   **Paint-to-Action**：通过视觉覆盖/纹理替换解决视觉领域偏移。
    *   **Eraser**：利用修复技术移除环境中的干扰物（Distractors）。
    *   **Prompt-Refiner**：将模糊或口语化的指令重写为标准指令。
    *   **Chaining-Step**：将长程任务分解为短程子任务。
    *   **Encore**：在检测到执行停滞时进行关键状态回滚和重试。
*   **离线存储整合（Memory Consolidation）**：
    *   异步处理执行轨迹，通过跨任务差异分析修正存储库中的归因描述和干预计划，实现自我进化。

*   **架构解耦**：将感知修正、逻辑推理与底层控制分离，使系统具有极强的可扩展性。

*   **通用性**：作为一种“包装器”方案，可以适配任何现有的 VLA 模型。

#### 8. 不足与局限
*   **推理延迟**：引入 LLM 编排和视觉处理工具（如 SAM3、修复算法）会显著增加单步决策的延迟，可能影响实时性要求极高的任务。

*   **工具库限制**：目前的 MCP 工具主要针对视觉和语言噪声，对于复杂的物理接触力反馈或极端的动力学变化覆盖有限。
*   **真实环境验证**：实验主要在模拟器（LIBERO）中完成，虽然模拟度高，但在光影更复杂的真实物理世界中的表现仍需进一步验证。
