## 摘要
语言模型难以泛化到预训练上下文长度之外，这限制了长程推理和检索能力。在长上下文数据上进行持续预训练会有所帮助，但由于注意力机制（Attention）的二次方缩放特性，其成本非常高昂。我们观察到，大多数 token 并不需要对整个序列进行（全局）注意力计算，而是可以依赖局部上下文。基于此，我们提出了 L2A（Learning To Attend），这是一种通过决定何时调用全局注意力来实现条件化（逐 token）长程记忆访问的层。我们在 Qwen 2.5 和 Qwen 3 模型上评估了 L2A，将其有效上下文长度从 32K 扩展到了 128K token。L2A 在跳过约 80% token 的全局注意力计算的同时，其性能与标准长上下文训练的差距在 3% 以内，优于之前的基准模型。我们还设计了定制的 Triton 内核，以便在 GPU 上高效实现这种逐 token 的条件注意力，与 FlashAttention 相比，训练吞吐量和首字延迟（time-to-first-token）提升了高达约 2 倍。此外，L2A 支持对高度稀疏的全局注意力层进行训练后剪枝，在性能损失微乎其微的情况下，将 KV 缓存内存占用减少了高达 50%。

## Abstract
Language models struggle to generalize beyond pretraining context lengths, limiting long-horizon reasoning and retrieval. Continued pretraining on long-context data can help but is expensive due to the quadratic scaling of Attention. We observe that most tokens do not require (Global) Attention over the entire sequence and can rely on local context. Based on this, we propose L2A (Learning To Attend), a layer that enables conditional (token-wise) long-range memory access by deciding when to invoke global attention. We evaluate L2A on Qwen 2.5 and Qwen 3 models, extending their effective context length from 32K to 128K tokens. L2A matches the performance of standard long-context training to within 3% while skipping Global Attention for $\sim$80% of tokens, outperforming prior baselines. We also design custom Triton kernels to efficiently implement this token-wise conditional Attention on GPUs, achieving up to $\sim$2x improvements in training throughput and time-to-first-token over FlashAttention. Moreover, L2A enables post-training pruning of highly sparse Global Attention layers, reducing KV cache memory by up to 50% with negligible performance loss.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：标准 Transformer 的全局注意力机制（Global Attention）具有随序列长度增加而呈二次方增长的计算复杂度。在扩展大语言模型（LLM）的上下文长度（如从 32K 到 128K）时，这种计算开销变得极其昂贵。
*   **研究动机**：作者观察到，并非所有 token 都需要查看整个历史信息。例如，处理“因此季度收入是”这类句子主要依赖近期上下文，而“如第2节所述”则需要检索数千个 token 之前的远期信息。
*   **整体含义**：论文试图回答“模型能否逐 token 地自主决定何时访问长程记忆，而不仅仅是决定检索什么”。通过这种**条件性记忆访问**，可以在不牺牲长文本处理能力的前提下，大幅提升训练和推理效率。

### 2. 方法论：L2A (Learning To Attend)
*   **核心思想**：将注意力层改造为由“局部注意力（Local Attention）”、“路由决策器（Router）”和“全局注意力（Global Attention）”组成的复合结构。
*   **关键技术细节**：
    *   **局部处理**：所有 token 首先通过滑动窗口注意力（SWA，窗口大小通常为 4K）处理局部信息。
    *   **路由决策**：利用局部表示，通过一个轻量级线性层+Sigmoid 激活函数生成决策值 $d_t$。若 $d_t=1$，则触发昂贵的全局注意力；若 $d_t=0$，则跳过。
    *   **训练策略**：使用直通估计器（STE）解决离散决策的不可导问题。为了防止路由塌陷（即模型为了省事全部选择跳过全局注意力），引入了随机调用机制（以 0.1 的概率强制所有 token 进行全局计算）和稀疏正则化项。
    *   **硬件加速**：开发了定制的 **Triton 内核**，通过重新排列活跃查询（Active Queries）并利用 FlashAttention-2 的分块思想，实现了硬件层面的真实加速。
    *   **层剪枝**：发现约 50% 的层在训练后几乎不调用全局注意力，因此可以将其全局 KV 缓存剪枝，仅保留局部缓存。

### 3. 实验设计
*   **基座模型**：Qwen 2.5 (1.5B, 7B) 和 Qwen 3 (8B)，预训练长度均为 32K。
*   **评估场景与数据集**：
    *   **长上下文基准**：HELMET（包含 JSON 检索、多键值检索等）、BabiLong（事实链推理）、MRCR（多轮指代消解）。
    *   **任务类型**：合成检索（Recall）、RAG、长文本 ICL、带引用的生成、段落重排序等。
    *   **短上下文基准**：MMLU、ARC、BoolQ 等，用于验证长文本训练是否损害了基础能力。
*   **对比方法**：
    *   **训练基准**：CLP（全量长上下文持续预训练，作为上限）。
    *   **稀疏注意力**：Native Sparse Attention (NSA)、$S^2$-Attn。
    *   **免训练方法**：SelfExtend、Dual Chunk Attention (DCA)。
    *   **推理加速**：FlexPrefill。

### 4. 资源与算力
*   **硬件型号**：使用 **NVIDIA H200 GPU**。
*   **训练规模**：
    *   Qwen 2.5 7B 和 Qwen 3 8B 使用 6M token 的 batch size。
    *   Qwen 2.5 1.5B 使用 4M token 的 batch size。
    *   训练总量：7B/8B 模型训练了 25B token，1.5B 模型训练了 16.7B token。
*   **效率提升**：相比标准 CLP 训练，L2A 实现了约 **2 倍** 的训练吞吐量提升。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 1.5B 到 8B 不同参数规模的模型，以及从 8K 到 128K 的多种上下文长度。
*   **消融实验**：非常充分。作者对比了“无上下文随机路由（Context-free Sparse）”、不同滑动窗口大小（0 到 4K）、不同正则化系数 $\lambda$ 以及测试时调节 Sigmoid 阈值对性能的影响。
*   **客观性**：通过在多个独立基准测试上运行，并与当前最先进的稀疏注意力方案（如 NSA）进行对比，证明了 L2A 在保持 80% 稀疏度的同时，性能最接近全量注意力模型。

### 6. 主要结论与发现
*   **性能保持**：L2A 在 128K 长度下的表现与全量训练（CLP）的差距在 **1.5% - 3%** 以内，显著优于其他稀疏或免训练方案。
*   **高效稀疏**：模型能够自动学习到任务的复杂性，在简单的检索任务中稀疏度极高，而在复杂的 ICL 任务中则更频繁地调用全局注意力。
*   **内存优化**：通过层剪枝，可以在几乎不损失性能的情况下减少 **50% 的 KV 缓存** 占用。
*   **基础能力**：长上下文扩展训练没有损害模型在短文本任务（如 MMLU）上的表现。

### 7. 优点
*   **自适应性**：不同于固定稀疏模式的方法，L2A 实现了“按需分配计算资源”，能够根据输入内容动态调整。
*   **软硬件协同**：不仅有算法创新，还提供了高效的 Triton 内核，解决了“算法稀疏但不减耗”的常见痛点。
*   **部署友好**：支持训练后剪枝，直接解决了长文本推理中最大的瓶颈——KV 缓存内存占用。

### 8. 不足与局限
*   **KV 缓存仍需全量存储（剪枝前）**：虽然计算减少了，但在未进行层剪枝的情况下，为了支持未来 token 可能的全局访问，仍需存储所有历史 token 的 KV 值。
*   **训练稳定性**：路由机制需要精细的超参数调节（如随机调用概率和正则化系数），否则容易出现路由塌陷。
*   **架构限制**：目前主要在 Transformer 架构上验证，虽然文中提到了 SSM（状态空间模型）作为局部模块的可能性，但尚未深入探索。
*   **扩展上限**：实验最高验证到 128K，对于百万级（1M+）超长上下文的极端情况，其路由器的学习难度和准确性尚待验证。
