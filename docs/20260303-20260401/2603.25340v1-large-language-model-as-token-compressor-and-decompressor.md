## 摘要
在本文中，我们提出了一个新颖的见解：现成的大语言模型（LLM）可以作为出色的 Token 压缩器和解压器。为了证明这一点，我们设计了一个自表达自编码学习框架，通过微调预训练 LLM，将长文本转换为由离散、变长潜码（称为 Z-token）组成的紧凑内部语言，并能从中精确重建原始文本。所得表示具有内容自适应性：通过轻量级的基于 LoRA 的适配器头，语义密集的片段会分配到更多 Z-token，而冗余或可预测的区域则会被大幅压缩。实验表明，我们的方法在 Wikipedia、CNN/DailyMail、HotpotQA 和 Qulac 风格的长查询数据集上实现了高达 18 倍的 Token 缩减，同时保持了重建保真度和下游任务性能。这一简单而有效的设计支持包括提示词压缩和直接在 Z-token 空间进行自回归生成在内的多种应用，为实现 Token 高效的长上下文推理提供了一条潜在路径。

## Abstract
In this paper, we establish the novel insight that an off-the-shelf LLM can function as an excellent token compressor and decompressor. To demonstrate, we design a self-expressive autoencoding learning framework fine-tunes a pretrained LLM to translate long texts into a compact internal language of discrete, variable-length latent codes, termed Z-tokens, and to reconstruct the original text exactly from them. The resulting representation is content-adaptive: semantically dense segments receive more Z-tokens, while redundant or predictable regions are aggressively compressed, via lightweight LoRA-based adapter heads. Empirically, our method achieves up to 18 times token reduction on Wikipedia, CNN/DailyMail, HotpotQA, and Qulac-style long-query datasets, while preserving reconstruction fidelity and downstream performance. This simple yet effective design supports applications including prompt compression and autoregressive generation directly in the Z-token space, offering a potential pathway toward token-efficient long-context reasoning.

*   **核心见解**：LLM 本身具备强大的语义抽象能力。论文提出将 LLM 视为一种“翻译器”，将自然语言翻译成其内部的紧凑离散语言（Z-tokens），从而实现高效的压缩与解压。

### 2. 方法论
*   **核心思想**：构建一个自表达的自动编码框架，利用 LoRA 适配器微调预训练 LLM，使其能够将长文本编码为离散、变长的潜码（Z-tokens），并能从中还原原始文本。
*   **关键技术细节**：
    *   **Z-token 空间**：扩展原始词表，定义专门的 Z-vocabulary。Z-tokens 是离散的符号，具有变长特性，能根据内容密度动态分配。
    *   **自回归压缩**：压缩器以自回归方式生成 Z-tokens，利用 Gumbel-Softmax 采样解决离散符号的梯度回传问题，并配合 Pass-through Estimator 保持训练稳定性。
    *   **双路径解压**：
        1.  **作为重构器**：将 Z-tokens 还原为原始表面文本。
        2.  **作为推理器**：直接在 Z-token 空间进行下游任务（如 QA、摘要）的推理，无需还原。
    *   **滑动窗口压缩**：针对超长文本，采用带重叠的滑动窗口处理，并通过一致性损失（Consistency Constraint）确保段落间的语义连贯。
    *   **计划采样（Schedule Sampling）**：在训练解压器时，逐渐从教师强制（Teacher Forcing）过渡到自由解码，以缓解自回归生成中的曝光偏差。

### 6. 主要结论与发现
*   **极高压缩率**：在保持高性能的同时，实现了最高 **18 倍** 的 Token 缩减。
*   **推理加速**：在摘要等任务中，推理速度提升了 **2.1x 至 4.6x**，并显著降低了显存占用。
*   **语义保真度**：在 4 倍压缩下，重构的 BLEU-4 分数接近完美（>99），且在下游 QA 任务中表现优于现有的固定长度压缩方法。
*   **Z-token 的多义性**：发现 Z-token 并非简单的 1:1 映射，而是具有上下文相关性的抽象语义单元，同一个 Z-token 在不同语境下可解压出不同的表达。

### 7. 优点
*   **内容自适应性**：变长压缩机制允许模型为复杂语义分配更多 Token，为冗余信息分配更少 Token，比固定比例压缩更智能。
*   **离散且可解释**：Z-tokens 作为离散符号，比连续向量更易于存储、检索，且通过解压分析展现出了一定的语义规律性。
*   **端到端集成**：将压缩与生成统一在同一个 LLM 架构下，保持了模型能力的完整性。

### 8. 不足与局限
*   **长程依赖挑战**：虽然引入了滑动窗口，但在处理极长文档时，跨窗口的全局信息协调机制仍有待进一步优化。
*   **任务感知局限**：目前的压缩过程主要是通用的语义蒸馏，尚未实现针对特定下游任务（如只保留事实性信息）的定制化压缩。
*   **模型规模覆盖**：实验主要集中在 4B 以下的中小规模模型，在超大规模模型（如 70B+）上的表现和成本效益尚需验证。
