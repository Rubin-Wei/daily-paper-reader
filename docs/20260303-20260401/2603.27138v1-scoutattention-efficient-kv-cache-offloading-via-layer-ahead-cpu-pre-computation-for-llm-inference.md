## 摘要
大语言模型在长上下文推理过程中面临严重的 GPU 显存容量限制，其中 KV 缓存的内存消耗严重限制了解码批处理大小。尽管现有研究探索了将 KV 缓存卸载到 DRAM，但这些方法要么需要频繁的 GPU-CPU 数据传输，要么对 CPU 计算提出了极高要求，导致系统在等待 I/O 操作或 CPU 处理完成时 GPU 利用率低下。我们提出了 ScoutAttention，这是一种新型 KV 缓存卸载框架，通过 GPU-CPU 协同注意力计算来加速大语言模型推理。为了防止 CPU 计算成为系统瓶颈，ScoutAttention 引入了 GPU-CPU 协同的分块稀疏注意力，显著降低了 CPU 负载。与传统的并行计算方法不同，我们的框架采用了一种新颖的层前 CPU 预计算算法，使 CPU 能够提前一层启动注意力计算，并辅以异步周期性召回机制，以保持极低的 CPU 计算负载。实验结果表明，ScoutAttention 在保持精度与基准线差距在 2.4% 以内的同时，与现有的卸载方法相比实现了 2.1 倍的加速。

## Abstract
Large language models encounter critical GPU memory capacity constraints during long-context inference, where KV cache memory consumption severely limits decode batch sizes. While existing research has explored offloading KV cache to DRAM, these approaches either demand frequent GPU-CPU data transfers or impose extensive CPU computation requirements, resulting in poor GPU utilization as the system waits for I/O operations or CPU processing to complete.   We propose ScoutAttention, a novel KV cache offloading framework that accelerates LLM inference through collaborative GPU-CPU attention computation. To prevent CPU computation from bottlenecking the system, ScoutAttention introduces GPU-CPU collaborative block-wise sparse attention that significantly reduces CPU load. Unlike conventional parallel computing approaches, our framework features a novel layer-ahead CPU pre-computation algorithm, enabling the CPU to initiate attention computation one layer in advance, complemented by asynchronous periodic recall mechanisms to maintain minimal CPU compute load. Experimental results demonstrate that ScoutAttention maintains accuracy within 2.4% of baseline while achieving 2.1x speedup compared to existing offloading methods.

### 1. 核心问题与整体含义（研究动机和背景）
在长上下文（Long-context）大语言模型推理中，**KV 缓存（KV Cache）**的显存占用是核心瓶颈。随着上下文增长，GPU 显存难以容纳庞大的 KV 缓存，限制了批处理大小（Batch Size）和吞吐量。
*   **现有方案的缺陷：** 
    *   **基于召回的方法（如 InfiniGen）：** 将 KV 缓存卸载到内存，需要时再通过 PCIe 传回 GPU。受限于 PCIe 带宽，GPU 经常处于等待 I/O 的空闲状态（占比约 61%）。
    *   **协同注意力方法（如 HGCA）：** 在 CPU 上直接计算部分注意力。由于 CPU 计算速度远慢于 GPU，导致 GPU 等待 CPU 计算结果（占比约 57%）。
*   **研究动机：** 开发一种既能消除 I/O 瓶颈，又能掩盖 CPU 计算延迟的协同推理框架。

### 2. 方法论：核心思想与关键技术
ScoutAttention 提出了 **GPU-CPU 协同计算 + 提前一层预计算** 的架构，核心包含三个关键技术：
*   **GPU-CPU 协同块稀疏注意力：** 
    *   将 KV 缓存分块，仅在 GPU 上保留块摘要（Digest）和少量关键块，其余卸载至内存。
    *   利用块稀疏性，GPU 识别出 Top-k 重要块。若重要块在内存中，则由 CPU 就地计算，最后将结果传回 GPU 合并。
*   **层前 CPU 预计算（Layer-Ahead CPU Pre-computation）：** 
    *   **核心创新：** 为了掩盖 CPU 延迟，ScoutAttention 在 GPU 处理第 $i$ 层时，利用第 $i$ 层的输入 $X_i$ 和第 $i+1$ 层的权重提前预测第 $i+1$ 层的查询向量（Query）。
    *   **流程：** 提前预测出下一层需要 CPU 计算的块，并异步启动 CPU 计算。当 GPU 真正运行到下一层时，CPU 的结果往往已经准备好，实现了计算流水线化。
*   **异步周期性 KV 缓存召回：** 
    *   由于解码过程中重要块会发生偏移（Drift），系统会定期将新的重要块从内存召回至 GPU。
    *   **异步设计：** 召回操作在层计算完成后触发，利用整个 Token 解码的时间窗口进行传输，避免阻塞推理主路径。
