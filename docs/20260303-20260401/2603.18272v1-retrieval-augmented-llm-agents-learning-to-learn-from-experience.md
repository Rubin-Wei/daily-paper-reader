## 摘要
虽然大语言模型（LLMs）推动了通用智能体的发展，但实现对未知任务的鲁棒泛化仍是一个重大挑战。目前的方法通常依赖于微调或利用检索经验的免训练记忆增强生成；然而两者都有局限性：微调往往难以外推到新任务，而经验检索的表现通常不如监督基线。在这项工作中，我们提出将这些方法结合起来，并系统地研究如何训练检索增强型 LLM 智能体，以在上下文中有效地利用检索到的轨迹。首先，我们使用 LoRA 建立了一个鲁棒的有监督微调（SFT）方案，其性能优于几种最先进的智能体训练流水线。其次，我们对经验检索的关键设计选择进行了详细分析，确定了存储、查询和轨迹选择的最优策略。最后，我们提出了一个将经验检索集成到微调过程中的流水线。我们的结果表明，这种结合方法显著提高了对未知任务的泛化能力，为构建能够从经验中学习的智能体提供了一个可扩展且有效的框架。

## Abstract
While large language models (LLMs) have advanced the development of general-purpose agents, achieving robust generalization to unseen tasks remains a significant challenge. Current approaches typically rely on either fine-tuning or training-free memory-augmented generation using retrieved experience; yet both have limitations: fine-tuning often fails to extrapolate to new tasks, while experience retrieval often underperforms compared to supervised baselines. In this work, we propose to combine these approaches and systematically study how to train retrieval-augmented LLM agents to effectively leverage retrieved trajectories in-context. First, we establish a robust supervised fine-tuning (SFT) recipe using LoRA that outperforms several state-of-the-art agent training pipelines. Second, we provide a detailed analysis of key design choices for experience retrieval, identifying optimal strategies for storage, querying, and trajectory selection. Finally, we propose a pipeline that integrates experience retrieval into the fine-tuning process. Our results demonstrate that this combined approach significantly improves generalization to unseen tasks, providing a scalable and effective framework for building agents that learn to learn from experience.

### 1. 核心问题与整体含义（研究动机和背景）
论文关注的核心问题是 **LLM 智能体在面对未见任务（Out-of-Distribution, OOD）时的泛化能力不足**。
*   **背景**：目前的 LLM 智能体主要依靠提示词工程（Prompting）或监督微调（SFT）。SFT 虽然在分布内任务表现强劲，但往往难以迁移到新任务；而现有的“记忆增强”方法（如检索过去经验）虽然灵活，但在性能上通常无法与经过微调的基准模型竞争。
*   **动机**：作者认为，一个真正实用的智能体应该具备“从经验中学习”的能力。因此，研究重点在于如何将**检索增强生成（RAG）**与**参数高效微调（LoRA）**结合，使模型不仅能记住知识，还能学会如何在推理时利用检索到的历史轨迹来解决新问题。

### 2. 方法论：ExpRAG 与 ExpRAG-LoRA
论文提出了 **ExpRAG** 框架，其核心思想是将智能体的历史交互轨迹视为“经验”，并将其转化为可检索的上下文。

*   **核心流程**：
    1.  **经验库构建**：离线收集智能体的交互轨迹（包括成功和失败的），将每条轨迹编码为向量存入索引。
    2.  **检索机制**：在推理的每一步（或初始步），根据当前任务描述和历史记录生成查询向量，检索 Top-K 条最相关的历史轨迹。
    3.  **上下文增强**：将检索到的轨迹格式化为 JSON 或文本块，注入到 LLM 的系统提示词（System Prompt）中。
    4.  **ExpRAG-LoRA（关键创新）**：在微调阶段，不仅给模型提供目标动作，还**同时提供检索到的参考轨迹**。这迫使模型在训练期间就学会“参考”外部经验，从而在测试阶段面对陌生任务时，能更好地模仿检索到的类似策略。

### 3. 实验设计
*   **数据集/场景**：
    *   **ALFWorld**：文本化的家庭交互环境（如：把蜡烛放进抽屉）。
    *   **ScienceWorld**：更复杂的科学实验环境（如：改变物质状态、测试导电性）。
*   **任务划分**：将任务分为 **Easy（简单/分布内）** 和 **Hard（困难/分布外）**。模型仅在 Easy 任务上训练，在 Hard 任务上评估泛化性。
*   **对比方法**：
    *   **免训练基准**：Zero-shot, ReAct, 以及多种 SOTA 记忆增强方法（如 Mem0, A-MEM, Reflexion）。
    *   **微调基准**：标准的 LoRA 微调、NAT、IWM、ETO 等。
*   **消融变量**：Top-K 数量（1, 2, 4）、检索模式（静态 vs 动态刷新）、索引组成（全量、仅简单、仅困难）、轨迹格式（JSON vs 文本）。

### 4. 资源与算力
*   **硬件**：所有模型训练均在 **单张 80 GB NVIDIA A100 GPU** 上完成。
*   **软件框架**：使用 **TorchTune** 进行 LoRA 微调，使用 Hugging Face **Transformers** 进行推理。
*   **训练时长**：论文提到智能体微调需要比常规 NLP 任务更多的 Epoch（ALFWorld 训练 9-50 个 Epoch，ScienceWorld 训练 29-50 个 Epoch）。

### 5. 实验数量与充分性
*   **实验规模**：非常充分。作者测试了 4 种主流开源模型（Ministral 3-8B, Gemma 3-4B, Qwen 2.5-7B, Qwen 2.5-7B-1M）。
*   **消融实验**：针对检索的每一个环节（检索频率、数量、格式、索引质量）都做了详尽的对比。
*   **客观性**：作者首先建立了一个极强的 LoRA 基线（在 ALFWorld 上达到 94.1% 成功率，远超之前论文报告的 80% 左右），确保了后续对比的公平性和结论的可靠性。

### 6. 主要结论与发现
*   **ExpRAG-LoRA 的优越性**：在未见过的 Hard 任务上，ExpRAG-LoRA 显著优于纯 LoRA。例如在 ALFWorld Hard 任务中，纯 LoRA 表现崩溃，而 ExpRAG-LoRA 保持了极高的成功率。
*   **延迟泛化（Grokking）现象**：作者发现智能体微调中存在“延迟泛化”，即在验证集 Loss 开始上升（看似过拟合）后，模型在环境中的实际任务成功率反而会继续提升。
*   **动态检索的局限**：每一步都重新检索（动态模式）虽然理论上更精准，但可能引入“上下文波动”，导致模型表现不稳定，尤其是在索引质量不高时。
*   **长上下文模型优势**：Qwen 2.5-7B-1M 等长上下文模型在增加检索数量（Top-K）时表现出更好的扩展性。

### 7. 优点
*   **简单且强大**：不依赖复杂的记忆读写控制器或强化学习，仅通过 RAG + SFT 的组合就刷新了多项基准。
*   **强基线意识**：论文指出许多复杂方法其实不如一个优化良好的简单微调基线，这种“回归基础”的研究态度对社区很有价值。
*   **实用性高**：证明了即使在推理时没有完美的专家演示，保留训练时的索引也能提供一定的泛化帮助。

### 8. 不足与局限
*   **依赖专家轨迹**：实验中的经验库是由环境提供的专家策略生成的。在现实场景中，获取高质量的专家轨迹可能成本很高。
*   **索引缺失风险**：当推理任务与经验库完全无关时，ExpRAG-LoRA 的表现会大幅下降，甚至不如普通 LoRA（分布偏移）。
*   **计算开销**：由于在 Prompt 中注入了多条长轨迹，推理时的 Token 消耗和延迟显著增加（虽然论文证明了步数会减少，但单步成本依然较高）。
