## 摘要
检索增强生成（RAG）系统引入了一个关键漏洞：上下文泄露，即攻击者利用指令遵循能力，通过自适应提取手段窃取个人身份信息（PII）。目前的防御方案在语义效用和延迟之间存在难以调和的权衡。我们提出了 SEAL-Tag，这是一种通过“先验证后路由（Verify-then-Route）”范式解决该问题的隐私保护运行时环境。SEAL-Tag 引入了 SEAL-Probe 协议，将审计转化为结构化的工具调用操作，使模型在生成草稿的同时产生一份可验证的 PII 证据表（PET）。为了裁定这些证据，我们采用了概率电路（PC）来强制执行可验证的逻辑约束，从而实现稳健的决策。为解决隐私“冷启动”问题，我们引入了 S0--S6 锚定合成流水线，用于生成高保真且具有溯源信息的 RAG 交互数据。我们还配合使用了两阶段课程学习方案，首先优化实体检测能力，随后使模型对齐严格的审计协议。评估结果表明，SEAL-Tag 确立了新的帕累托前沿，在保持与不安全基线相当的效用和速度的同时，将自适应泄露降低了 8 倍以上。

## Abstract
Retrieval-Augmented Generation (RAG) systems introduce a critical vulnerability: contextual leakage, where adversaries exploit instruction-following to exfiltrate Personally Identifiable Information (PII) via adaptive extraction. Current defenses force a rigid trade-off between semantic utility and latency. We present SEAL-Tag, a privacy-preserving runtime environment that resolves this via a Verify-then-Route paradigm. SEAL-Tag introduces the SEAL-Probe protocol, transforming auditing into a structured tool-use operation where the model generates a verifiable PII-Evidence Table (PET) alongside its draft. To adjudicate this evidence, we employ a Probabilistic Circuit (PC) that enforces verifiable logical constraints for robust decision-making. To overcome the privacy "Cold Start" problem, we introduce the S0--S6 Anchored Synthesis Pipeline, generating high-fidelity, provenanced RAG interactions. We pair this with a Two-Stage Curriculum that first optimizes for entity detection before aligning the model to the rigorous audit protocol. Our evaluation demonstrates that SEAL-Tag establishes a new Pareto frontier, reducing adaptive leakage by over 8$\times$ while matching the utility and speed of unsafe baselines.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究背景**：检索增强生成（RAG）已成为企业 AI 的主流架构，但其“信任边界”模糊。系统会将私密文档（如医疗记录、财务日志）直接喂给大模型（LLM），攻击者可通过指令遵循能力诱导模型泄露这些个人身份信息（PII）。
*   **核心问题**：现有的隐私防御手段存在“不可能三角”：
    1.  **脱敏器（Scrubbers）**：如 Microsoft Presidio，虽快但缺乏语义理解，容易误删（如将“华盛顿州”误认为人名），破坏回答质量。
    2.  **黑盒拒绝（Black Boxes）**：如 Llama Guard，不透明且容易被“越狱”绕过。
    3.  **LLM 裁判（LLM Judges）**：如 GPT-4 审计，虽准确但延迟极高，不适合实时应用。
*   **研究目标**：开发一种既能保持高语义效用、具备可审计性，又能实现微秒级决策延迟的隐私保护 RAG 框架。

### 2. 方法论：核心思想与关键技术
SEAL-Tag 提出了 **“先验证后路由”（Verify-then-Route）** 的运行时环境，核心流程如下：

*   **三阶段生成契约（Three-Block Contract）**：
    1.  **草稿阶段（Draft）**：模型先生成一个候选回答 `<ANSWER>`，不进行内部抑制，以保持推理能力。
    2.  **审计阶段（SEAL-Probe）**：模型像调用工具一样，生成一个结构化的 **PII 证据表（PET）**。PET 包含实体溯源（来源哪个文档）、链接风险（多个属性组合是否能识别个人）、共识信号等维度。
    3.  **决策阶段（Final）**：外部决策头根据 PET 决定是允许（ALLOW）、脱敏（MASK）还是拒绝（REFUSE）。
*   **概率电路（Probabilistic Circuit, PC）决策头**：
    *   **核心思想**：放弃不透明的神经网络分类器，改用 PC。PC 是一种可处理的生成模型，能够强制执行**单调性逻辑约束**（例如：检测到的 PII 越多，风险得分必须越高）。
    *   **优势**：计算是确定性的、数学可解释的，且推理速度极快（微秒级）。
*   **S0–S6 锚定合成流水线**：
    *   为了解决隐私数据无法公开获取的“冷启动”问题，该流水线从 PII 锚定开始，模拟世界背景、注入对抗性工件、合成检索段落，最后生成带有金标准标签的指令微调数据。
*   **两阶段课程学习（Two-Stage Curriculum）**：
    1.  **阶段 I：PII 感知 SFT**。优化模型对各类稀有 PII 模式（如 IMEI、加密地址）的敏感度。
    2.  **阶段 II：协议对齐 SFT**。训练模型严格遵守 PET 的 JSON 格式和溯源逻辑，解决“格式崩溃”问题。

### 6. 主要结论与发现
*   **安全性**：SEAL-Tag 将自适应泄露风险（ASR）降低了 **8 倍以上**（在 CopyBreakRAG 攻击下从 81.9% 降至 9.5%）。
*   **效用保持**：在 PopQA 等任务上，SEAL-Tag 的准确率几乎与无防御模型持平（51.07% vs 51.26%），成功消除了“隐私税”。
*   **决策质量**：概率电路（PC）的预期校准误差（ECE）仅为 0.03，远优于神经分类器（0.16），有效避免了“过度自信的泄露”。
*   **效率**：系统总延迟开销仅为 **+18ms**，而 LLM-as-a-Judge 方案需要 +1450ms，证明了其在边缘设备部署的可行性。

### 7. 优点（亮点）
*   **架构创新**：将隐私审计从“生成任务”转变为“结构化工具调用”，通过 PET 提供了清晰的推理链。
*   **逻辑严密**：引入概率电路（PC）保证了决策的单调性和可验证性，这是传统深度学习模型难以实现的。
*   **数据合成方案**：S0-S6 流水线为隐私研究提供了一种不依赖真实敏感数据的高质量训练方案。
*   **高性能**：在极低延迟下实现了极高的安全增益，打破了 RAG 隐私保护的性能瓶颈。

### 8. 不足与局限
*   **模型依赖性**：SEAL-Tag 依赖于模型生成复杂 JSON 的能力，对于参数量极小的模型（如 1B 以下），可能会出现格式对齐困难。
*   **合成数据偏差**：虽然 S0-S6 尽量模拟真实世界，但合成 PII 与真实复杂环境下的隐私分布仍可能存在细微差异。
*   **多轮对话状态**：虽然提到了链接攻击，但对于超长多轮对话中跨度极大的信息聚合，PET 的追踪能力可能面临挑战。
*   **应用限制**：目前主要针对文本 PII，对于多模态 RAG（如包含图像、图表）的隐私保护尚未涉及。
