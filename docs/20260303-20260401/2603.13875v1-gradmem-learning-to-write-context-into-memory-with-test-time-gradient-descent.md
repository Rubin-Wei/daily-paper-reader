## 摘要
许多大语言模型应用需要基于长上下文进行条件化生成。Transformer 通常通过存储每层过去激活的大型 KV 缓存来支持这一点，这会产生巨大的内存开销。一个理想的替代方案是压缩记忆：一次性读取上下文，将其存储在紧凑状态中，并根据该状态回答多个查询。我们在上下文移除设置下对此进行研究，即模型在推理时必须在无法访问原始上下文的情况下生成答案。我们引入了 GradMem，它通过逐样本的测试时优化将上下文写入记忆。给定上下文，GradMem 在保持模型权重冻结的同时，对一小组前缀记忆标记执行几步梯度下降。与仅前向传播的方法不同，GradMem 显式地优化模型级的自监督上下文重建损失，从而实现具有迭代误差校正的损失驱动写入操作。在关联键值检索任务中，GradMem 在相同记忆大小下优于仅前向传播的记忆写入器，且额外的梯度步骤比重复的前向写入能更有效地扩展容量。我们进一步证明 GradMem 的能力超出了合成基准测试：配合预训练语言模型，它在包括 bAbI 和 SQuAD 变体在内的自然语言任务上取得了具有竞争力的结果，且仅依赖于编码在记忆中的信息。

## Abstract
Many large language model applications require conditioning on long contexts. Transformers typically support this by storing a large per-layer KV-cache of past activations, which incurs substantial memory overhead. A desirable alternative is ompressive memory: read a context once, store it in a compact state, and answer many queries from that state. We study this in a context removal setting, where the model must generate an answer without access to the original context at inference time. We introduce GradMem, which writes context into memory via per-sample test-time optimization. Given a context, GradMem performs a few steps of gradient descent on a small set of prefix memory tokens while keeping model weights frozen. GradMem explicitly optimizes a model-level self-supervised context reconstruction loss, resulting in a loss-driven write operation with iterative error correction, unlike forward-only methods. On associative key--value retrieval, GradMem outperforms forward-only memory writers with the same memory size, and additional gradient steps scale capacity much more effectively than repeated forward writes. We further show that GradMem transfers beyond synthetic benchmarks: with pretrained language models, it attains competitive results on natural language tasks including bAbI and SQuAD variants, relying only on information encoded in memory.

### 1. 核心问题与整体含义（研究动机和背景）
大语言模型（LLM）在处理长上下文时，通常依赖 KV-cache 存储过去的所有激活值。这导致了巨大的内存开销，且难以生成可移植、可复用的上下文表示。现有的“压缩记忆”方案（如 RMT）多采用单向前向传播（Forward-only）将信息写入固定大小的向量，但这种方式缺乏反馈机制，无法纠正写入过程中的错误。
**GradMem 的核心动机**是：借鉴“测试时训练”（Test-Time Training, TTT）的思想，将记忆写入过程视为一个优化问题。通过在推理阶段对一小组记忆标记（Memory Tokens）进行梯度下降，使模型能够迭代地校正误差，从而在极小的内存空间内“挤进”更多的上下文信息。

### 2. 方法论：核心思想与关键技术
GradMem 提出了一种基于梯度下降的上下文写入机制，其核心流程分为 **WRITE（写入）** 和 **READ（读取）** 两个阶段：

*   **核心思想**：将记忆表示为 $m$ 个 $d$ 维的可学习向量 $M$（作为 Transformer 的前缀嵌入）。在处理特定上下文 $C$ 时，保持模型权重 $\theta$ 冻结，仅通过梯度下降更新 $M$。
*   **WRITE 阶段（自监督重构）**：
    *   定义一个模型层的自监督目标：**上下文重构损失** $L_{write}$。即给定当前的记忆 $M$ 和已有的上下文片段，预测下一个上下文标记的交叉熵损失。
    *   **更新公式**：从元学习得到的初始值 $M_0$ 开始，执行 $K$ 步梯度更新：$M_{k+1} = M_k - \alpha \nabla_{M_k} L_{write}(M_k; C)$。
    *   这种方式让模型能够识别哪些信息尚未被编码（损失高的地方），并针对性地更新记忆。
*   **READ 阶段（下游任务预测）**：
    *   原始上下文 $C$ 被移除。模型仅基于优化后的记忆 $\hat{M}$ 和新的查询 $Q$ 来预测目标 $Y$。
*   **元学习训练（Meta-Learning）**：
    *   在训练阶段，通过对 WRITE 阶段的优化路径进行反向传播（二阶导数），同时优化模型参数 $\theta$ 和记忆初始值 $M_0$，使模型学会如何通过少量的梯度步数（通常 $K \le 5$）有效地存储信息。

### 6. 主要结论与发现
*   **梯度胜过前向**：在相同记忆大小下，梯度更新写入的信息量显著超过纯前向传播（RMT）。
*   **计算换容量**：增加梯度步数 $K$ 能稳定提升记忆容量，而简单地重复前向读取上下文（Rereading）收益有限且不稳定。
*   **推理时可扩展**：即使训练时只用了 2 步梯度，推理时增加到 10 步或更多仍能继续提升准确率，展现了良好的外推性。
*   **选择性记忆**：分析发现，GradMem 在优化过程中会优先降低“Value”标记的损失，而对“Key”标记保持较高损失，说明它学会了结构化压缩，优先存储对回答问题更有用的信息。

### 7. 优点（亮点）
*   **理论优雅**：将记忆写入统一到优化框架下，解决了前向写入模型缺乏反馈的问题。
*   **高效压缩**：能够将数百个标记的信息压缩进极少数（如 8 个）向量中，压缩率极高。
*   **任务无关性**：WRITE 阶段使用通用的重构损失，不需要针对特定下游任务设计写入规则。
*   **工程优化**：针对二阶导数计算提出了高效的注意力机制实现，具有实际参考价值。

### 8. 不足与局限
*   **训练成本高**：元学习（Meta-learning）需要通过优化路径求导，训练速度慢且显存压力大，难以直接扩展到超大规模参数模型。
*   **推理延迟**：虽然 READ 阶段很快，但 WRITE 阶段需要多次梯度计算，对于实时性要求极高的单次查询场景可能不划算（除非上下文被多次复用）。
*   **模型规模限制**：实验主要集中在 100M-200M 级别的模型，尚未在大规模参数（如 7B+）模型上验证其性能增益是否依然显著。
*   **重构目标的局限**：简单的标记重构可能不是最优的自监督目标，对于需要深层语义理解的任务，可能需要更复杂的 WRITE 损失函数。
